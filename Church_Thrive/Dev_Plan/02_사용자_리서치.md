## 핵심 사용자 인사이트

### 1. 말씀노트 STT = 게이트웨이 기능
- 모든 교인이 매주 예배에서 사용할 수 있는 기능
- 개인 사용(B2C)에서 시작 → 교회 전체 채택(B2B)으로 확산
- **가장 높은 사용 빈도** = 앱 습관 형성의 핵심

### 2. 사무간사 = 파워유저
- 가장 많은 시간을 시스템에서 보내는 사용자
- Excel 임포트/대량 데이터 입력이 원활하지 않으면 **전체 도입 실패**
- PC 환경 최적화 필수 (PWA 반응형)

### 3. 카카오톡 통합 = 필수
- 한국 교회의 모든 커뮤니케이션은 카카오톡 기반
- 카카오 로그인, 알림톡, 공유 기능은 **대체 불가**
- 카카오톡이 안 되면 앱을 안 쓴다

### 4. 오프라인 지원 = 생존 요건
- 일요일 오전 예배 시간은 네트워크 부하 최대
- 교회 내부 Wi-Fi 불안정 환경 일반적
- 녹음/노트 작성은 **100% 오프라인 동작** 필수

### 5. 접근성 = 도입 성패
- 60대 이상 교인 비율이 높은 한국 교회 특성
- 최소 16px 폰트, 큰 터치 영역, 단순한 네비게이션
- 전화번호 OTP 로그인 옵션 필수

## 사용자 여정 맵

### 주요 사용자 여정: 말씀노트 (일반교인)

```
[예배 전]
  교인이 앱 실행 → 말씀노트 탭 → 녹음 시작 (1탭)
    ↓
[예배 중]
  설교 녹음 진행 (오프라인 동작)
  + 텍스트 메모 동시 입력 가능
    ↓
[예배 후]
  녹음 중지 → 자동 저장
  온라인 연결 시 → Supabase Storage 업로드
    ↓
[STT 변환] (Phase 2)
  Edge Function → Google Cloud STT API 호출
  → 후처리 (성경용어 교정)
  → 알림 (변환 완료)
    ↓
[정리 & 공유]
  변환된 텍스트 + 메모 통합 편집
  → 성경 구절 자동 링크
  → 카카오톡 공유
    ↓
[교역자 피드백]
  담당 교역자가 말씀노트 열람
  → 피드백/코멘트 작성
  → 성도에게 알림
```

### 주요 사용자 여정: 교회 가입

```
[신규 교인]
  앱 다운로드 → 아이디(이메일/전화번호) + 비밀번호 입력
    ↓
[교회 선택]
  소속 교회 검색 (교회명/주소)
  → 교회 선택
    ↓
[승인 대기]
  가입 요청 → 교회 관리자/교역자에게 알림
  → 대기 화면 표시 (앱 기능 접근 불가)
    ↓
[승인 완료]
  관리자가 승인 → 교회 멤버로 등록
  → 본인 교적 데이터 입력/수정 가능
  → 앱 전체 기능 접근 가능
```
