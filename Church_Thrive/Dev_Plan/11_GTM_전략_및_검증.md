# 11. Go-to-Market 전략 및 검증 계획

## Go-to-Market 전략

### 1. 부트캠프 우선 전략

- 기존 ChurchThrive 15일 부트캠프 졸업생 **127명** → 첫 베타 테스터
- 졸업생 네트워크를 통한 초기 교회 확보
- Day 1-3에 앱 온보딩 포함하여 자연스러운 도입
- 졸업생 교회 = 최초 레퍼런스 케이스

### 2. 말씀노트로 진입 (B2C2B)

```
[전략 플로우]

개별 교인이 말씀노트 사용 (B2C)
     │
     ▼
교인이 교회 내 공유 → 다른 교인 사용
     │
     ▼
교회 전체 채택 (B2B) → 유료 전환
```

- **B2C2B 모델**: 개인(교인) → 커뮤니티(교회) → 기관(유료)
- 말씀노트 STT = 게이트웨이 기능 (매주 사용 → 습관 형성)
- 개인 사용에서 시작 → 교회 전체 도입으로 확산

### 3. PWA + 앱 동시 출시

- 웹(PWA): 설치 없이 브라우저에서 바로 사용
- 네이티브 앱: App Store / Google Play
- **최대 접근성 확보**: 어떤 기기에서든 접근 가능

### 4. 교단 파트너십

- 예장통합, 감리교 등 주요 교단 MOU 추진
- 교단 차원의 일괄 도입 제안
- 교단 특화 기능 (교단별 교적 양식 등) 제공

### 5. 15일 부트캠프 통합

- 부트캠프 커리큘럼에 앱 활용 교육 포함
- Day 1-3: 앱 설치 + 교인관리 데이터 입력
- Day 4-7: 말씀노트 + 행정관리 실습
- Day 8-15: 전체 기능 활용 + 커스터마이징

### 6. 무료 인프라 강조

- **"교회 부담 제로"** 메시지
- 무료 티어: 50명 이하 교회 완전 무료
- 무료 교회 홈페이지 제공 = 강력한 진입 인센티브

### 테스트 전략

#### 단위 테스트
- **도구**: Vitest
- **커버리지 목표**: 비즈니스 로직 80%
- **대상**: DB 쿼리, RLS 정책, 데이터 변환, 유효성 검증

#### E2E 테스트
- **도구**: Playwright
- **핵심 사용자 여정**:
  - 가입 → 교회 선택 → 승인 → 로그인
  - 교인 등록 (QR + 수동)
  - 말씀노트 작성 (녹음 + 텍스트)
  - 출석 체크 (QR + 수동)
  - 공지 작성 → 알림 발송

#### 부하 테스트
- **도구**: k6
- **시나리오**: 일요일 오전 피크 시뮬레이션
  - 동시 접속: 1,000명
  - 말씀노트 동시 작성: 500명
  - 출석 체크 동시: 300명
- **목표**: p95 응답 시간 < 200ms

#### 보안 테스트
- **도구**: OWASP ZAP + Snyk
- **대상**:
  - RLS 정책 검증 (교회 간 데이터 격리)
  - API 인증/인가 테스트
  - SQL Injection, XSS 등 OWASP Top 10
  - 의존성 취약점 스캔

#### 파일럿 테스트
- **대상**: 3-5개 교회 실사용 (MVP 출시 전)
- **기간**: 2-4주
- **수집 데이터**:
  - 사용 빈도, 기능별 사용률
  - 에러/크래시 리포트
  - 사용자 피드백 (인터뷰 + 설문)
  - NPS 점수

### 성공 지표 (파일럿 기준)

| 지표 | 목표 |
|------|------|
| 주간 활성 사용률 | > 60% |
| 말씀노트 주간 작성률 | > 40% |
| 크래시율 | < 1% |
| NPS | > 30 |
| 재사용 의향 | > 80% |
| 유료 전환 의향 | > 20% |

## 피드백 루프

```
[파일럿 사용] → [데이터 수집] → [분석]
                                    │
                  ┌─────────────────┤
                  ▼                 ▼
           [기능 개선]        [UX 개선]
                  │                 │
                  └────────┬────────┘
                           ▼
                    [다음 릴리스]
                           │
                           ▼
                    [파일럿 재검증]
```

- 2주 스프린트 사이클
- 매 스프린트 끝 파일럿 교회 피드백 수집
- 우선순위 재조정 → 다음 스프린트 반영
