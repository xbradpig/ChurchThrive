## 상세 리스크 분석 및 완화 방안

### R1: 한국어 STT 정확도 부족 (심각도: 높음)

**설명**: 한국어 설교는 성경 용어, 고유명사, 한영 혼합 등 일반 한국어와 다른 특성이 있어 STT 정확도가 낮을 수 있음.

**영향**: STT가 핵심 차별화 요소이므로, 정확도가 낮으면 사용자 이탈 → 유료 전환 실패.

**완화 방안**:
1. **3중 STT 폴백**: Google Cloud STT → Whisper API → CLOVA Speech
2. **성경 용어 사전**: 커스텀 사전으로 성경 고유명사 교정
3. **사용자 교정 UX**: 변환된 텍스트를 사용자가 쉽게 수정할 수 있는 UI
4. **지속적 개선**: 교정 데이터 수집 → 후처리 로직 개선
5. **기대치 관리**: "보조 도구"로 포지셔닝, 100% 정확도를 약속하지 않음

**모니터링**: STT 정확도 메트릭 대시보드, 사용자 교정 빈도 추적

---

### R2: 사무간사 미채택 (심각도: 높음)

**설명**: 사무간사는 교회 행정의 핵심 인력으로, 이 사용자가 시스템을 채택하지 않으면 교회 전체 도입이 실패.

**영향**: 교회 단위 유료 전환 불가, 데이터 입력 병목.

**완화 방안**:
1. **Excel 임포트 마법사**: 기존 명부를 드래그앤드롭으로 쉽게 업로드
2. **대량 데이터 입력 최적화**: 키보드 단축키, 탭 네비게이션, 일괄 편집
3. **PC 환경 최적화**: 데스크톱에서의 작업 효율 극대화
4. **교육 자료**: 사무간사 전용 빠른 시작 가이드
5. **1:1 온보딩 지원**: 초기 데이터 마이그레이션 직접 도움

**모니터링**: 사무간사 사용 빈도, 기능별 사용률, 피드백 수집

---

### R3: 일요일 피크 장애 (심각도: 높음)

**설명**: 일요일 오전 예배 시간에 모든 교인이 동시 접속하므로 서버 부하 급증. 장애 발생 시 교회의 신뢰 상실.

**영향**: 서비스 신뢰도 하락 → 교회 이탈.

**완화 방안**:
1. **오프라인 우선 설계**: 녹음/노트/출석 = 100% 오프라인 동작
2. **Cloudflare CDN**: 정적 자산 글로벌 분산
3. **Supabase 자동 스케일링**: Pro 이상 티어에서 자동 확장
4. **사전 부하 테스트**: k6로 피크 시뮬레이션 (매 릴리스)
5. **일요일 특별 모니터링**: 자동 알림 + 대기 인력

**모니터링**: 일요일 오전 전용 대시보드, 응답 시간 실시간 추적

---

### R4: PIPA 위반 (심각도: 높음)

**설명**: 교인 개인정보(이름, 전화번호, 주소, 세례일 등)를 다루므로 개인정보보호법(PIPA) 준수 필수.

**영향**: 법적 제재, 과징금, 서비스 중단 가능.

**완화 방안**:
1. **법률 자문**: PIPA 전문 변호사 검토 (MVP 전)
2. **Privacy-by-Design**: 설계 단계부터 개인정보 보호 반영
3. **동의서 관리**: 수집/이용 동의, 제3자 제공 동의 체계
4. **데이터 암호화**: 민감 정보 AES-256 암호화 저장
5. **접근 로그**: 개인정보 접근 이력 완전 기록
6. **연간 감사**: 정기 개인정보 보호 감사
7. **데이터 삭제권**: 교인 탈퇴 시 완전 삭제 보장

**모니터링**: 접근 로그 감사, 분기별 PIPA 체크리스트 점검

---

### R5: 고령 교인 채택 저조 (심각도: 중간)

**설명**: 60대 이상 교인이 디지털 도구에 익숙하지 않아 앱 사용을 거부하거나 어려워할 수 있음.

**영향**: 말씀노트 STT 기능의 핵심 타겟(예배 참석 교인)이 사용하지 않으면 게이트웨이 전략 실패.

**완화 방안**:
1. **큰 폰트/간단 UI**: 최소 16px, 큰 버튼, 단순 네비게이션
2. **전화번호 OTP**: 이메일 없이 전화번호만으로 로그인
3. **카카오톡 기반**: 카카오톡은 60대도 사용하는 필수 앱
4. **녹음 버튼 원탭**: 가장 많이 쓰는 기능은 원탭으로 시작
5. **교회 내 교육**: 부트캠프/교회 차원의 사용법 교육 지원
6. **가족 도움**: 자녀 세대가 부모 세대 앱 설정 도움

**모니터링**: 연령별 사용률, 60대 이상 이탈률, 기능별 난이도 피드백

---

### R6: Supabase 무료 한도 초과 (심각도: 중간)

**설명**: Supabase 무료 티어 (500MB DB, 1GB Storage)를 초과하면 유료 전환 필요.

**영향**: 예상보다 빠른 비용 증가.

**완화 방안**:
1. **유료 전환 시점 모니터링**: DB/Storage 사용량 실시간 추적
2. **Storage 최적화**: 오디오 압축 (Opus 코덱), 이미지 리사이징
3. **Supabase Pro**: $25/월로 8GB DB + 100GB Storage (매우 합리적)
4. **단계적 스케일링**: 500교회 → Pro, 2,000교회 → Team
5. **오래된 데이터 아카이브**: 비활성 데이터 별도 저장소 이관

**모니터링**: Supabase Dashboard 주간 점검, 용량 80% 알림

---

### R7: 무료 도구 경쟁 (심각도: 중간)

**설명**: 만나 등 무료 교회 관리 앱이 존재하며, 대형 플랫폼(네이버/카카오)이 진입할 가능성.

**영향**: 유료 전환 저항, 시장 점유율 확보 어려움.

**완화 방안**:
1. **STT 차별화**: 경쟁사에 없는 AI STT 기능으로 차별화
2. **LVI 교회건강진단**: ChurchThrive 고유 컨설팅 자산
3. **훈련관리 통합**: 행정 + 영적성장 도구 통합은 유일
4. **컨설팅 생태계**: 앱 + 부트캠프 + 컨설팅 번들
5. **교회 홈페이지**: 무료 홈페이지 제공으로 초기 가입 유인

**모니터링**: 경쟁사 동향 분기별 분석, 사용자 이탈 사유 추적

---

### R8: 예산 초과 (심각도: 낮음)

**설명**: 개발 일정 지연 또는 예상 외 비용 발생으로 예산 초과.

**영향**: 자금 소진으로 프로젝트 중단 위험.

**완화 방안**:
1. **Supabase/Cloudflare 무료 티어**: 인프라 비용 최소화
2. **MVP 범위 관리**: 핵심 기능만 먼저 출시, 확장은 Phase별 진행
3. **2주 스프린트**: 짧은 사이클로 진행 상황 지속 확인
4. **비용 트래킹**: 주간 비용 리뷰
5. **예비비**: 총 예산의 10% 예비비 확보

**모니터링**: 주간 번다운 차트, 월간 예산 대비 실적 리뷰
