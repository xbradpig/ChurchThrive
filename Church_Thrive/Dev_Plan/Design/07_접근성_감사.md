# 07. 접근성 감사 및 가이드라인

---

## 1. 접근성 감사 개요

### 1.1 적용 기준

| 항목 | 기준 |
|------|------|
| **국제 표준** | WCAG 2.1 Level AA |
| **한국 법률** | 장애인차별금지 및 권리구제 등에 관한 법률 (장애인차별금지법) |
| **인증** | 한국 웹 접근성 인증 (WA 인증) 준비 |
| **모바일 표준** | 모바일 앱 접근성 지침 (한국정보화진흥원) |
| **플랫폼 가이드** | Apple Human Interface Guidelines (접근성), Material Design Accessibility |

### 1.2 ChurchThrive 특별 고려 사항

ChurchThrive는 일반 앱과 달리 **한국 교회 고령 교인(60대+)**이 핵심 사용자에 포함된다. 페르소나 4 '최영자 집사(63세)'는 디지털 리터러시가 낮고 기본 앱만 사용하며, **갤럭시 A 시리즈**(중저가 안드로이드)를 사용한다.

따라서 WCAG 2.1 AA 기준 외에 다음 사항을 추가로 적용한다.

- 최소 폰트 16px (WCAG 기준 14px보다 2px 상향)
- 터치 타겟 최소 48x48px (WCAG 기준 44x44px보다 4px 상향)
- 단순 제스처만 사용 (최대 1-finger gesture)
- 글자 크기 조절 기능 내장 (시스템 설정 외 앱 자체 기능)
- 첫 사용 시 가이드 오버레이 제공

### 1.3 법적 준수 요건

#### 장애인차별금지법 관련

- 장애인차별금지법 제21조: 정보통신 접근에서의 차별 금지
- 장애인차별금지법 시행령 제14조: 웹사이트 및 모바일 앱 접근성 보장
- 위반 시 시정권고 및 과태료 부과 가능
- 공공기관 필수, 민간 기업도 권고 (2025년 이후 단계적 의무화 추세)

#### 웹 접근성 인증 마크 (WA 인증)

- 한국웹접근성인증평가원(KWACC) 또는 지정 인증기관에서 심사
- 심사 기준: 한국형 웹 콘텐츠 접근성 지침 2.2 (KWCAG 2.2)
- 인증 유효기간: 1년 (갱신 필요)
- Phase 3에서 인증 취득 목표

**규칙**:
- Body 텍스트는 절대 16px 미만 사용 금지
- Caption(12px)은 비필수 정보(타임스탬프 등)에만 사용
- 필수 정보는 반드시 14px 이상
- 폰트: **Pretendard** (가변 폰트, 한글 최적화)

### 2.3 폰트 확대 지원

시스템 폰트 크기 설정과 연동하여 앱 내 텍스트를 비례적으로 확대한다.

| 시스템 설정 | 배율 | Body 실제 크기 | Heading 1 실제 크기 |
|------------|------|---------------|-------------------|
| 가장 작게 | 85% | 13.6px | 20.4px |
| 작게 | 90% | 14.4px | 21.6px |
| **기본** | **100%** | **16px** | **24px** |
| 크게 | 115% | 18.4px | 27.6px |
| 가장 크게 | 135% | 21.6px | 32.4px |
| 특대 | 150% | 24px | 36px |
| 최대 | 200% | 32px | 48px |

**구현 방식**:
- React Native: `PixelRatio.getFontScale()` 활용
- PWA: CSS `rem` 기반 + 시스템 `font-size` 연동
- 최대 200%까지 지원 (레이아웃 깨짐 방지 테스트 필수)
- 폰트 확대 시 레이아웃 재배치: 가로 → 세로 전환 (필요 시)

**앱 자체 글자 크기 조절** (고령 교인 대응):
- 설정 > 글자 크기에서 앱 전용 크기 조절 가능
- [A-] [기본] [A+] 3단계 버튼 제공
- 시스템 설정과 독립적으로 동작

### 2.4 색맹 대응

전 세계 남성의 약 8%, 여성의 약 0.5%가 색각이상을 가지고 있다. 한국 남성도 약 5.9%로 추정된다.

| 정보 유형 | 색상 | 추가 수단 | 예시 |

#### 차트/그래프 색맹 대응

- 원형 차트: 색상 + 패턴(빗금, 점, 격자) 병행
- 막대 차트: 색상 + 데이터 라벨 직접 표시
- 라인 차트: 색상 + 선 스타일(실선, 점선, 파선) 구분

#### 색맹 시뮬레이션 테스트

- 적록 색맹(Protanopia/Deuteranopia): 가장 흔한 유형
- 청황 색맹(Tritanopia): 드물지만 확인 필요
- 도구: Chrome DevTools > Rendering > Emulate vision deficiencies

## 3. 운동/조작 접근성

### 3.1 터치 타겟 크기

WCAG 2.5.5 (AAA) 기준 44x44px이나, ChurchThrive는 고령 사용자를 위해 **48x48px**을 목표로 한다.

| 컴포넌트 | 최소 크기 | 목표 크기 | 비고 |
|----------|----------|-----------|------|
| 버튼 (Primary/Secondary) | 44x44px | 48x52px | 높이 52px 권장 |
| 아이콘 버튼 | 44x44px | 48x48px | 터치 영역 포함 |
| 체크박스 | 44x44px | 48x48px | 터치 영역 시각 크기보다 크게 |
| 라디오 버튼 | 44x44px | 48x48px | 터치 영역 시각 크기보다 크게 |
| 목록 항목 | 44px (높이) | 56px (높이) | 리스트 아이템 전체 행 |
| 네비게이션 탭 | 44x44px | 48x48px (영역) | 아이콘+라벨 포함 |
| FAB (플로팅 버튼) | 48x48px | 56x56px | 충분한 크기 |
| 녹음 버튼 | 56x56px | **64x64px** | 고령 교인 대응, 특대 크기 |
| 토글 스위치 | 44x24px | 52x32px | 조작부 전체 영역 |
| 드롭다운 | 44px (높이) | 52px (높이) | 선택 영역 |

#### 터치 타겟 규칙

```
터치 영역 = max(시각적 크기, 최소 44x44px)

[시각적 크기가 작은 경우]
├── 아이콘: 24x24px 시각 크기 → 48x48px 터치 영역 (padding으로 확장)
├── 체크박스: 20x20px 시각 크기 → 48x48px 터치 영역
└── 텍스트 링크: 터치 영역 좌우 8px, 상하 12px 확장
```

### 3.2 버튼 간격

| 배치 | 최소 간격 | 권장 간격 | 비고 |
|------|----------|-----------|------|
| 인접 버튼 (수평) | 8px | 12px | 실수 터치 방지 |
| 인접 버튼 (수직) | 8px | 12px | 목록 항목 간 |
| 모달 버튼 ([취소] [확인]) | 12px | 16px | 의도적 구분 |
| 파괴적 동작 버튼 | 16px | 24px | 삭제/취소 등 (오터치 방지) |

### 3.3 스와이프 대안

모든 스와이프 제스처에는 반드시 **버튼 대안**을 제공한다.

| 스와이프 동작 | 기능 | 버튼 대안 |
|--------------|------|-----------|
| 목록 항목 좌 스와이프 | 삭제/보관 | 롱프레스 → 컨텍스트 메뉴 → [삭제] |
| 말씀노트 섹션 좌/우 스와이프 | 섹션 전환 | 상단 탭 바 직접 터치 |
| 바텀시트 하향 스와이프 | 시트 닫기 | [닫기] 버튼 (우상단) 또는 배경 터치 |
| 풀투리프레시 (하향 스와이프) | 데이터 새로고침 | [새로고침] 버튼 (우상단) |
| 카드 스와이프 | 카드 간 이동 | [이전] [다음] 버튼 |

### 3.4 한 손 조작 (Thumb Zone)

스마트폰을 한 손으로 잡았을 때 엄지가 편하게 닿는 영역(Thumb Zone)에 핵심 기능을 배치한다.

```
┌─────────────────────────┐
│  Hard to Reach (상단)    │  ← 부가 정보, 헤더, 뒤로가기
│  (화면 상단 1/3)         │
├─────────────────────────┤
│  OK Zone (중단)          │  ← 컨텐츠 영역, 스크롤 목록
│  (화면 중단 1/3)         │
├─────────────────────────┤
│  Easy Zone (하단)        │  ← 핵심 동작 버튼, 네비게이션
│  (화면 하단 1/3)         │     녹음 버튼, FAB, 탭 바
│                          │
│  ┌──────────────────┐   │
│  │  하단 네비게이션   │   │
│  └──────────────────┘   │
└─────────────────────────┘
```

**배치 원칙**:
- 핵심 동작 버튼 (녹음, 저장, 확인): 하단 고정
- 네비게이션 탭: 화면 최하단
- FAB: 우측 하단 (엄지 도달 용이)
- 뒤로가기/닫기: 좌상단 (시스템 규칙 따름) + 하향 스와이프 대안
- 파괴적 동작(삭제): 의도적으로 상단 배치 (실수 방지)

### 3.5 제스처 복잡도 제한

| 허용 제스처 | 설명 | 대상 |
|------------|------|------|
| 탭 (1-finger) | 기본 선택 | 모든 사용자 |
| 롱프레스 (1-finger) | 컨텍스트 메뉴 | 모든 사용자 (버튼 대안 필수) |
| 스와이프 (1-finger) | 섹션 전환, 삭제 | 모든 사용자 (버튼 대안 필수) |
| 풀투리프레시 (1-finger) | 새로고침 | 모든 사용자 (버튼 대안 필수) |
| 스크롤 (1-finger) | 목록 이동 | 모든 사용자 |

| 금지 제스처 | 이유 |
|------------|------|
| 핀치 줌 (2-finger) | 고령 사용자 어려움 → [+][-] 버튼 대안 제공 |
| 회전 (2-finger) | 불필요 |
| 3-finger 제스처 | 접근성 기능과 충돌 가능 |
| 복잡한 드래그 | 정밀 조작 어려움 → 버튼 기반 대안 제공 |

## 4. 인지 접근성

### 4.1 단순한 네비게이션: 3탭 규칙

**핵심 원칙**: 어떤 페이지에서든 주요 기능에 **최대 3탭 이내** 도달

| 기능 | 경로 | 탭 수 |
|------|------|-------|
| 말씀노트 녹음 시작 | 홈 → [말씀] 탭 → [+] FAB → 녹음 | **3탭** |
| 출석 체크 | 홈 → [오늘 출석] 카드 → 예배 선택 | **2-3탭** |
| 교회 공지 확인 | 홈 → [알림] 탭 → 공지 항목 | **2탭** |
| 새가족 QR 등록 | 홈 → [교회] 탭 → [QR 등록] | **2탭** |
| 내 프로필 수정 | 홈 → [더보기] → [내 프로필] | **2탭** |
| 설정 | 홈 → [더보기] → [설정] | **2탭** |

**네비게이션 구조 원칙**:
- 최대 깊이: 4단계 (탭 → 리스트 → 상세 → 편집)
- 항상 현재 위치 표시 (활성 탭 하이라이트)
- 뒤로가기로 이전 화면 복귀 가능
- 홈 버튼 항상 접근 가능 (하단 네비게이션)

### 4.2 명확한 레이블

#### 아이콘 + 텍스트 병행 규칙

| 위치 | 아이콘만 허용 | 아이콘 + 텍스트 필수 |
|------|-------------|-------------------|
| 하단 네비게이션 | X | O (아이콘 + 텍스트 라벨) |
| 페이지 헤더 버튼 | O (뒤로가기, 검색, 설정 등 표준 아이콘만) | - |
| 기능 버튼 | X | O (항상 텍스트 라벨 포함) |
| 툴바 (에디터) | O (표준 에디터 아이콘: B, I, U) | 토글 시 툴팁 표시 |
| 탭 | X | O (텍스트 라벨 필수) |
| FAB | O (+ 아이콘, 충분히 보편적) | 첫 사용 시 툴팁 |

**규칙**: 아이콘만 사용 가능한 경우에도 반드시 **접근성 라벨(aria-label, contentDescription)** 제공

### 4.3 에러 메시지 가이드라인

에러 메시지는 **무엇이 잘못되었는지** + **어떻게 해결하는지** 두 가지를 모두 포함한다.

| 상황 | 나쁜 예 | 좋은 예 |
|------|---------|---------|
| 필수 필드 미입력 | "필수 항목입니다" | "이름을 입력해주세요" |
| 비밀번호 규칙 불충족 | "유효하지 않은 비밀번호" | "비밀번호는 8자 이상, 영문과 숫자를 포함해야 합니다" |
| 네트워크 오류 | "오류가 발생했습니다" | "인터넷 연결을 확인해주세요. Wi-Fi 또는 데이터를 켜고 다시 시도해주세요" |
| 서버 오류 | "500 Error" | "잠시 문제가 발생했습니다. 잠시 후 다시 시도해주세요" |
| QR 인식 실패 | "인식 실패" | "QR 코드를 인식할 수 없습니다. 화면을 밝게 하고, QR이 카메라 중앙에 오도록 해주세요" |
| 녹음 권한 거부 | "권한 없음" | "녹음을 위해 마이크 권한이 필요합니다. [설정으로 이동] 에서 마이크를 허용해주세요" |
| 파일 크기 초과 | "파일 크기 초과" | "사진 크기가 10MB를 초과합니다. 더 작은 사진을 선택해주세요" |
| 중복 출석 체크 | "이미 체크됨" | "이미 출석 체크된 교인입니다 (체크 시간: 오전 9:15)" |

**에러 표시 규칙**:
- 인라인 에러: 해당 필드 바로 아래, 빨간색 텍스트 + 아이콘
- 토스트 에러: 화면 상단에서 내려옴, 빨간색 배경, 3초 표시
- 모달 에러: 복구 불가능한 치명적 에러만 사용
- 에러 필드: 테두리 빨간색 + 흔들림 애니메이션 (200ms)

### 4.4 진행률 표시

사용자가 현재 상태를 항상 알 수 있도록 진행률을 명확히 표시한다.

| 작업 | 표시 방식 | 세부 사항 |
|------|-----------|-----------|
| 페이지 로딩 | 스켈레톤 UI | 실제 콘텐츠 형태의 회색 플레이스홀더 |
| 녹음 중 | 파형 + 시간 카운터 + "녹음 중" 텍스트 | 실시간 업데이트 |
| 파일 업로드 | 프로그레스 바 + 퍼센트 | "32% 업로드 중..." |
| STT 변환 | 프로그레스 바 + 단계 텍스트 | "음성을 텍스트로 변환 중... (3/5단계)" |
| 데이터 동기화 | 회전 아이콘 + "동기화 중..." | 완료 시 체크 아이콘 |
| 홈페이지 생성 | 프로그레스 바 + 단계별 메시지 | "디자인을 적용하고 있습니다..." |
| 폼 제출 | 버튼 내 스피너 + 비활성화 | 중복 제출 방지 |

**규칙**:
- 3초 이상 걸리는 작업은 반드시 진행률 표시
- 불확정 로딩(시간 모름)은 스피너/스켈레톤 사용
- 확정 로딩(시간 예측 가능)은 프로그레스 바 사용
- "잠시만 기다려주세요"만 표시 금지 → 구체적 상태 표시

### 4.5 실행 취소 및 확인

중요한 작업 전에는 항상 확인 단계를 거친다.

| 동작 | 확인 방식 | 실행 취소 |
|------|-----------|-----------|
| 말씀노트 삭제 | 확인 모달: "이 말씀노트를 삭제하시겠습니까?" | 30초 내 "실행 취소" 토스트 |
| 녹음 파일 삭제 | 확인 모달: "이 녹음 파일을 삭제하시겠습니까? 복구할 수 없습니다" | 불가 (명시) |
| 출석 마감 | 확인 모달: "출석을 마감하시겠습니까? 마감 후에도 수정할 수 있습니다" | 마감 해제 가능 |
| 교인 비활성화 | 확인 모달 + 사유 입력 | 관리자 재활성화 가능 |
| 피드백 전송 | 즉시 전송 (가벼운 동작) | 3초 내 "전송 취소" |
| 홈페이지 발행 | 확인 모달 | 비공개 전환 가능 |

## 5. 청각 접근성

### 5.1 오디오 콘텐츠 대안

| 오디오 기능 | 텍스트 대안 | 구현 방법 |
|------------|-----------|-----------|
| 설교 녹음 | STT 자동 변환 텍스트 | Phase 2 (Google STT + Whisper) |
| 말씀 암송 TTS | 텍스트 원문 항상 표시 | TTS 재생 시 해당 텍스트 하이라이트 |
| 녹음 재생 | 재생 중 파형 + 타임라인 시각화 | 청각 없이도 재생 상태 인지 가능 |
| 알림 사운드 | 시각 알림 (뱃지, 배너) + 진동 | 소리 없이도 알림 인지 가능 |

### 5.2 알림 다중 채널

모든 알림은 **최소 2개 이상의 채널**을 통해 전달한다.

| 알림 유형 | 시각 | 소리 | 진동 | 비고 |
|-----------|------|------|------|------|
| 푸시 알림 | O (배너) | O (선택) | O (선택) | 시각 필수, 소리/진동 선택 |
| 앱 내 알림 | O (뱃지 + 목록) | X | X | 항상 시각 표시 |
| 녹음 시작/중지 | O (UI 변경) | X | O (진동) | 시각 + 진동 |
| 에러 | O (텍스트 + 아이콘) | X | O (진동) | 시각 + 진동 |
| 출석 체크 완료 | O (체크 아이콘) | X | O (진동) | 시각 + 진동 |
| 토스트 메시지 | O (텍스트 배너) | X | X | 시각만 (짧은 정보) |

### 5.3 TTS 음성 듣기 대응

말씀 암송 등 TTS 기능 사용 시:
- 재생 중인 텍스트를 **실시간 하이라이트** (카라오케 스타일)
- 자막 바 항상 표시 (토글 불가 - 기본 표시)
- 재생 속도 조절: 0.5x / 0.75x / 1.0x / 1.25x / 1.5x
- 일시정지/재개 버튼 명확히 표시

## 6. 스크린 리더 대응

### 6.1 ARIA 레이블 가이드

주요 컴포넌트별 ARIA 레이블 및 역할 설정:

#### 네비게이션

```html
<!-- 하단 네비게이션 -->
<nav aria-label="메인 네비게이션">
  <a role="tab" aria-selected="true" aria-label="홈, 현재 선택됨">
    <HomeIcon aria-hidden="true" />
    <span>홈</span>
  </a>
  <a role="tab" aria-selected="false" aria-label="말씀노트">
    <BookIcon aria-hidden="true" />
    <span>말씀</span>
  </a>
  <!-- ... -->
</nav>
```

#### 폼 필드

```html
<!-- 입력 필드 -->
<label for="name" id="name-label">이름 (필수)</label>
<input
  id="name"
  type="text"
  aria-labelledby="name-label"
  aria-required="true"
  aria-invalid="false"
  aria-describedby="name-error"
  placeholder="이름을 입력하세요"
/>
<span id="name-error" role="alert" aria-live="polite">
  <!-- 에러 메시지가 여기에 표시됨 -->
</span>
```

#### 녹음 버튼

```html
<!-- 녹음 버튼 (대기 상태) -->
<button
  aria-label="설교 녹음 시작"
  aria-pressed="false"
  role="switch"
>
  <MicIcon aria-hidden="true" />
  <span>녹음</span>
</button>

<!-- 녹음 버튼 (녹음 중) -->
<button
  aria-label="녹음 중지, 현재 32분 15초 녹음됨"
  aria-pressed="true"
  role="switch"
>
  <StopIcon aria-hidden="true" />
  <span>녹음 중 32:15</span>
</button>
```

#### 출석 체크 목록

```html
<ul role="list" aria-label="출석 체크 목록, 총 150명">
  <li role="listitem">
    <input
      type="checkbox"
      id="member-1"
      aria-label="홍길동 집사 출석 체크"
      aria-checked="false"
    />
    <label for="member-1">
      <img src="..." alt="홍길동 집사 프로필 사진" />
      <span>홍길동</span>
      <span>집사</span>
    </label>
  </li>
</ul>
```

#### 바텀시트

```html
<div
  role="dialog"
  aria-modal="true"
  aria-label="말씀노트 공유하기"
  aria-describedby="share-desc"
>
  <p id="share-desc">공유 방법을 선택하세요</p>
  <button aria-label="카카오톡으로 공유">카카오톡</button>
  <button aria-label="PDF로 내보내기">PDF</button>
  <button aria-label="이미지로 공유">이미지</button>
  <button aria-label="링크 복사">링크 복사</button>
  <button aria-label="공유 패널 닫기">닫기</button>
</div>
```

### 6.2 읽기 순서 최적화

스크린 리더가 논리적 순서로 콘텐츠를 읽도록 DOM 순서를 최적화한다.

| 화면 | 읽기 순서 |
|------|-----------|
| 홈 대시보드 | 인사말 → 오늘 출석 카드 → 공지 카드 → 최근 활동 → 하단 네비 |
| 말씀노트 작성 | 뒤로가기 → 제목 → 녹음 버튼 → 녹음 상태 → 에디터 → 저장 |
| 출석 체크 | 뒤로가기 → 제목 → 출석 카운터 → 모드 탭 → 교인 목록 → 마감 버튼 |
| 교인 상세 | 뒤로가기 → 사진 → 이름 → 직분 → 연락처 → 출석 이력 → 편집 |

**규칙**:
- CSS `order` 속성으로 시각적 순서를 변경한 경우, `tabindex`로 읽기 순서 보정
- 장식적 이미지: `aria-hidden="true"` 또는 `alt=""`
- 의미 있는 이미지: 구체적인 `alt` 텍스트 제공 (예: `alt="김성훈 목사 프로필 사진"`)

### 6.3 포커스 관리

모달, 바텀시트, 토스트 등 동적 UI 요소의 포커스를 적절히 관리한다.

| UI 요소 | 열릴 때 | 닫힐 때 |
|---------|---------|---------|
| 모달 | 모달 내 첫 번째 포커스 가능 요소로 포커스 이동 | 모달을 연 요소로 포커스 복귀 |
| 바텀시트 | 시트 내 첫 번째 버튼으로 포커스 이동 | 시트를 연 요소로 포커스 복귀 |
| 토스트 | 포커스 이동 없음 (aria-live로 알림) | - |
| 드롭다운 | 첫 번째 옵션으로 포커스 이동 | 드롭다운 버튼으로 포커스 복귀 |
| 검색 결과 | 첫 번째 검색 결과로 포커스 이동 | 검색 입력 필드로 포커스 복귀 |

**포커스 트랩 (Focus Trap)**:
- 모달이 열려 있을 때 Tab 키로 모달 밖으로 포커스 이동 불가
- 모달 내에서 Tab: 첫 요소 → 마지막 요소 → 다시 첫 요소 (순환)
- Esc 키: 모달/바텀시트 닫기

### 6.4 동적 콘텐츠 알림 (aria-live)

| 콘텐츠 유형 | aria-live 값 | 설명 |
|------------|-------------|------|
| 토스트 메시지 | `polite` | 현재 읽기 완료 후 알림 |
| 에러 메시지 | `assertive` | 즉시 알림 (인라인 에러) |
| 출석 카운터 업데이트 | `polite` | 카운터 변경 시 알림 |
| 녹음 시간 | `off` | 1초마다 읽으면 방해 → 수동 확인만 |
| 검색 결과 수 | `polite` | "5개의 검색 결과가 있습니다" |
| 로딩 상태 | `polite` | "데이터를 불러오고 있습니다" |
| 새 알림 도착 | `polite` | "새 알림이 1건 있습니다" |
| 업로드 완료 | `polite` | "녹음 파일 업로드가 완료되었습니다" |

```html
<!-- 출석 카운터 예시 -->
<div aria-live="polite" aria-atomic="true">
  <span>현재 출석: </span>
  <span aria-label="42명 출석, 전체 150명">42 / 150명</span>
</div>

<!-- 토스트 메시지 예시 -->
<div role="status" aria-live="polite">
  녹음이 저장되었습니다
</div>

<!-- 에러 메시지 예시 -->
<div role="alert" aria-live="assertive">
  이름을 입력해주세요
</div>
```

| 탐색 순서 | `accessibilityTraversalBefore/After` |

#### React Native 접근성 속성

```jsx
// 녹음 버튼 예시

  accessible={true}
```

## 7. 고령 교인 특별 가이드라인 (ChurchThrive 특화)

이 섹션은 ChurchThrive의 핵심 사용자인 60대 이상 교인을 위한 특화 가이드라인이다. 페르소나 '최영자 집사(63세)'를 기준으로 작성한다.

### 7.1 큰 버튼 + 명확한 텍스트

| 요소 | 일반 기준 | 고령 교인 기준 | 적용 화면 |
|------|----------|---------------|-----------|
| Primary 버튼 | 44px 높이 | **52px 높이**, 최소 너비 200px | 모든 화면 |
| 녹음 버튼 | 48px | **64x64px**, 화면 중앙, 빨간색 | 말씀노트 |
| 체크박스 | 20px 시각 | **24px 시각, 48px 터치** | 출석 체크 |
| 텍스트 라벨 | 14px | **16px 이상** | 모든 버튼 |
| 아이콘 | 20px | **24px 이상** | 모든 화면 |

### 7.2 녹음 버튼 특별 사양

녹음 기능은 60대 교인의 핵심 사용 기능(게이트웨이 기능)이므로 특별한 접근성 사양을 적용한다.

```
녹음 버튼 사양:
├── 크기: 64x64px (화면에서 가장 큰 버튼)
├── 위치: 화면 중앙 정렬, 상단 영역
├── 색상: 빨간색 (#EF4444), 대비율 충분
├── 아이콘: 마이크 (32px, 흰색), 누구나 인지 가능
├── 레이블: "녹음" (16px, 버튼 아래, 항상 표시)
├── 주변 여백: 최소 24px (다른 요소와 충분한 거리)
├── 녹음 중 상태:
│   ├── 색상 변경: 더 진한 빨간 (#DC2626)
│   ├── pulse 애니메이션 (시각적 상태 변화)
│   ├── "녹음 중" 텍스트 표시
│   ├── 시간 카운터 표시 (큰 글씨, 18px)
│   └── 진동 피드백
├── 첫 사용 가이드:
│   ├── 오버레이: "이 버튼을 누르면 설교 녹음이 시작됩니다"
│   ├── 화살표 + 스팟라이트 애니메이션
│   └── [알겠습니다] 버튼으로 닫기
└── 접근성: aria-label="설교 녹음 시작" + accessibilityHint
```

### 7.3 단순한 플로우

각 화면에서 **1가지 주요 액션**만 명확히 제시한다.

| 화면 | 주요 액션 | 표현 방식 |
|------|-----------|-----------|
| 로그인 | 카카오 로그인 | 가장 큰 버튼, 화면 중앙 하단 |
| 말씀노트 목록 | 새 노트 작성 | FAB (+) 버튼, 우하단 |
| 새 말씀노트 | 녹음 시작 | 녹음 버튼, 화면 중앙 상단 |
| 홈 대시보드 | 출석 체크 | "오늘 출석" 카드, 가장 상위 |
| 교인 목록 | 교인 검색 | 검색 바, 화면 최상단 |

**원칙**:
- 화면에 동시에 보이는 동작 버튼은 최대 2-3개
- 주요 동작 버튼은 크기와 색상으로 구분 (가장 크고, Primary 색상)
- 부가 동작은 텍스트 링크 또는 작은 아이콘으로 구분

### 7.4 도움말 툴팁 (첫 사용 가이드)

첫 사용 시 핵심 기능을 안내하는 **온보딩 오버레이**를 제공한다.

```
온보딩 가이드 플로우:
├── 1단계: 홈 화면 진입 시
│   ├── 스팟라이트: "말씀" 탭 강조
│   ├── 설명: "여기서 설교 녹음과 말씀노트를 관리할 수 있습니다"
│   └── [다음] [건너뛰기]
├── 2단계: 말씀노트 화면
│   ├── 스팟라이트: FAB (+) 버튼 강조
│   ├── 설명: "이 버튼을 눌러 새 말씀노트를 만드세요"
│   └── [다음] [건너뛰기]
├── 3단계: 녹음 버튼
│   ├── 스팟라이트: 녹음 버튼 강조
│   ├── 설명: "이 버튼을 누르면 설교 녹음이 시작됩니다"
│   └── [완료] [건너뛰기]
└── 재표시: 설정 > 도움말 > "사용 가이드 다시 보기"
```

**규칙**:
- 첫 사용 시 자동 표시, 이후에는 표시하지 않음
- [건너뛰기] 옵션 항상 제공
- 설정에서 언제든 다시 볼 수 있음
- 총 3-5단계 이내 (너무 많으면 무시됨)
- 각 단계: 반투명 배경 + 밝은 스팟라이트 + 설명 말풍선

### 7.5 전화번호 OTP 로그인

60대 교인은 이메일 계정 관리가 어려울 수 있으므로 **전화번호 기반 로그인**을 최우선으로 제공한다.

```
전화번호 OTP 로그인 플로우:
├── 1. 전화번호 입력
│   ├── 큰 입력 필드 (높이 56px, 폰트 18px)
│   ├── 숫자 키패드 자동 표시
│   ├── 자동 하이픈 포맷 (010-XXXX-XXXX)
│   └── placeholder: "전화번호를 입력하세요"
├── 2. 인증번호 발송
│   ├── [인증번호 받기] 큰 버튼 (52px 높이)
│   ├── SMS 발송 후: "인증번호가 전송되었습니다" 토스트
│   └── 재발송: 60초 카운트다운 후 [다시 보내기]
├── 3. 인증번호 입력
│   ├── 6자리 분리 입력 필드 (각 48x56px)
│   ├── 숫자 키패드 자동 표시
│   ├── 자동 포커스 이동 (1칸 입력 → 다음 칸)
│   ├── SMS 자동 완성 지원 (Android)
│   └── 유효시간: 3분 (카운트다운 표시)
└── 4. 로그인 완료
    ├── 자동 인증 완료
    └── 홈 화면 진입
```

### 7.6 글자 크기 조절 버튼

시스템 설정과 별도로 앱 자체에서 글자 크기를 조절할 수 있는 기능을 제공한다.

```

├── 위치: 설정 > 화면 설정 > 글자 크기
├── 또는: 모든 텍스트 화면 우상단 [Aa] 버튼 (빠른 접근)
├── 조절 방식:
│   ├── [A-] 작게 (90%)
│   ├── [가] 기본 (100%) ← 기본 선택
│   ├── [A+] 크게 (120%)
│   ├── [A++] 매우 크게 (140%)
│   └── 슬라이더: 80% ~ 160% (세밀 조절)
├── 미리보기:
│   └── 조절 화면에서 실시간 미리보기 텍스트 표시
│       "이 글자 크기가 편하신가요?"
├── 적용 범위:
│   ├── 본문 텍스트
│   ├── 버튼 라벨
│   ├── 메뉴 텍스트
│   └── 제외: 시스템 UI, 하단 네비게이션 (레이아웃 유지)
└── 저장: 사용자별 설정 저장 (로컬 + 서버 동기화)
```

## 9. 접근성 구현 우선순위

### P0 - MVP 필수 (Phase 1)

MVP 출시 전 반드시 충족해야 하는 접근성 요건:

| # | 항목 | 적용 범위 | 검증 방법 |
|---|------|-----------|-----------|
| 1 | 색상 대비율 4.5:1 이상 | 모든 텍스트/배경 조합 | axe-core 자동 검사 |
| 2 | 최소 폰트 크기 16px (Body) | 모든 화면 | eslint 규칙 + 수동 검사 |

| 4 | 녹음 버튼 64x64px | 말씀노트 화면 | 수동 측정 |
| 5 | 키보드 네비게이션 (웹) | 모든 화면 (PWA) | Tab 키 테스트 |
| 6 | 기본 ARIA 라벨 | 모든 버튼, 폼 필드, 이미지 | axe-core 자동 검사 |
