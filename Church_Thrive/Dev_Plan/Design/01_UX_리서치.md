## 1. UX 리서치 개요

### 1.1 리서치 목표

ChurchThrive App은 한국 개신교 교회(50~300명 중형교회 중심)를 위한 교회 총괄 관리 SaaS이다. UX 리서치의 목표는 다음과 같다.

| # | 목표 | 측정 지표 |
|---|------|-----------|
| 1 | 디지털 리터러시가 낮은 60대 이상 교인도 **독립적으로** 말씀노트를 사용할 수 있도록 한다 | 무지원 태스크 완료율 90% 이상 |
| 2 | 사무간사(파워유저)의 **기존 Excel 워크플로우를 완전 대체**한다 | Excel 임포트 200명 < 5분, 주간 반복 업무 시간 50% 감소 |
| 3 | 주일예배 시 **오프라인 환경에서도 핵심 기능이 100% 동작**하도록 한다 | 네트워크 단절 상태 녹음/노트/출석 성공률 100% |
| 4 | **카카오톡과 자연스럽게 연결**되어 별도의 학습 비용 없이 공유/알림이 작동한다 | 카카오 공유 태스크 완료 시간 < 10초 |
| 5 | 5개 페르소나가 **역할에 맞는 최적화된 경험**을 받도록 한다 | 페르소나별 NPS 40+ |

### 1.2 리서치 방법론

#### Phase A: 발견 단계 (Discovery)

| 방법 | 대상 | 목적 | 산출물 |
|------|------|------|--------|
| **맥락적 탐구 (Contextual Inquiry)** | 파일럿 교회 3~5곳 | 예배 현장에서 실제 행동 관찰 | 행동 패턴 맵, 페인 포인트 목록 |
| **심층 인터뷰** | 페르소나별 3~5명 (총 15~25명) | 니즈, 동기, 장벽 파악 | 어피니티 다이어그램, 인사이트 카드 |
| **설문조사** | ChurchThrive 부트캠프 졸업생 127명 | 정량 데이터 수집 | 통계 분석 리포트 |
| **경쟁 분석** | 교회사랑넷, 플록, 처치플러스 등 | 시장 기회 및 차별점 도출 | 경쟁 벤치마크 매트릭스 |

#### Phase B: 정의 단계 (Define)

| 방법 | 목적 | 산출물 |
|------|------|--------|
| **페르소나 워크숍** | 5개 페르소나 검증 및 정제 | 검증된 페르소나 카드 |
| **사용자 여정 맵핑** | 터치포인트별 감정/페인 포인트 시각화 | 여정 맵 5종 |
| **Jobs-to-be-Done 분석** | 핵심 과업 우선순위화 | JTBD 프레임워크 |
| **카드 소팅** | 정보 구조 검증 | IA(정보 구조) 설계안 |

#### Phase C: 검증 단계 (Validate)

| 방법 | 대상 | 목적 | 산출물 |
|------|------|------|--------|
| **종이 프로토타입 테스트** | 페르소나별 2~3명 | 초기 콘셉트 검증 | 개선 요구사항 |
| **유저빌리티 테스트 (비대면)** | 총 15~20명 | 태스크 완료율/시간 측정 | 유저빌리티 리포트 |
| **A/B 테스트** | 베타 사용자 전체 | 핵심 화면 변형 비교 | 전환율 데이터 |
| **일기 연구 (Diary Study)** | 파일럿 교회 교인 10~15명 (2주간) | 실제 사용 맥락에서 장기 경험 추적 | 사용 패턴, 이탈 요인 분석 |

### 1.3 핵심 리서치 질문

```
RQ1. 60대 이상 교인이 말씀노트 STT를 독립적으로 사용하기 위한
     최소한의 인터페이스 요소는 무엇인가?

RQ2. 사무간사의 Excel 기반 워크플로우를 ChurchThrive로 전환할 때
     가장 큰 저항 요인은 무엇이며, 이를 어떻게 해소할 수 있는가?

RQ3. 주일예배 시 오프라인 환경에서 사용자가 기능 동작을 신뢰할 수 있도록
     어떤 시각적/청각적 피드백이 필요한가?

RQ4. 카카오톡에 익숙한 사용자에게 자연스러운 공유 경험을 제공하기 위한
     인터랙션 패턴은 무엇인가?

RQ5. 5개 페르소나 각각에게 최적화된 홈 화면과 네비게이션 구조는 무엇인가?
```

### 1.4 리서치 윤리

- 모든 참여자에게 연구 목적, 절차, 데이터 활용 범위를 사전 고지한다
- 개인정보보호법(PIPA) 준수하여 연구 데이터를 관리한다
- 참여자의 동의 없이 녹음/촬영하지 않는다
- 교회 내부 데이터(교인 정보, 재정 등)는 연구에 활용하지 않는다
- 연구 참여에 대한 적절한 보상(기프티콘 등)을 제공한다

## 4. UX 원칙 (Design Principles)

### 4.1 오프라인 우선 (Offline-First)

> "네트워크가 없어도 핵심 기능은 반드시 동작한다"

**원칙 선언**: 주일예배 시 교회 내부 Wi-Fi 불안정과 동시 접속 부하를 고려하여, 핵심 기능은 오프라인 상태에서 100% 동작해야 한다. 온라인은 동기화를 위한 "보너스"이다.

**적용 규칙**:

| 우선순위 | 기능 | 오프라인 동작 | 동기화 전략 |
|---------|------|-------------|-------------|
| P0 (필수) | 설교 녹음 | IndexedDB/AsyncStorage 저장 | Background Sync로 자동 업로드 |
| P0 (필수) | 말씀노트 작성 | 로컬 저장 + 자동 백업 | Last-Write-Wins + 충돌 UI 프롬프트 |
| P0 (필수) | 출석 체크 | 로컬 큐 저장 | 온라인 복원 시 배치 동기화 |
| P1 (우선) | 교인 검색 | 로컬 캐시 (최근 접근 데이터) | 앱 시작 시 증분 동기화 |
| P1 (우선) | 공지 확인 | Service Worker 캐시 | Stale-While-Revalidate |
| P1 (우선) | 성경 읽기 | 사전 다운로드 | 전체 성경 데이터 로컬 저장 |

**시각적 피드백 규칙**:

```
┌──────────────────────────────────────────────────────┐
│  오프라인 상태 표시 패턴                               │
│                                                      │
│  [온라인]    상단 바: 정상 (표시 없음)                 │
```

### 4.2 3탭 규칙 (Three-Tap Rule)

> "모든 주요 기능은 3탭 이내에 도달할 수 있어야 한다"

**원칙 선언**: 디지털 리터러시가 낮은 사용자도 직관적으로 원하는 기능에 도달할 수 있도록, 핵심 기능의 접근 경로를 3탭 이내로 제한한다.

**기능별 탭 수 설계**:

```
기능                           탭 수   경로
───────────────────────────────────────────────────────
말씀노트 녹음 시작              1탭    플로팅 버튼 터치
말씀노트 목록 보기              1탭    하단 네비 "말씀노트" 탭
대시보드 확인                   1탭    하단 네비 "홈" 탭
출석 체크 시작                  2탭    홈 → 출석체크 위젯
교인 검색                      2탭    홈 → 검색 아이콘
공지 확인                      2탭    홈 → 공지 위젯
설교 아카이브                   2탭    말씀노트 → 아카이브 탭
구역모임 출석                   2탭    홈 → 구역모임 위젯
카카오톡 공유                   3탭    말씀노트 → 노트 선택 → 공유
심방 기록 작성                  3탭    홈 → 교인 선택 → 심방 기록
훈련 프로젝트 접근              2탭    하단 네비 "사역" → 훈련
리포트 생성                    3탭    관리 → 리포트 → 유형 선택
```

**네비게이션 구조** (모바일):

```
┌──────────────────────────────────────────────────┐
│                                                  │
│              [앱 콘텐츠 영역]                     │
│                                                  │
│                          ┌──────┐                │
│                          │  ●   │ 플로팅 녹음 버튼│
│                          │ 녹음  │ (항상 표시)     │
│                          └──────┘                │
│                                                  │
├──────────────────────────────────────────────────┤
│  ┌────┐   ┌────┐   ┌────┐   ┌────┐   ┌────┐    │
│  │ 홈  │   │말씀 │   │사역 │   │교회 │   │더보기│    │
│  │     │   │노트 │   │    │   │    │   │    │    │
│  └────┘   └────┘   └────┘   └────┘   └────┘    │
```

### 4.3 고령자 친화 (Senior-Friendly)

> "어르신이 손주에게 도움을 요청하지 않아도 사용할 수 있어야 한다"

**원칙 선언**: 한국 교회의 60대 이상 교인 비율이 높은 현실을 반영하여, 고령자가 독립적으로 사용할 수 있는 접근성 수준을 보장한다.

**타이포그래피 기준**:

```
폰트 패밀리: Pretendard (한글 가독성 최적화)

크기 계층:
  제목 (H1):     28px / Bold     → 화면 제목, 주요 수치
  부제목 (H2):   22px / SemiBold → 섹션 제목
  소제목 (H3):   18px / Medium   → 카드 제목, 리스트 항목
  본문:          16px / Regular  → 일반 텍스트 (최소 기준)
  보조 텍스트:    14px / Regular  → 타임스탬프, 부가 정보
  캡션:          12px / Regular  → 법적 고지 등 (최소 사용)

줄간격: 1.6 ~ 1.8 (한글 가독성 최적화)
자간: -0.01em (Pretendard 기본)
```

**터치 타겟 기준**:

```
최소 터치 영역:    44px x 44px  (WCAG 2.1 AA)
권장 터치 영역:    48px x 48px  (고령자 대응)
핵심 버튼 (녹음):  56px x 56px  (실수 방지)
터치 간 간격:      최소 8px     (오터치 방지)
```

**색상 및 대비**:

```
배경:             #FFFFFF (White)
주요 텍스트:       #1A1A1A (거의 Black, 대비 15.3:1)
보조 텍스트:       #6B7280 (Gray, 대비 4.6:1)
주요 액션:        #228B22 (Forest Green, 대비 3.1:1 → 텍스트에는 미사용, 버튼에만)
보조 액션:        #1A365D (Deep Blue, 대비 10.7:1)
오류:             #DC2626 (Red, 대비 4.6:1)
성공:             #16A34A (Green, 대비 3.2:1 → 아이콘+텍스트 병행)
비활성:           #D1D5DB (Light Gray)

※ Forest Green은 충분한 대비를 위해 White 배경의 버튼 fill로 사용하고,
   버튼 내부 텍스트는 #FFFFFF (White)로 표시한다.
```

**고령자 전용 UX 패턴**:

| # | 패턴 | 설명 |
|---|------|------|
| 1 | 확인 다이얼로그 | 삭제/전송 등 되돌릴 수 없는 액션 전 반드시 "정말 삭제하시겠습니까?" 확인 |
| 2 | 존댓말 오류 메시지 | "네트워크에 연결할 수 없습니다. Wi-Fi를 확인해 주세요." |
| 3 | 되돌리기 지원 | 모든 편집 작업에 "되돌리기" 버튼 제공 |
| 4 | 아이콘+텍스트 병행 | 아이콘만 사용하지 않고 반드시 텍스트 레이블 함께 표시 |
| 5 | 진행 단계 표시 | 다단계 작업 시 "1/3단계" 등 현재 위치 명확히 표시 |
| 6 | 자동 저장 | 입력 중 자동 저장 + "저장되었습니다" 피드백 |

### 4.5 역할별 적응형 UI (Role-Adaptive UI)

> "각 역할이 필요한 것만 보여준다"

**원칙 선언**: 5개 페르소나의 디지털 리터러시와 사용 목적이 다르므로, 역할에 따라 UI 복잡도와 기능 노출을 조절한다.

**역할별 UI 복잡도 매트릭스**:

```
                    담임목사   사무간사   사역리더   일반교인   청년리더
                    ────────  ────────  ────────  ────────  ────────
 UI 복잡도           중        높        낮        최소      중~높
 주 사용 기기        모바일+PC  PC 중심   모바일만   모바일만   모바일+PC
 네비게이션 항목     4~5개     6~7개     3~4개     3개       5~6개
 대시보드 위젯       4개       5개       3개       2개       4개
 설정 메뉴 노출      전체      전체      제한적     최소      전체
 데이터 테이블       표시      확장 테이블 미표시     미표시    표시
 단축키 지원         부분      전체      미지원     미지원    부분
 고급 필터           표시      표시      미표시     미표시    표시
```

**적응형 UI 구현 전략**:

```
1. 역할 감지
   └── 사용자 프로필의 role 필드 기반 (ChurchAdmin, Staff, Leader, Member, YouthLeader)

2. 레이아웃 분기
   ├── role === 'Staff' → 사이드바 + 넓은 테이블 + 멀티패널 (PC 최적화)
   ├── role === 'Member' → 간결한 카드 UI + 큰 버튼 (모바일 최적화)
   └── 그 외 → 역할별 위젯 구성 자동 적용

3. 기능 노출 제어
   ├── RBAC 기반 메뉴 필터링
   ├── 불필요한 메뉴 항목 자동 숨김
   └── 고급 기능은 "더보기" 뒤에 배치
```

## 6. 감정 디자인 (Emotional Design)

### 6.1 디자인 모드 개요

ChurchThrive는 사용 맥락에 따라 3가지 감정 모드를 제공한다. 각 모드는 색상, 타이포그래피, 애니메이션, 사운드를 조절하여 해당 맥락에 적합한 감정적 경험을 제공한다.

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   예배 모드              관리 모드              성장 모드        │
│   (Worship Mode)        (Admin Mode)           (Growth Mode)   │
│                                                                 │
│   ┌──────────┐          ┌──────────┐          ┌──────────┐     │
│   │          │          │          │          │          │     │
│   │  집중    │          │  효율    │          │  격려    │     │
│   │  차분    │          │  명확    │          │  진행률  │     │
│   │  경건    │          │  생산성  │          │  성취감  │     │
│   │          │          │          │          │          │     │
│   └──────────┘          └──────────┘          └──────────┘     │
│                                                                 │
│   적용 시점:             적용 시점:             적용 시점:       │
│   - 말씀노트 녹음 중     - 교인관리             - 암송 연습      │
│   - 성경 읽기/통독       - 리포트/결재          - 통독 진행      │
│   - 설교 청취            - 출석/행정            - 훈련 참여      │
│   - 기도제목 작성        - Excel 임포트         - 필사           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 6.2 예배 모드 (Worship Mode) - 집중, 차분

> "예배에 방해가 되지 않는, 차분하고 경건한 인터페이스"

**적용 시점**: 말씀노트 녹음 중, 성경 읽기/통독, 설교 청취, 기도제목 작성

**시각적 특성**:

| 요소 | 사양 | 의도 |
|------|------|------|
| 배경색 | #FAFAF5 (Warm Off-White) | 종이와 유사한 따뜻한 톤으로 눈의 피로 감소 |
| 텍스트 색상 | #2D2D2D (Soft Black) | 순수 블랙보다 부드러운 인상 |
| 강조색 | #1A365D (Deep Blue) | Forest Green 대신 절제된 톤 사용 |
| 구분선 | #E8E4DF (Warm Gray) | 최소한의 시각적 구분 |
| 성경 구절 하이라이트 | #FFF3CD (Soft Gold) | 말씀 구절을 은은하게 강조 |

**타이포그래피**:

```
성경 본문:      Pretendard, 18px, Regular, 줄간격 2.0
설교 노트:      Pretendard, 16px, Regular, 줄간격 1.8
메모:           Pretendard, 16px, Regular, 줄간격 1.6
타임스탬프:     Pretendard, 12px, Light, #9CA3AF
```

**인터랙션 특성**:

| 특성 | 사양 | 의도 |
|------|------|------|
| 애니메이션 속도 | 느림 (400~600ms) | 급하지 않은, 차분한 전환 |
| 전환 효과 | fade-in/out | 갑작스러운 변화 방지 |
| 알림 | 무음 진동만 (50ms, 부드러운 단발) | 예배 방해 방지 |
| 화면 밝기 | 자동 감소 제안 (야간/실내 감지) | 눈의 피로 경감 |
| 상태바 | 녹음 시간 + 파형만 최소 표시 | 예배 집중 유지 |
| 네비게이션 | 하단 바 자동 숨김 (녹음 중) | 화면 공간 최대화, 집중 모드 |

**마이크로인터랙션**:

```
녹음 시작:      부드러운 원형 확장 (ripple) + 약한 진동
녹음 중:        미세한 파형 애니메이션 (차분한 움직임)
녹음 중지:      원형 축소 + "저장 완료" 페이드인
페이지 전환:    크로스 페이드 (400ms)
성경 구절 터치: 부드러운 하이라이트 확장 (300ms)
```

### 6.3 관리 모드 (Admin Mode) - 효율, 명확

> "데이터를 빠르고 정확하게 처리할 수 있는, 명확하고 효율적인 인터페이스"

**적용 시점**: 교인관리, 리포트/결재, 출석/행정, Excel 임포트

**시각적 특성**:

| 요소 | 사양 | 의도 |
|------|------|------|
| 배경색 | #FFFFFF (Pure White) | 깨끗하고 전문적인 인상 |
| 텍스트 색상 | #1A1A1A (Near Black) | 최대 가독성 |
| 주요 액션 | #228B22 (Forest Green) | 브랜드 컬러로 핵심 액션 강조 |
| 보조 액션 | #1A365D (Deep Blue) | 보조 기능 구분 |
| 데이터 테이블 | 줄무늬 (#F9FAFB / #FFFFFF) | 행 구분 용이 |
| 상태 뱃지 | 녹/황/적 트래픽 라이트 | 직관적 상태 인지 |

**타이포그래피**:

```
테이블 헤더:    Pretendard, 14px, SemiBold, #374151
테이블 셀:      Pretendard, 14px, Regular, #1A1A1A
수치 데이터:    Pretendard, 14px, Medium (tabular figures)
버튼 텍스트:    Pretendard, 14px, SemiBold
라벨:           Pretendard, 14px, Medium, #6B7280
```

**인터랙션 특성**:

| 특성 | 사양 | 의도 |
|------|------|------|
| 애니메이션 속도 | 빠름 (150~250ms) | 업무 효율 극대화 |
| 전환 효과 | instant/slide | 빠른 화면 전환 |
| 피드백 | 즉각적 (진동 없음, 시각 피드백만) | 빠른 작업 흐름 유지 |
| 키보드 단축키 | 전체 지원 (사무간사 PC) | Excel 사용자 전환 비용 감소 |
| 인라인 편집 | 더블클릭으로 즉시 편집 모드 | 별도 편집 화면 이동 불필요 |
| 멀티 선택 | Ctrl+Click / Shift+Click | 대량 데이터 처리 효율 |

**키보드 단축키 (사무간사 PC 전용)**:

```
Ctrl+N          새 교인 등록
Ctrl+F          교인 검색 (초성 검색)
Ctrl+S          저장
Ctrl+E          Excel 내보내기
Ctrl+I          Excel 임포트
Ctrl+P          인쇄/PDF
Ctrl+Z          되돌리기
Tab             다음 필드 이동 (Excel 호환)
Enter           저장 후 다음 행
Escape          편집 취소
```

**마이크로인터랙션**:

```
행 호버:        배경 #F3F4F6 (즉시, 0ms 딜레이)
체크박스 토글:   체크 아이콘 스케일인 (100ms)
저장:           녹색 체크 아이콘 페이드인 (200ms)
삭제:           행 슬라이드 아웃 (200ms) + Undo 토스트
필터 적용:      결과 카운트 숫자 카운트업 애니메이션
정렬:           컬럼 화살표 회전 (150ms)
```

### 6.4 성장 모드 (Growth Mode) - 격려, 진행률

> "영적 성장의 여정을 시각적으로 확인하고, 지속적인 동기부여를 받는 인터페이스"

**적용 시점**: 암송 연습, 통독 진행, 훈련 참여, 필사

**시각적 특성**:

| 요소 | 사양 | 의도 |
|------|------|------|
| 배경색 | #F0FDF4 (Mint Tint) | 성장/자연의 느낌 |
| 주요 색상 | #228B22 (Forest Green) | 성장과 생명의 상징 |
| 보조 색상 | #F59E0B (Amber) | 성취/보상의 느낌 |
| 진행률 바 | Forest Green 그라데이션 | 진행 상황 시각화 |
| 완료 뱃지 | Gold (#F59E0B) | 달성 축하 |

**진행률 시각화 패턴**:

```
[통독 진행률]
┌──────────────────────────────────────────┐
│  66권 통독 현황                           │
│                                          │
│  ████████████████░░░░░░░░░░  42/66권     │
│                               63.6%      │
│                                          │
│  구약 ██████████████░░░░  32/39권        │
│  신약 ██████████░░░░░░░░  10/27권        │
│                                          │
│  🔥 연속 읽기: 15일 (최고 기록!)          │
│                                          │
└──────────────────────────────────────────┘

[암송 진행률]
┌──────────────────────────────────────────┐
│  이번 달 암송                             │
│                                          │
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐    │
│  │ ✅ │ │ ✅ │ │ ✅ │ │ 🔄 │ │ ○  │    │
│  │요3:16│ │시23:1│ │빌4:13│ │롬8:28│ │사40:31│   │
│  └────┘ └────┘ └────┘ └────┘ └────┘    │
│                                          │
│  3/5구절 완료   [다음 암송 시작하기]       │
│                                          │
└──────────────────────────────────────────┘
```

**격려 메시지 시스템**:

```
[일일 격려]
  통독 연속 7일:    "한 주를 말씀과 함께 보내셨습니다!"
  통독 연속 30일:   "한 달간 꾸준히 말씀을 읽으셨습니다. 정말 대단합니다!"
  암송 1구절 완료:  "말씀을 마음에 새기셨습니다"
  훈련 출석 100%:   "모든 모임에 참석하셨습니다. 충성된 자세가 아름답습니다"

[마일스톤]
  통독 25% 완료:    "성경의 4분의 1을 읽으셨습니다!"
  통독 50% 완료:    "절반을 넘었습니다! 끝까지 함께 달려봐요"
  통독 100% 완료:   "성경 66권 통독을 완료하셨습니다! 축하합니다!"
  암송 10구절:      "10개의 말씀을 마음에 품으셨습니다"
  암송 100구절:     "100구절 암송 달성! 놀라운 헌신입니다"

[격려 표시 방식]
  - 모달 팝업: 마일스톤 달성 시 (전체 화면 축하 애니메이션)
  - 토스트: 일일 격려 (3초 표시)
  - 뱃지: 프로필에 영구 표시 (달성 기록)
  - 공유: "카카오톡으로 나누기" 옵션 제공
```

**인터랙션 특성**:

| 특성 | 사양 | 의도 |
|------|------|------|
| 애니메이션 | 생동감 있음 (300~500ms, spring easing) | 활력과 성장의 느낌 |
| 진행률 바 | 애니메이션으로 채워짐 (500ms) | 성장 과정을 시각적으로 느끼게 |
| 완료 표시 | 체크 + 짧은 반짝 파티클 | 성취감과 기쁨 |
| 연속 기록 | 불꽃 아이콘 + 연속일수 카운터 | 지속 동기 부여 |
| 비교 차트 | 주간/월간 트렌드 그래프 | 성장 패턴 인식 |
| 공유 CTA | "이 기쁨을 나눠보세요" 버튼 | 자연스러운 바이럴 유도 |

**마이크로인터랙션**:

```
암송 단계 완료:   체크 마크 + 짧은 컨페티 (500ms)
통독 한 권 완료:  책 아이콘 → 금색 전환 + 바운스 (300ms)
연속 기록 갱신:   불꽃 아이콘 확대 + 카운터 증가 (숫자 롤링)
훈련 과제 제출:   종이비행기 날아가기 애니메이션 (400ms)
필사 완료:       잉크 번짐 효과 → 깨끗한 텍스트 전환 (600ms)
마일스톤 달성:    전체 화면 골드 파티클 + 뱃지 등장 (1.5초)
```
