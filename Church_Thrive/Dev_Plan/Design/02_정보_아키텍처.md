## 1. 정보 아키텍처 개요

### 1.1 IA 설계 원칙

| 원칙 | 설명 | 적용 방법 |
|------|------|-----------|
| **3탭 원칙** | 모든 핵심 기능에 3탭(터치) 이내 도달 | 홈 대시보드에서 바로가기 + 바텀 탭 직접 접근 |
| **역할 기반 노출** | 사용자 역할에 따라 필요한 정보만 노출 | RBAC 기반 네비게이션 필터링 |
| **점진적 공개** | 복잡한 기능은 단계적으로 노출 | 기본 뷰 → 상세 뷰 → 고급 설정 순서 |
| **일관성** | 동일한 패턴으로 정보 구조 반복 | 목록 → 상세 → 편집 패턴 통일 |
| **접근성 우선** | 60대 이상 사용자도 쉽게 탐색 | 큰 터치 영역 (44px+), 명확한 라벨, 최소 16px 폰트 |
| **오프라인 대응** | 네트워크 없이도 핵심 기능 탐색 가능 | Service Worker 기반 캐싱, 오프라인 인디케이터 |
| **교회 맥락 우선** | 교회 용어와 워크플로우에 맞는 구조 | 교회 직분 체계, 예배 중심 시간 흐름 반영 |
| **멀티테넌시 격리** | 모든 데이터 구조에 `church_id` 기반 격리 적용 | URL, API, DB 전 계층에서 교회 데이터 완전 분리 |

### 1.2 IA 설계 목표

```
┌──────────────────────────────────────────────────────────────┐
│                    IA 설계 목표                                │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  1. 발견성 (Findability)                                     │
│     - 사용자가 원하는 기능을 빠르게 찾을 수 있는 구조         │
│     - 초성 검색, 글로벌 검색으로 콘텐츠 접근성 극대화         │
│                                                              │
│  2. 이해성 (Understandability)                               │
│     - 교회 용어 기반 라벨링 (예: "교적" > "프로필")           │
│     - 자연스러운 정보 그룹핑                                  │
│                                                              │
│  3. 확장성 (Scalability)                                     │
│     - Phase 2~4 기능 추가 시 구조 변경 최소화                │
│     - "더보기" 메뉴를 통한 확장 기능 수용                    │
│                                                              │
│  4. 멀티 디바이스 대응                                       │
│     - 모바일(PWA/RN): 바텀 탭 네비게이션                     │
│     - 데스크톱/태블릿: 사이드 네비게이션                      │
│     - 동일한 정보 구조, 다른 네비게이션 패턴                  │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

### 1.3 계층 구조 용어 정의

```
L0 (Root)     : 앱 진입점 - 인증/온보딩/대시보드
L1 (Section)  : 하단탭/사이드바의 주요 섹션 (홈, 교인, 말씀, 행정, 더보기)
L2 (Page)     : 섹션 내 개별 페이지 (교인목록, 말씀노트 목록, 공지사항 등)
L3 (Detail)   : 상세/편집 화면 (교인 상세, 노트 상세, 공지 상세 등)
L4 (Sub)      : 모달/시트/팝업 (필터, 공유, 설정 등)
```

### 1.4 사용자 역할 정의

| 역할 코드 | 한국어명 | 대표 페르소나 | 주요 사용 맥락 | 주요 기기 |
|-----------|---------|-------------|---------------|----------|
| `admin` | 담임목사 | 김성훈 목사 (55세) | 대시보드 확인, 전체 관리 | 갤럭시 S + PC |
| `pastor` | 교역자 | - | 교인 관리, 피드백, 심방 | 모바일 + PC |
| `staff` | 사무간사 | 이수진 간사 (32세) | 데이터 입력, 행정, 리포트 | PC 위주 |
| `leader` | 사역리더 | 박은혜 권사 (45세) | 구역 출석, 활동 보고 | 모바일만 |
| `member` | 일반교인 | 최영자 집사 (63세) | 말씀노트, 공지 확인 | 갤럭시 A |

### 1.5 플랫폼별 IA 전략

| 플랫폼 | 네비게이션 패턴 | 화면 깊이 | 주 사용자 |
|--------|----------------|-----------|-----------|
| **모바일 (PWA/RN)** | Bottom Tab (5개) + Stack Navigation | L0 → L1 → L2 → L3 | 전체 (특히 member, leader) |
| **데스크톱 (PWA)** | Left Sidebar + Top Header + Breadcrumb | L0 → L1 → L2 → L3 | staff, pastor, admin |
| **태블릿** | Collapsible Sidebar + Main Content | L0 → L1 → L2 → L3 | pastor, admin |

## 2. 글로벌 네비게이션 구조

### 2.1 모바일 바텀 탭 네비게이션 (5탭)

```
┌─────────────────────────────────────────────────────────────┐
│                        앱 화면                               │
│                                                             │
│                     (콘텐츠 영역)                            │
│                                                             │
├─────────┬─────────┬──────────┬─────────┬────────────────────┤
│   홈    │  교인   │ 말씀노트  │  행정   │   더보기            │
│  🏠     │  👥     │   📖     │  📋     │    ⋯               │
│ Home    │ Members │  Note    │ Admin   │   More              │
└─────────┴─────────┴──────────┴─────────┴────────────────────┘
```

| 탭 순서 | 탭 이름 | 아이콘 | 설명 | 우선순위 |
|---------|--------|--------|------|---------|
| 1 | 홈 | `Home` (집 모양) | 역할별 대시보드, 오늘의 요약 | 최고 |
| 2 | 교인 | `Users` (사람들) | 교인 목록, 검색, 가족 관리 | 높음 |
| 3 | 말씀노트 | `BookOpen` (책) | 말씀노트 목록, 녹음, 작성 | 최고 (게이트웨이) |
| 4 | 행정 | `ClipboardList` (문서) | 공지, 조직도, 출석, 구역 | 높음 |
| 5 | 더보기 | `MoreHorizontal` (점 3개) | 설정, 프로필, 확장 기능 | 보통 |

> **설계 근거**: 말씀노트를 중앙(3번째 탭)에 배치하여 시각적 강조. 모든 교인이 매주 사용하는 게이트웨이 기능이므로 가장 접근하기 쉬운 위치에 배치한다.

### 8.2 초성 검색 구현 전략

```

│                                                              │
│  동작 방식:                                                  │
│  ┌──────────────────────────────────────────────────┐       │

│  │  입력: 홍길 -> 결과: 홍길동, 홍길순                │       │
```

## 부록 B: 접근성 고려사항

### IA 관점 접근성 원칙

| 원칙 | 구현 방법 |
|------|----------|
| 단순한 깊이 | 최대 3~4단계 깊이로 정보 구조 제한 |
| 명확한 라벨 | 교회 용어 기반, 아이콘 + 텍스트 라벨 동시 사용 |
| 예측 가능한 위치 | 동일 패턴 반복 (목록 -> 상세 -> 편집) |
| 뒤로가기 지원 | 모든 하위 화면에서 상위 화면으로 복귀 경로 명확 |
| 현재 위치 표시 | 브레드크럼 (데스크톱), 헤더 타이틀 (모바일) |
| 큰 터치 영역 | 바텀 탭 아이콘 최소 48x48px |
| 스크린 리더 | 모든 네비게이션 항목에 aria-label 적용 |
