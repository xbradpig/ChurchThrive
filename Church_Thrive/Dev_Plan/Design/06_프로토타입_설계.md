# 06. 인터랙티브 프로토타입 설계

---

## 1. 프로토타입 개요

### 1.1 목적

ChurchThrive App의 **핵심 사용자 여정(Critical User Journey)**에 대한 인터랙션을 실제 사용자 환경에서 검증한다. 프로토타입을 통해 다음을 달성한다.

- 화면 전환 흐름의 자연스러움 검증
- 고령 교인(60대+)의 주요 태스크 완수 가능성 확인
- 제스처/애니메이션/피드백의 적절성 평가
- 개발 착수 전 UX 이슈 사전 발견 및 수정

### 1.2 프로토타입 도구

| 도구 | 용도 | 비고 |
|------|------|------|
| **Figma** | 디자인 프로토타입 | 화면 전환, 마이크로 인터랙션, 컴포넌트 라이브러리 |
| **Next.js** | 기능 프로토타입 | 실제 데이터 연동, 녹음 기능, QR 스캔 등 네이티브 기능 검증 |
| **Expo (React Native)** | 모바일 네이티브 프로토타입 | 진동 피드백, 푸시 알림, 백그라운드 녹음 검증 |

### 1.3 프로토타입 범위

MVP 핵심 플로우 **6개**를 대상으로 한다.

| # | 플로우 | 대상 페르소나 | 우선순위 |
|---|--------|--------------|----------|
| 1 | 회원가입 - 교회승인 - 첫 화면 | 전체 (최영자 집사 중심) | P0 |
| 2 | 말씀노트 녹음 - 작성 - 공유 | 일반교인 (최영자 집사) | P0 |
| 3 | QR 새가족 등록 | 사무간사 (이수진 간사) + 방문자 | P0 |
| 4 | 출석 체크 (QR + 수동) | 사역리더 (박은혜 권사) | P0 |
| 5 | 교역자 말씀노트 피드백 | 담임목사 (김성훈 목사) | P0 |
| 6 | 교회 홈페이지 자동 빌드 | 담임목사 + 사무간사 | P1 |

### 1.4 프로토타입 원칙

1. **실제 디바이스 테스트 우선**: Galaxy A 시리즈(저사양)에서 반드시 검증
2. **오프라인 시나리오 포함**: 예배당 Wi-Fi 불안정 상황 고려
3. **고령자 우선 설계**: 60대 교인이 완수하면 나머지 페르소나도 가능
4. **3탭 규칙 준수**: 핵심 기능까지 최대 3탭 이내 도달

## 2. 핵심 프로토타입 플로우 상세

### 플로우 1: 회원가입 - 교회승인 - 첫 화면

**목표**: 신규 사용자가 앱을 설치한 후 교회 승인을 받고 홈 화면에 진입하기까지의 전체 여정

**타겟 완료 시간**: 3분 이내 (승인 대기 시간 제외)

**대상 페르소나**: 최영자 집사 (63세, 디지털 리터러시 낮음)

#### Step별 상세 시나리오

```
Step 1: 앱 실행 - 스플래시 화면
├── 화면: ChurchThrive 로고 + 슬로건 "모든 교회가 건강하게 성장하도록"
├── 지속시간: 1.5초
├── 애니메이션: 로고 fade-in (0 - 500ms) + scale(0.9 - 1.0)
├── 배경: Primary Blue (#1A73E8) 단색
└── 전환: fade-out (300ms) - Step 2로 이동
```

```
Step 2: 로그인 / 회원가입 선택 화면
├── 상단: ChurchThrive 로고 (소형)
├── 중앙 일러스트: 교회 건물 + 사람들 일러스트 (따뜻한 톤)
├── 하단 버튼 영역:
│   ├── [카카오 로그인] - 노란색 (#FEE500), 카카오 아이콘 + "카카오로 시작하기"
│   │   크기: width 100%, height 52px, border-radius 12px
│   ├── [이메일 가입] - 흰색 외곽선 버튼, "이메일로 가입하기"
│   │   크기: width 100%, height 52px, border-radius 12px
│   └── [전화번호 로그인] - 텍스트 링크, "전화번호로 로그인"
├── 최하단: "이미 계정이 있으신가요? 로그인" 텍스트 링크
└── 전환: slide-left (250ms)
```

```
Step 3: 카카오 로그인 or 이메일 가입 선택
├── [카카오 로그인 선택 시]
│   ├── 카카오 OAuth 웹뷰 호출
│   ├── 카카오 동의 화면
│   ├── 동의 완료 - 자동으로 Step 4로 이동
│   └── 카카오에서 받아온 이름/프로필 자동 채움
├── [이메일 가입 선택 시]
│   ├── 이메일 입력 필드 (placeholder: "이메일 주소를 입력하세요")
│   ├── 비밀번호 입력 필드 (8자 이상, 영문+숫자)
│   │   실시간 유효성 체크 (조건 충족 시 초록색 체크 아이콘)
│   ├── 비밀번호 확인 필드
│   └── [다음] 버튼 - Step 4로 이동
├── [전화번호 로그인 선택 시]
│   ├── 전화번호 입력 (숫자 키패드 자동 표시)
│   ├── [인증번호 발송] 버튼
│   ├── 6자리 OTP 입력 (자동 포커스, 자동 완성 지원)
│   └── 인증 완료 - Step 4로 이동
└── 전환: slide-left (250ms)
```

```
Step 4: 기본정보 입력
├── 헤더: "기본 정보를 입력해주세요" (20px, Bold)
├── 폼 필드:
│   ├── 이름 (카카오 로그인 시 자동 채움, 수정 가능)
│   │   placeholder: "이름을 입력하세요"
│   ├── 전화번호 (숫자 키패드, 010-XXXX-XXXX 형식 자동 포맷)
│   │   placeholder: "010-0000-0000"
│   └── 생년월일 (선택, 하단 날짜 피커)
│       placeholder: "생년월일 (선택)"
├── 입력 필드 크기: height 52px, font-size 16px
├── [다음] 버튼: Primary Blue, 하단 고정, height 52px
│   비활성 상태: 필수 필드 미입력 시 회색(#D1D5DB)
│   활성 상태: Primary Blue(#1A73E8), 탭 가능
└── 전환: slide-left (250ms)
```

```
Step 5: 교회 검색
├── 헤더: "소속 교회를 선택해주세요" (20px, Bold)
├── 검색 입력 필드:
│   ├── 돋보기 아이콘 + placeholder "교회명을 입력하세요"
│   ├── height 52px, font-size 16px
│   ├── 2글자 이상 입력 시 자동완성 리스트 표시
│   └── 초성 검색 지원 (ㅅㅁ → 새문안교회)
├── 자동완성 리스트:
│   ├── 각 항목: 교회명 (Bold) + 주소 (Grey, 14px)
│   ├── 항목 높이: 60px (터치 영역 확보)
│   ├── 구분선: 1px solid #E5E7EB
│   └── 최대 표시: 10개 (스크롤)
├── 검색 결과 없음:
│   ├── "검색 결과가 없습니다"
│   └── [교회 신규 등록 요청] 버튼
└── 전환: slide-left (250ms)
```

```
Step 6: 교회 선택 확인
├── 선택한 교회 정보 카드:
│   ├── 교회명 (20px, Bold)
│   ├── 교단 (14px, Grey)
│   ├── 주소 (14px, Grey)
│   ├── 담임목사 (14px, Grey)
│   └── 교인 수 (14px, Grey) - 약 ○○명
├── 확인 메시지: "이 교회가 맞으신가요?"
├── [확인] 버튼: Primary Blue, 하단 고정
├── [다시 검색] 텍스트 링크
└── 전환: fade (300ms) - 확인 모달 느낌
```

```
Step 7: 승인 대기 화면
├── 중앙 애니메이션:
│   ├── 교회 아이콘 (Lottie): 부드러운 pulse 애니메이션
│   ├── "승인을 기다리고 있습니다" (18px, Medium)
│   ├── "교회 관리자가 승인하면 알려드릴게요" (14px, Grey)
│   └── 예상 시간: "보통 1-2일 이내 승인됩니다" (14px, Grey)
├── 진행 상태 표시:
│   ├── [1. 가입 신청] -------- 완료 (체크)
│   ├── [2. 관리자 확인 중] --- 진행 중 (회전 아이콘)
│   └── [3. 승인 완료] -------- 대기 (회색)
├── 하단:
│   ├── "다른 교회를 찾으시나요?" 텍스트 링크
│   └── "문의하기" 텍스트 링크
├── 푸시 알림 권한 요청: "승인 알림을 받으시겠습니까?" (OS 권한 팝업)
└── 앱 종료 후 재진입 시 이 화면 유지
```

```
Step 8: 승인 완료 - 홈 화면 진입
├── 트리거: 관리자 승인 시 즉시 또는 앱 재실행 시
├── 푸시 알림: "○○교회 가입이 승인되었습니다! 환영합니다 🎉"
├── 알림 터치 → 앱 실행 → 환영 모달:
│   ├── "환영합니다, ○○○님!"
│   ├── 교회명 + 소속 표시
│   ├── [시작하기] 버튼
│   └── 전환: confetti 애니메이션 (Lottie, 2초)
├── 홈 화면 진입:
│   ├── 첫 방문 온보딩 오버레이 (3장):
│   │   ├── 1/3: "말씀노트로 설교를 기록하세요" (녹음 아이콘 하이라이트)
│   │   ├── 2/3: "출석 체크를 간편하게" (출석 카드 하이라이트)
│   │   └── 3/3: "교회 소식을 확인하세요" (공지 카드 하이라이트)
│   └── 각 오버레이: 반투명 배경 + 밝은 스팟라이트 + [다음] / [건너뛰기]
└── 전환: slide-left (250ms) + fade overlay
```

#### 전환 애니메이션 요약

| 전환 유형 | 애니메이션 | 시간 | 이징 |
|-----------|-----------|------|------|
| 순방향 이동 | slide-left | 250ms | ease-out |
| 역방향 이동 | slide-right | 250ms | ease-out |
| 모달 표시 | fade + scale(0.95-1.0) | 250ms | ease-out |
| 스플래시 종료 | fade-out | 300ms | ease-in |
| 환영 confetti | Lottie play | 2000ms | linear |

### 플로우 2: 말씀노트 녹음 - 작성 - 공유

**목표**: 교인이 주일 예배에서 설교를 녹음하고 노트를 작성한 후 카카오톡으로 공유하기까지의 여정

**타겟 완료 시간**: 녹음 시작까지 2탭 이내, 전체 플로우 5분 이내 (녹음 시간 제외)

**대상 페르소나**: 최영자 집사 (63세), 정민준 형제 (27세)

#### Step별 상세 시나리오

```
Step 1: 홈 화면 - 하단 네비게이션 "말씀" 탭 터치
├── 하단 네비게이션 바:
│   ├── [홈] - house 아이콘 + "홈"
│   ├── [말씀] - book 아이콘 + "말씀" ← 터치 대상
│   ├── [교회] - church 아이콘 + "교회"
│   ├── [알림] - bell 아이콘 + "알림" (뱃지 표시)
│   └── [더보기] - menu 아이콘 + "더보기"
├── 네비게이션 아이콘 크기: 24x24px, 텍스트 10px
├── 네비게이션 전체 높이: 56px (Safe Area 제외)
├── 선택 시: 아이콘 + 텍스트 색상 Primary Blue, 미선택 Grey
└── 전환: fade (150ms) - 탭 간 이동은 부드러운 페이드
```

```
Step 2: 말씀노트 목록 화면
├── 상단 헤더:
│   ├── "말씀노트" (20px, Bold)
│   ├── 검색 아이콘 (우측)
│   └── 필터 아이콘 (우측, 검색 옆)
├── 노트 목록 (카드형):
│   ├── 각 카드:
│   │   ├── 날짜 (14px, Grey, "2025년 2월 2일 주일")
│   │   ├── 제목 (16px, Bold, "사랑의 빚을 지라")
│   │   ├── 설교자 (14px, Grey, "김성훈 목사")
│   │   ├── 미리보기 텍스트 (14px, 2줄 말줄임)
│   │   ├── 녹음 아이콘 (녹음본 있는 경우)
│   │   └── 태그 (선택, 작은 뱃지)
│   ├── 카드 간격: 12px
│   └── 스와이프 좌: 삭제 버튼 (빨간색)
├── 플로팅 액션 버튼 (FAB):
│   ├── 위치: 우측 하단, 하단 네비 위 16px
│   ├── 크기: 56x56px
│   ├── 색상: Primary Blue (#1A73E8)
│   ├── 아이콘: + (흰색, 24px)
│   ├── 그림자: elevation 8dp
│   └── 터치 피드백: ripple effect
├── 빈 상태 (노트가 없을 때):
│   ├── 일러스트: 노트 + 마이크 아이콘
│   ├── "아직 말씀노트가 없습니다"
│   └── "설교를 녹음하고 노트를 작성해보세요"
└── 전환: slide-left (250ms)
```

```
Step 3: 새 노트 작성 화면
├── 상단 헤더:
│   ├── [< 뒤로] 버튼 (좌측)
│   ├── "새 말씀노트" (18px, Bold, 중앙)
│   └── [저장] 버튼 (우측, 변경 사항 있을 때만 활성)
├── 녹음 섹션 (상단, 높이 120px):
│   ├── 녹음 버튼:
│   │   ├── 크기: 64x64px (60대 교인 대응, 터치 영역 충분)
│   │   ├── 색상: 빨간색 (#EF4444), 원형
│   │   ├── 아이콘: 마이크 (흰색, 32px)
│   │   ├── 레이블: "녹음" (14px, 버튼 아래)
│   │   └── 위치: 화면 중앙 정렬
│   ├── 녹음 대기 상태:
│   │   └── "버튼을 눌러 설교 녹음을 시작하세요" (14px, Grey)
│   └── 구분선: 1px solid #E5E7EB
├── 기본 입력 필드:
│   ├── 제목 (placeholder: "설교 제목", font-size 18px, Bold)
│   ├── 설교자 (placeholder: "설교자", font-size 14px)
│   ├── 성경 본문 (placeholder: "성경 본문 (예: 로마서 13:8)", font-size 14px)
│   └── 날짜: 오늘 날짜 자동 입력, 수정 가능
├── 에디터 영역:
│   ├── 서론 / 본론 / 결론 / 적용 탭 (수평 스크롤)
│   ├── 각 섹션별 placeholder 텍스트
│   ├── 리치 에디터: 볼드, 이탤릭, 밑줄, 하이라이트
│   └── font-size: 16px (최소), line-height: 1.6
└── 전환: slide-up (250ms) - 새 화면이 아래에서 올라옴
```

```
Step 4: 녹음 시작

## 3. 인터랙션 명세

### 3.1 제스처 맵

| 제스처 | 용도 | 적용 화면 | 대안 조작 |
|--------|------|-----------|-----------|
| **탭** | 기본 선택/실행 | 모든 화면 | - |
| **롱프레스** | 컨텍스트 메뉴 / 인라인 코멘트 | 말씀노트 텍스트, 교인 목록 | 메뉴 버튼 |
| **스와이프 좌** | 삭제/보관 액션 노출 | 목록 항목 (노트, 알림 등) | 롱프레스 → 메뉴 |
| **스와이프 좌/우** | 섹션 전환 | 말씀노트 섹션 (서론/본론/결론/적용) | 탭 바 터치 |
| **스와이프 하** | 바텀시트 닫기 | 모든 바텀시트 | [닫기] 버튼 |
| **풀투리프레시** | 데이터 새로고침 | 목록 화면 (노트, 교인, 공지 등) | 새로고침 버튼 |
| **핀치 줌** | 사진 확대/축소 | 갤러리, 교회 사진 | 버튼 줌 |

**고령 교인(60대+) 대응 원칙**:
- 모든 스와이프 동작에 반드시 **버튼 대안** 제공
- 핀치 줌 대신 **[+] [-] 버튼** 제공
- 롱프레스 대신 **점 3개 메뉴(more)** 버튼 제공
- 복잡한 멀티터치 제스처 사용 금지 (최대 1-finger gesture)

### 3.2 전환 애니메이션

| 전환 유형 | 애니메이션 | 시간 | 이징 함수 | 비고 |
|-----------|-----------|------|-----------|------|
| **페이지 전환 (순방향)** | slide-left + fade | 250ms | ease-out (cubic-bezier(0, 0, 0.2, 1)) | 다음 화면 진입 |
| **페이지 전환 (역방향)** | slide-right + fade | 250ms | ease-out | 뒤로 가기 |
| **모달 표시** | fade + slide-up + scale(0.95→1.0) | 250ms | ease-out | 중앙 모달 |
| **모달 닫기** | fade-out + slide-down | 200ms | ease-in | - |
| **바텀시트 표시** | slide-up + backdrop fade | 250ms | ease-out | 하단에서 올라옴 |
| **바텀시트 닫기** | slide-down | 200ms | ease-in | 스와이프 or 버튼 |
| **토스트 표시** | slide-down from top | 150ms | ease-out | 상단에서 내려옴 |
| **토스트 사라짐** | fade-out + slide-up | 150ms | ease-in | 2-3초 후 자동 |
| **탭 전환** | fade | 150ms | ease-in-out | 네비게이션 탭 간 |
| **카드 확장** | expand + fade-in content | 300ms | ease-out | 목록→상세 전환 |

#### Reduced Motion 대응

```
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

- `prefers-reduced-motion` 미디어 쿼리 필수 적용
- 모션 감소 모드에서는 모든 애니메이션을 **즉시 전환(instant)**으로 대체
- 로딩 스피너는 유지 (기능적 목적)

├── QR 표시까지 탭 수
├── 방문자 등록 완료 시간: 목표 2분 이내
├── 폼 필드 이해도: 각 필드의 의미를 이해했는지
└── 교인관리 실시간 반영 확인 여부
```

#### 태스크 5: 출석 체크 (수동)

```
시나리오: "주일예배 1부 출석을 체크합니다.
출석 체크 화면으로 이동하여 수동 체크 모드에서
교인 5명을 선택하여 출석 체크해주세요."

측정 기준:
├── 출석 화면 진입까지 탭 수
├── 예배 선택 이해도
├── 체크박스 인지 및 사용 편의성
├── 5명 체크 소요 시간: 목표 30초 이내 (초성 검색 미사용)
└── 실시간 카운터 인지 여부
```

### 4.3 측정 항목

| 측정 항목 | 설명 | 목표 |
|-----------|------|------|
| **태스크 완료율** | 도움 없이 태스크를 완료한 비율 | 90% 이상 |
| **소요시간** | 각 태스크 시작-완료 시간 | 목표 시간 이내 |
| **에러율** | 잘못된 터치/잘못된 경로 비율 | 10% 이하 |
| **에러 복구율** | 에러 발생 후 스스로 복구한 비율 | 80% 이상 |
| **SUS 점수** | System Usability Scale (10문항) | 70점 이상 (Good) |
| **NPS** | "이 앱을 추천하겠습니까?" (0-10) | 40+ (Good) |
| **첫 동작 정확도** | 첫 번째 터치가 올바른 대상인 비율 | 85% 이상 |
| **도움 요청 횟수** | 테스트 중 질문한 횟수 | 태스크당 1회 이하 |

### 4.4 60대 교인 특별 테스트

60대 교인(페르소나: 최영자 집사)을 대상으로 추가 측정하는 항목:

| 테스트 항목 | 측정 방법 | 합격 기준 |
|------------|-----------|-----------|
| 말씀노트 녹음 시작까지 탭 수 | 화면 녹화 + 탭 카운트 | 3탭 이내 |
| 녹음 버튼 인지 시간 | 버튼을 찾아 터치하기까지 시간 | 5초 이내 |
| 글자 가독성 | "이 글자가 잘 보이시나요?" | "잘 보인다" 응답 |
| 버튼 터치 성공률 | 의도한 버튼을 정확히 터치하는 비율 | 95% 이상 |
| 네비게이션 이해도 | 홈으로 돌아가는 방법을 아는지 | 자력 복귀 가능 |
| 에러 메시지 이해도 | 에러 발생 시 대응 행동 | 자력 해결 가능 |
| 전체 앱 인상 | "이 앱을 매주 사용하실 의향이 있으신가요?" | 긍정 응답 |

### 4.5 테스트 진행 절차

```
1. 사전 준비 (5분)
   ├── 참가자 환영 및 동의서 작성
   ├── 녹화/녹음 동의 확인
   └── "정답은 없습니다. 앱을 테스트하는 것이지 참가자를 테스트하는 것이 아닙니다"

2. 사전 인터뷰 (5분)
   ├── 현재 스마트폰 사용 습관
   ├── 교회 관련 앱 사용 경험
   └── 디지털 기기에 대한 자신감 (1-5점)

3. 태스크 수행 (30-40분)
   ├── 5개 태스크 순차 진행
   ├── Think Aloud 프로토콜 (생각을 말로 표현)
   ├── 관찰자: 행동/표정/발언 기록
   └── 도움 요청 시 최소한의 힌트 제공 (3단계)

4. 사후 설문 (10분)
   ├── SUS 설문 (10문항)
   ├── NPS 질문
   ├── 자유 의견: "가장 좋았던 점 / 어려웠던 점"
   └── 추가 기능 요청

5. 디브리핑 (5분)
   └── 감사 인사 + 보상 지급 (문화상품권 3만원)
```

### 4.6 테스트 결과 활용

```
테스트 결과 → 이슈 분류 → 우선순위 결정 → 수정 반영

이슈 분류:
├── Critical (P0): 태스크 완료 불가 → 즉시 수정
├── Major (P1): 완료는 가능하나 큰 혼란 → MVP 전 수정
├── Minor (P2): 불편하지만 기능에 지장 없음 → Phase 2 수정
└── Enhancement: 개선 아이디어 → 백로그 등록

수정 후 재테스트:
├── Critical/Major 이슈 수정 후 동일 참가자 3명으로 재테스트
└── 합격 기준 충족 시 개발 착수
```
