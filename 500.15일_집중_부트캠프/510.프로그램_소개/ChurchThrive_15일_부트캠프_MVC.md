## 🎯 Mission (사명) - 우리가 존재하는 이유

### 핵심 사명

```yaml
ChurchThrive 15일 부트캠프의 사명:

"모든 교회 사역자가
 시스템적 사고와 도구를 갖추어
 하나님 나라 확장에 집중하도록 돕는다"
```

### 사명 배경: 왜 이 부트캠프가 필요한가?

#### 문제 인식

```markdown
대한민국 교회 사역자들의 현실:
━━━━━━━━━━━━━━━━━━━━━━━━

📊 통계 (2024년 교회 실태 조사):

  1. 행정 업무 과부하
     - 목회자의 62%: "행정에 하루 3시간 이상"
     - 원하는 설교 준비 시간: 주 20시간
     - 실제 설교 준비 시간: 주 8시간
     → 격차: 60%

  2. 정보 관리 혼돈
     - 78%: "파일 찾는 데 하루 평균 30분 이상"
     - 91%: "중요한 정보 놓쳐서 문제 발생 경험"
     - 83%: "체계적 관리 방법을 모름"

  3. 번아웃 위기
     - 목회자 번아웃률: 47% (2024년 기준)
     - 주 평균 근무 시간: 63시간
     - 워라밸 만족도: 2.3/5.0

  4. 시스템 부재
     - 85%: "우리 교회는 시스템이 없음"
     - 72%: "담당자 바뀌면 모든 게 리셋"
     - 68%: "지식이 개인에게 종속됨"

━━━━━━━━━━━━━━━━━━━━━━━━
결론: 사역자는 열심히 일하지만
      '효율적으로' 일하는 법을 모른다.
━━━━━━━━━━━━━━━━━━━━━━━━
```


## 💎 Core Values (핵심 가치) - 우리가 지키는 원칙

### Value 1: 실용주의 (Practicality)

```yaml
정의:
  "이론보다 실전, 지식보다 적용"

원칙:
  - 모든 학습은 '우리 교회' 데이터로
  - 70% 이해하면 다음 단계로
  - 완벽한 시스템보다 작동하는 시스템
  - 졸업 = 즉시 사용 가능한 시스템 보유

실천:
  - 예제 중심 교육 (따라하기)
  - 실습 시간 70% (이론 30%)
  - 과제 = 실제 교회 문제 해결
  - 졸업 작품 = 실제 운영 시스템

금지:
  - 추상적 개념 나열
  - 쓸모없는 고급 기능 과시
  - "알면 좋은" 내용 과다
  - 실전 없는 이론
```

### Value 2: 사람 중심 (People-Centered)

```yaml
정의:
  "도구는 수단, 사람이 목적"

원칙:
  - 시스템은 사람을 섬긴다 (지배 X)
  - 효율은 영혼 돌봄을 위해
  - 자동화는 관계에 집중하기 위해
  - 기술은 사랑을 확장하는 도구

실천:
  - 성도 프로파일 중심 시스템
  - 관계 네트워크 시각화
  - 기도 제목 관리 핵심
  - 개인 맞춤형 케어 자동화

금지:
  - 숫자만 보는 경영
  - 효율을 위한 효율
  - 사람을 데이터로 환원
  - 관계 없는 자동화
```

### Value 3: 지속 가능성 (Sustainability)

```yaml
정의:
  "15일로 끝이 아니라 평생 성장"

원칙:
  - 일회성 교육 아닌 습관 형성
  - 졸업 = 시작점
  - 시스템은 성장하는 유기체
  - 커뮤니티는 평생 자산

실천:
  - 졸업 후 평생 학습 자료 접근
  - 월간 졸업생 모임
  - 지속적 콘텐츠 업데이트
  - 인증 코치 양성 (졸업생 재투자)

금지:
  - "돈 받고 끝" 비즈니스
  - 졸업 후 방치
  - 구버전 콘텐츠 고수
  - 폐쇄적 운영
```

### Value 4: 탁월함 (Excellence)

```yaml
정의:
  "하나님께 영광 돌릴 최고 품질"

원칙:
  - 업계 최고 수준 추구
  - 지속적 개선 (Kaizen)
  - 실패는 학습 기회
  - "대충"은 없다

실천:
  - 코치 엄격한 선발 및 교육
  - 커리큘럼 분기별 업데이트
  - 참여자 피드백 100% 검토
  - 성공률 85% 이상 유지

금지:
  - 현상 유지 안주
  - 비판 무시
  - 품질보다 양 추구
  - 검증 없는 새 기능 추가
```

### Value 5: 접근성 (Accessibility)

```yaml
정의:
  "모든 교회, 모든 사역자를 위해"

원칙:
  - 규모 무관 (30명~1,500명)
  - 역할 무관 (목사, 전도사, 평신도)
  - 나이 무관 (20대~60대)
  - 실력 무관 (컴맹도 가능)

실천:
  - 합리적 가격 (50만원)
  - 소형 교회 장학금 (30% 할인)
  - 초보자 눈높이 콘텐츠
  - 온라인 완전 지원

금지:
  - 엘리트주의
  - 가격 장벽
  - 전문 용어 남발
  - 특정 교단/그룹 편향
```

### Value 6: 공동체 (Community)

```yaml
정의:
  "혼자보다 함께, 경쟁보다 협력"

원칙:
  - 동료 학습 = 최고의 학습
  - 성공은 나눌수록 커진다
  - 앞선 이는 뒤따르는 이를 돕는다
  - 커뮤니티가 곧 자산

실천:
  - 온라인 커뮤니티 운영
  - 스터디 그룹 매칭
  - 졸업생 멘토링 프로그램
  - 성공 사례 적극 공유

금지:
  - 개인주의 조장
  - 순위/등수 공개
  - 과도한 경쟁 유발
  - 지식 독점
```

### Value 7: 투명성 (Transparency)

```yaml
정의:
  "숨기지 않고, 꾸미지 않고, 있는 그대로"

원칙:
  - 성공도 실패도 공개
  - 가격 정책 명확
  - 커리큘럼 사전 공개
  - 성과 데이터 투명 공개

실천:
  - 홈페이지에 모든 정보 공개
  - 졸업생 솔직 후기 (편집 금지)
  - 완주율, 만족도 통계 공개
  - 환불 정책 명확

금지:
  - 과장 광고
  - 부정적 후기 삭제
  - 숨겨진 비용
  - 데이터 조작
```

```markdown
ChurchThrive 부트캠프의 모든 의사결정은

     ☐ 공동체
```

```yaml
가치_충돌_시_우선순위:

1순위: Mission (사명)
  - 사역자를 돕는가?
  - 이것이 최우선

2순위: People-Centered (사람 중심)
  - 사람이 먼저
  - 시스템은 다음

3순위: Excellence (탁월함)
  - 품질 타협 불가
  - "대충"은 배반

4순위: Accessibility (접근성)
  - 소수보다 다수
  - 엘리트보다 대중

5순위: Practicality (실용주의)
  - 이론보다 실전
  - 작동하는 게 최고

6순위: Sustainability (지속 가능성)
  - 단기보다 장기
  - 지금보다 미래

7순위: Community (공동체)
  - 혼자보다 함께
  - 독점보다 공유

예시:
  "최신 AI 기능 추가?"
    → Practicality (실용성): 실제 도움되나? ✅
    → Accessibility (접근성): 모두 쓸 수 있나? ❌
    → 결정: 일부만 선택적 제공 (고급 과정)
```

## 🤝 참여자에게 드리는 약속

### ChurchThrive 부트캠프가 당신에게 약속합니다

```markdown
━━━━━━━━━━━━━━━━━━━━━━━━
15일 부트캠프 참여자에게 드리는 7가지 약속
━━━━━━━━━━━━━━━━━━━━━━━━

약속 1: 완주 지원
  "당신이 포기하지 않는 한, 우리도 포기하지 않습니다"

  - 전담 코치 배정 (1:10 비율)
  - 24시간 커뮤니티 지원
  - 막히면 즉시 도움
  - 재수강 무료 (1회)

약속 2: 실용성 보장
  "졸업 = 즉시 사용 가능한 시스템 보유"

  - 100% 실전 중심 교육
  - 예제가 아닌 "당신의 교회" 데이터
  - 졸업 작품 = 실제 운영 시스템
  - 졸업 후 바로 적용

약속 3: 맞춤형 코칭
  "당신의 교회는 유일하고, 코칭도 유일합니다"

  - 교회 규모별 맞춤 가이드
  - 역할별 맞춤 조언
  - 개인 진도에 맞춘 피드백
  - 1:1 화상 상담 (필요 시)

약속 4: 평생 학습 지원
  "15일은 시작, 평생이 여정입니다"

  - 졸업 후 학습 자료 평생 무료
  - 월간 졸업생 모임
  - 콘텐츠 업데이트 자동 제공
  - 커뮤니티 평생 회원

약속 5: 투명한 운영
  "숨기는 것 없이, 있는 그대로 보여드립니다"

  - 모든 비용 사전 공지
  - 커리큘럼 100% 공개
  - 성과 데이터 투명 공개
  - 환불 정책 명확 (7일 이내 100% 환불)

약속 6: 성과 책임
  "효과 없으면 책임지겠습니다"

  - 완주율 85% 보장 (미달 시 개선)
  - 만족도 4.5/5.0 유지
  - 불만족 시 재수강 무료
  - 피드백 100% 반영

약속 7: 지속적 개선
  "오늘보다 나은 내일을 만들겠습니다"

  - 분기별 커리큘럼 업데이트
  - 참여자 피드백 즉시 반영
  - 최신 도구 즉시 도입
  - 품질 향상 끊임없이

━━━━━━━━━━━━━━━━━━━━━━━━
이 약속을 지키지 못하면,
등록비 전액 환불 + 추가 보상을 드립니다.
━━━━━━━━━━━━━━━━━━━━━━━━
```

## 👨‍🏫 코치팀의 서약

### ChurchThrive 코치로서의 헌신

```markdown
━━━━━━━━━━━━━━━━━━━━━━━━
코치 서약문
━━━━━━━━━━━━━━━━━━━━━━━━

나는 ChurchThrive 15일 부트캠프 코치로서
다음을 엄숙히 서약합니다:

1. 섬김의 자세
   나는 참여자를 "가르치는" 것이 아니라
   "섬기는" 것임을 명심하겠습니다.

2. 인내와 존중
   나는 참여자의 속도를 존중하고
   100번 물어도 100번 친절히 답하겠습니다.

3. 탁월함 추구
   나는 항상 최고 품질의 코칭을 제공하기 위해
   지속적으로 학습하고 개선하겠습니다.

4. 투명성 유지
   나는 모르는 것은 모른다고 인정하고
   확인 후 정확히 답변하겠습니다.

5. 비밀 보장
   나는 참여자의 교회 정보와 개인 정보를
   철저히 보호하겠습니다.

6. 차별 금지
   나는 교회 규모, 교단, 개인 배경에 관계없이
   모든 참여자를 동등하게 대하겠습니다.

7. 책임감
   나는 내가 맡은 참여자의 완주를 위해
   최선을 다하겠습니다.

8. 지속 동행
   나는 졸업 후에도 참여자와의 관계를 유지하며
   평생 동반자가 되겠습니다.

━━━━━━━━━━━━━━━━━━━━━━━━
이 서약을 어기는 코치는 즉시 자격이 박탈됩니다.
━━━━━━━━━━━━━━━━━━━━━━━━

서명: ________________
날짜: ________________
```

## 🎯 MVC 적용 사례

### 실제 의사결정에서 MVC는 어떻게 작동하는가?

#### 사례 1: "부트캠프를 30일로 늘릴까?"

```yaml
제안:
  "참여자들이 시간 부족을 호소합니다.
   30일로 늘리면 어떨까요?"

MVC 검토:

Mission 부합:
  - 사역자를 돕는가? 🤔
  - 30일 = 일정 부담 증가
  - 완주율 오히려 감소 가능
  → ⚠️ 부분 부합

Vision 정합성:
  - 2030 비전: "모든 교회"
  - 30일 = 장벽 상승
  → ❌ 불합

Values 체크:
  - Practicality: 15일 = 실용적 ✅
  - Accessibility: 30일 = 접근성 ↓ ❌
  - Sustainability: 장기 지속 🤔

결정: ❌ 기각
  대신 "15일 + 선택적 연장 프로그램" 도입
```

#### 사례 2: "등록비를 100만원으로 올릴까?"

```yaml
제안:
  "품질 향상을 위해 투자가 필요합니다.
   등록비를 100만원으로 올리면 어떨까요?"

MVC 검토:

Mission 부합:
  - "모든" 사역자를 돕는가?
  - 100만원 = 소형 교회 부담
  → ❌ 불합

Vision 정합성:
  - 2030: "모든 교회"
  - 100만원 = 소수만 접근 가능
  → ❌ 불합

Values 체크:
  - Excellence: 품질 투자 필요 ✅
  - Accessibility: 가격 장벽 ↑ ❌
  - People-Centered: 사람보다 돈? ❌

결정: ❌ 기각
  대신 "50만원 유지 + 고급 과정 별도 개설"
```

#### 사례 3: "AI 자동 코칭 도입?"

```yaml
제안:
  "ChatGPT 기반 24시간 AI 코칭을 도입하면
   코치 인건비를 절감하고 즉시 응답 가능합니다."

MVC 검토:

Mission 부합:
  - 사역자를 돕는가? ✅
  - 즉시 응답 = 도움
  → ✅ 부합

Vision 정합성:
  - 확장성 ↑ (코치 부족 해결)
  → ✅ 부합

Values 체크:
  - Practicality: 즉시 답변 ✅
  - People-Centered: AI vs 사람? 🤔
  - Excellence: AI 품질 검증 필요 🤔
  - Accessibility: 24시간 접근 ✅

결정: ⚠️ 조건부 승인
  - AI는 "보조" 역할 (1차 답변)
  - 복잡한 질문은 사람 코치에게 에스컬레이션
  - AI 답변 품질 모니터링 필수
  - 사람 코치를 대체하지 않음
```
