## 왜 팀 협업이 필요한가?

### 교회 협업 시나리오

#### 시나리오 1: 담임목사 + 부목사

**협업 필요성**:
```
담임목사
  └─ 주일 설교 작성
  └─ 목회 계획 수립

부목사
  └─ 수요 예배 준비
  └─ 교육부 자료 작성

→ 하나의 Vault에서 함께 작업!
```

**효과**:
- ✅ 서로의 자료 참고
- ✅ 중복 작업 방지
- ✅ 통일된 자료 관리

---

#### 시나리오 2: 목회자 + 행정 사역자

**협업 필요성**:
```
목회자
  └─ 설교 노트
  └─ 심방 기록

행정 사역자
  └─ 주보 작성
  └─ 재정 관리
  └─ 회의록 작성

→ 실시간 동기화!
```

**효과**:
- ✅ 주보에 설교 제목 자동 반영
- ✅ 일정 실시간 공유
- ✅ 효율적 업무 분담

---

#### 시나리오 3: 다수의 사역자 팀

**협업 필요성**:
```
담임목사 + 부목사 + 교육전도사 + 청년전도사 + 행정간사

→ 5명이 하나의 Vault 공유
→ 각자 담당 영역 작업
→ 서로 참고 및 연결
```

**효과**:
- ✅ 통합 지식 베이스
- ✅ 크로스 레퍼런스
- ✅ 팀워크 향상

### 추천 솔루션 (상황별)

#### 🥇 중소형 교회 (2-3명)
**추천**: **OneDrive** 또는 **Google Drive**
- 설치 쉬움
- 자동 동기화
- 모바일 지원

#### 🥈 기술에 자신 있는 팀
**추천**: **Git (GitHub)**
- 최고의 충돌 관리
- 완벽한 버전 관리
- 무제한 용량

#### 🥉 완전 무료 + 무제한
**추천**: **Syncthing**
- 용량 제한 없음
- 서버 불필요 (P2P)
- 완전 프라이버시

#### 🏢 NAS 보유 교회
**추천**: **NAS WebDAV**
- 자체 서버 (완전 통제)
- NAS 용량만큼 무제한
- 빠른 로컬 네트워크 속도
- 추가 비용 없음

## 방법 1: OneDrive (가장 추천!)

### 🌟 장점

- ✅ Windows에 기본 내장
- ✅ 자동 동기화
- ✅ 모바일 앱 우수
- ✅ Office 통합
- ✅ 무료 5GB (충분)

---

### Step 1: OneDrive 설치 및 설정 (5분)

#### Windows (기본 내장)

1. **OneDrive 실행**
   - 작업 표시줄 우측 하단 구름 아이콘 클릭
   - 또는 시작 메뉴 → "OneDrive"

2. **Microsoft 계정 로그인**
   - 개인 계정 또는 회사 계정

3. **동기화 폴더 선택**
   - 기본: `C:\Users\[사용자]\OneDrive\`
   - 변경 가능

4. **완료!**

#### Mac

1. **OneDrive 다운로드**
   - https://www.microsoft.com/en-us/microsoft-365/onedrive/download

2. **설치 및 로그인**

3. **동기화 시작**

---

### Step 2: Vault를 OneDrive로 이동 (10분)

#### 방법 1: 새 Vault 생성 (추천)

1. **Obsidian 실행**
   - "Create new vault"

2. **위치 선택**
   ```
   C:\Users\[사용자]\OneDrive\우리교회
   ```

3. **Vault 생성**
   - 이제 OneDrive에 자동 동기화!

---

#### 방법 2: 기존 Vault 이동

1. **기존 Vault 폴더 복사**
   ```
   원본: C:\Users\[사용자]\Documents\우리교회
   대상: C:\Users\[사용자]\OneDrive\우리교회
   ```

2. **Obsidian에서 Vault 열기**
   - "Open folder as vault"
   - OneDrive 경로 선택

3. **동기화 확인**
   - 파일 생성 → OneDrive 아이콘 확인 (체크 표시)

---

### Step 3: 팀원 초대 (5분)

#### OneDrive 공유 설정

1. **OneDrive 웹 접속**
   - https://onedrive.live.com

2. **"우리교회" 폴더 우클릭**
   - "공유" 선택

3. **초대 방법 선택**

   **방법 A: 이메일 초대** (추천)
   ```
   - 팀원 이메일 입력
   - 권한 선택: "편집 가능"
   - "보내기"
   ```

   **방법 B: 링크 공유**
   ```
   - "링크 복사" 클릭
   - 권한 설정: "편집 가능"
   - 카카오톡으로 링크 전송
   ```

4. **팀원 수락**
   - 이메일 확인 → "열기" 클릭
   - OneDrive에 공유 폴더 추가됨

---

### Step 4: 팀원 설정 (각자 10분)

**팀원이 할 일**:

1. **OneDrive 설치 및 로그인**
   - 각자 PC에 설치

2. **공유 폴더 동기화**
   - OneDrive 설정 → "동기화할 폴더 선택"
   - "우리교회" 체크 ✅

3. **Obsidian에서 Vault 열기**
   ```
   C:\Users\[팀원]\OneDrive\우리교회
   ```

4. **완료!**
   - 이제 실시간 동기화 시작

---

### Step 5: 협업 시작! (즉시)

**담임목사**:
```
1. 설교 노트 작성
2. 저장 (Ctrl+S)
3. 자동으로 OneDrive 동기화 ↑
```

**행정 사역자** (1분 후):
```
1. 설교 노트 자동으로 동기화 ↓
2. 주보에 설교 제목 복사
3. 저장
4. 자동 동기화 ↑
```

**실시간 협업!** ✨

## 방법 2: Google Drive

### 🌟 장점

- ✅ 무료 15GB (가장 많음!)
- ✅ 모든 OS 지원
- ✅ 구글 계정 하나로
- ✅ 웹에서도 접근 가능

---

### Step 1: Google Drive 설치 (10분)

#### Windows/Mac

1. **Google Drive Desktop 다운로드**
   - https://www.google.com/drive/download/

2. **설치 및 로그인**
   - Google 계정으로 로그인

3. **동기화 방식 선택**

   **방법 A: 미러링** (추천)
   ```
   - PC에 폴더 생성
   - Google Drive와 양방향 동기화
   - 오프라인 작업 가능
   ```

   **방법 B: 스트리밍**
   ```
   - 클라우드에만 저장
   - 필요할 때 다운로드
   - 용량 절약 (느림)
   ```

   **추천**: 미러링 선택!

4. **동기화 폴더**
   ```
   C:\Users\[사용자]\Google Drive\우리교회
   ```

---

### Step 2: Vault 설정 (OneDrive와 동일)

1. **Vault를 Google Drive로 이동**
2. **공유 설정**
   - Google Drive 웹 → 폴더 우클릭 → "공유"
3. **팀원 초대**
   - 이메일 입력 → "편집자" 권한

---

### 주의사항

**⚠️ Google Drive 동기화 느림**
- OneDrive보다 살짝 느림
- 큰 파일 (이미지 많을 때) 주의
- Wi-Fi 환경 권장

**해결책**:
- 이미지 압축
- 첨부파일 최소화
- 필요시 OneDrive로 전환

---

## 방법 3: iCloud Drive

### 🌟 장점

- ✅ Mac 사용자에게 최적
- ✅ 애플 생태계 통합
- ✅ 자동 동기화

### ⚠️ 단점

- ❌ Windows 지원 제한적
- ❌ 무료 5GB만
- ❌ 동기화 때때로 불안정

---

### Step 1: iCloud 설정 (Mac)

1. **시스템 설정**
   - Apple 메뉴 → "시스템 설정"
   - "Apple ID" → "iCloud"

2. **iCloud Drive 활성화**
   - "iCloud Drive" 체크 ✅

3. **Vault 이동**
   ```
   ~/Library/Mobile Documents/com~apple~CloudDocs/우리교회
   ```

---

### Step 2: Windows 설정 (선택)

**⚠️ Windows용 iCloud 필요**
- 다운로드: https://support.apple.com/ko-kr/HT204283
- 설치 후 로그인
- iCloud Drive 동기화 활성화

**추천**: Windows 팀원이 있다면 OneDrive 사용 권장

## 방법 4: Syncthing (완전 무료!)

### 🌟 장점

- ✅ **무제한 용량** (자체 서버)
- ✅ 완전 무료
- ✅ P2P (중앙 서버 없음)
- ✅ 프라이버시 최고
- ✅ 빠른 동기화

### ⚠️ 단점

- ❌ 설정 복잡
- ❌ iOS 지원 없음
- ❌ 웹 접근 불가

---

### Step 1: Syncthing 설치 (15분)

#### Windows

1. **다운로드**
   - https://syncthing.net/downloads/
   - "Windows" 버전 다운로드

2. **설치**
   - 압축 해제
   - `syncthing.exe` 실행

3. **웹 UI 자동 열림**
   - http://127.0.0.1:8384

#### Mac

```bash
brew install syncthing
syncthing
```

---

### Step 2: 폴더 공유 설정 (10분)

**담임목사 (PC A)**:

1. **폴더 추가**
   - 웹 UI → "폴더 추가"
   - 경로: `C:\Users\[사용자]\우리교회`
   - 폴더 ID: `church-vault`

2. **디바이스 ID 확인**
   - 우측 상단 "작업" → "장치 ID 표시"
   - ID 복사 (예: `ABCD-1234-...`)

---

**부목사 (PC B)**:

1. **Syncthing 설치 및 실행**

2. **디바이스 추가**
   - "원격 장치 추가"
   - 담임목사 장치 ID 입력

3. **폴더 수락**
   - 담임목사가 폴더 공유 요청
   - "수락" 클릭
   - 저장 경로 선택

4. **동기화 시작!**

---

### Step 3: 추가 팀원 (반복)

- 각 팀원이 Syncthing 설치
- 서로의 장치 ID 교환
- 폴더 공유 요청 → 수락
- **모든 PC가 P2P로 연결!**

---

### 장점

**시나리오**:
```
담임목사 PC ←→ 부목사 PC
       ↕            ↕
  행정간사 PC ←→ 교육전도사 PC
```

- 중앙 서버 없이 P2P 동기화
- 빠름! (로컬 네트워크 사용)
- 프라이버시 완벽

---

## 방법 5: Git (고급, 가장 강력)

### 🌟 장점

- ✅ **최고의 충돌 관리**
- ✅ **완벽한 버전 관리**
- ✅ 무제한 용량
- ✅ 변경 이력 추적
- ✅ 브랜치 작업 가능

### ⚠️ 단점

- ❌ 기술적 학습 필요
- ❌ 수동 commit/push
- ❌ 초보자에게 어려움

---

### Step 1: Git 및 GitHub Desktop 설치

**(앞서 웹 퍼블리싱 가이드 참조)**

1. GitHub 계정 생성
2. GitHub Desktop 설치
3. Repository 생성

---

### Step 2: Vault를 Git Repository로

1. **GitHub Desktop**
   - File → "Add Local Repository"
   - Vault 폴더 선택

2. **초기 커밋**
   - Summary: "Initial commit"
   - Commit to main

3. **GitHub에 Push**
   - "Publish repository"
   - Private으로 설정 ✅

---

### Step 3: 팀원 초대

1. **GitHub 웹**
   - Repository → Settings
   - Collaborators → "Add people"
   - 팀원 GitHub 계정 입력

2. **팀원 수락**
   - 이메일 확인 → 초대 수락

3. **팀원 Clone**
   - GitHub Desktop → "Clone repository"
   - 로컬에 복사

---

### Step 4: Git 워크플로우

**매일 작업 시작**:
```
1. GitHub Desktop 열기
2. "Fetch origin" (최신 변경 가져오기)
3. "Pull" (로컬에 적용)
4. Obsidian에서 작업 시작
```

**작업 완료 후**:
```
1. 변경 사항 확인
2. Commit 메시지 작성: "주보 업데이트"
3. "Commit to main"
4. "Push origin" (GitHub에 업로드)
```

---

### Step 5: 충돌 해결 (고급)

**충돌 발생 시**:
```
1. GitHub Desktop에 충돌 알림
2. "Open in Visual Studio Code" (또는 에디터)
3. 충돌 부분 수동 병합
   <<<<<<< HEAD
   내 변경 사항
   =======
   팀원 변경 사항
   >>>>>>> branch-name
4. 수동으로 선택 또는 병합
5. 저장 → Commit → Push
```

**장점**: 데이터 손실 없음!

## 방법 6: NAS WebDAV (자체 서버)

### 🌟 장점

- ✅ **완전한 자체 호스팅** (클라우드 의존 없음)
- ✅ **NAS 용량만큼 무제한** (1TB, 2TB, 4TB...)
- ✅ **빠른 로컬 네트워크** (사무실 내 초고속)
- ✅ **완벽한 프라이버시** (교회 서버에만 저장)
- ✅ **추가 비용 없음** (NAS 이미 보유 시)
- ✅ **권한 관리 세밀함**

### ⚠️ 단점

- ❌ **NAS 하드웨어 필요** (초기 투자 20-100만원)
- ❌ **설정 복잡** (IT 지식 필요)
- ❌ **외부 접근 어려움** (VPN 또는 포트포워딩 필요)
- ❌ **NAS 관리 필요** (업데이트, 백업)

---

- 대용량 파일 관리 필요 (동영상, 사진)

## Step 2: Windows에서 연결 (15분)

### 방법 A: 네트워크 드라이브 연결 (추천)

1. **파일 탐색기 열기**

2. **네트워크 드라이브 연결**


3. **설정**
   ```yaml
   드라이브: Z:
   폴더: http://[NAS_IP]:5005/우리교회

   ✅ 로그인할 때 다시 연결
   ✅ 다른 자격 증명을 사용하여 연결
   ```

   ```
   사용자 이름: 담임목사

   ✅ 자격 증명 저장
   ```

   - 이제 `Z:\우리교회` 폴더 생성됨!

## WebDAV vs 클라우드 비교

### WebDAV의 장점

**1. 속도**
```
로컬 네트워크 (사무실 내):
  OneDrive: 10-30 MB/s
  NAS WebDAV: 100-1000 MB/s (초고속!)

외부 접근:
  OneDrive: 인터넷 속도 의존
  NAS WebDAV: 인터넷 속도 의존 (동일)
```

**2. 용량**
```
OneDrive: 5GB (무료), 100GB (2,390원/월)
NAS: 2TB, 4TB, 8TB... (무제한)
```

**3. 프라이버시**
```
OneDrive: Microsoft 서버
NAS: 교회 서버 (완전 통제)
```

---

### 클라우드의 장점

**1. 설정 간편**
```
OneDrive: 5분
NAS: 30-60분
```

**2. 외부 접근**
```
OneDrive: 언제 어디서나
NAS: VPN 또는 QuickConnect 필요
```

**3. 백업**
```
OneDrive: 자동 백업 (Microsoft)
NAS: 수동 백업 필요
```

## 주의사항

### ⚠️ 주의 1: NAS 백업 필수

**NAS도 고장날 수 있습니다!**

**백업 전략**:
```
1차 백업: NAS RAID (하드디스크 이중화)
2차 백업: 외장하드 주간 백업
3차 백업: 클라우드 (중요 파일만)
```

---

### ⚠️ 주의 2: 외부 접근 보안

**절대 금지**:
- 포트 포워딩 + 약한 비밀번호
- HTTPS 없이 WebDAV 노출

**권장**:
- VPN 사용
- HTTPS (SSL 인증서)
- 강력한 비밀번호 + 2단계 인증

---

### ⚠️ 주의 3: 성능 관리

**느려지는 이유**:
- NAS 하드디스크 노후화
- 동시 접속자 너무 많음
- 네트워크 과부하

**해결**:
- NAS RAM 업그레이드
- SSD 캐시 추가
- 기가비트 네트워크 사용

## 문제 해결

### 문제 1: 동기화가 안 돼요!

**체크리스트**:
- [ ] 인터넷 연결 확인
- [ ] OneDrive/Google Drive 실행 중인가?
- [ ] 동기화 일시 중지 상태 아닌가?
- [ ] 저장 공간 부족한가? (무료 용량 초과)

**해결**:
```
OneDrive 아이콘 우클릭
→ "설정"
→ "계정"
→ "동기화 재개"
```

---

### 문제 2: 충돌 파일이 생겼어요!

**예시**:
```
주보.md
주보-충돌복사본-2025-01-14.md  ← 충돌!
```

**해결 방법**:

1. **두 파일 비교**
   - 원본과 충돌복사본 열기
   - 차이점 확인

2. **수동 병합**
   - 필요한 부분 복사
   - 원본에 붙여넣기

3. **충돌복사본 삭제**

4. **재발 방지**
   - 시간대 분리 규칙 적용

---

### 문제 3: 삭제한 파일 복구하고 싶어요!

**OneDrive**:
```
1. OneDrive 웹 접속
2. "휴지통" 클릭
3. 파일 선택 → "복원"
```

**보관 기간**: 30일

**Git**:
```
1. GitHub Desktop → History
2. 삭제 전 커밋 찾기
3. "Revert" 클릭
```

**보관 기간**: 영구!

---

### 문제 4: 동기화가 너무 느려요!

**원인**:
- 파일 너무 많음 (10,000개+)
- 큰 이미지 파일
- 느린 인터넷

**해결**:

1. **이미지 압축**
   - 2MB 이상 이미지 압축
   - PNG → JPG 변환

2. **첨부파일 정리**
   - 불필요한 파일 삭제
   - 외부 링크 사용 (YouTube, Google Drive 링크)

3. **선택적 동기화**
   - 과거 문서 (5년 전) 제외
   - "문서보관" 폴더 동기화 해제

---

### 문제 5: 팀원이 실수로 중요 파일을 삭제했어요!

**OneDrive/Google Drive**:
- 휴지통에서 30일 내 복구 가능

**Git (최고)**:
```
1. GitHub Desktop → History
2. 삭제 커밋 찾기
3. Revert (되돌리기)
4. 파일 복구!
```

**교훈**: Git 사용하면 안전! 🔒

## FAQ

### Q1: 동시에 같은 파일을 편집하면 어떻게 되나요?

**A**: 충돌 발생 가능

**OneDrive/Google Drive**:
- 충돌 복사본 생성
- 수동 병합 필요

**Git**:
- 자동 병합 시도
- 충돌 시 수동 병합 (데이터 손실 없음)

**추천**:
1. 시간대 분리
2. 카카오톡 소통
3. 또는 Git 사용

---

### Q2: 몇 명까지 협업 가능한가요?

**A**: 제한 거의 없음

**OneDrive/Google Drive**:
- 공유 가능: 100명+
- 실용적: 5-10명

**Git**:
- 무제한

**추천**:
- 소형 교회 (2-3명): OneDrive
- 중형 교회 (5-10명): Google Drive 또는 Git
- 대형 교회 (10명+): Git

---

### Q3: 모바일에서도 편집 가능한가요?

**A**: 가능합니다!

**설정**:
1. 모바일 Obsidian 앱 설치
2. OneDrive/Google Drive 연결
3. Vault 동기화
4. 편집 가능!

**주의**:
- 모바일 편집 시 충돌 주의
- 간단한 수정만 권장
- 본격 작업은 PC에서

---

### Q4: 비용이 정말 무료인가요?

**A**: 네!

**무료 용량**:
- OneDrive: 5GB
- Google Drive: 15GB
- iCloud: 5GB
- Syncthing: 무제한
- Git (GitHub): 무제한

**충분한가?**
- 텍스트 위주: 5GB로 10년 가능
- 이미지 많으면: 15GB 추천 (Google Drive)
- 무제한 원하면: Syncthing 또는 Git

---

### Q5: 보안은 괜찮나요?

**A**: 대부분 안전

**암호화**:
- OneDrive: ✅ 암호화
- Google Drive: ✅ 암호화
- iCloud: ✅ 암호화
- Syncthing: ✅ 암호화 (P2P)
- Git: ⚠️ Private repository 필수

**추가 보안**:
- 2단계 인증 활성화
- Private repository (Git)
- 민감한 정보는 비공개 폴더

---

### Q6: 이미 사용 중인데 방식을 바꿀 수 있나요?

**A**: 가능합니다!

3. **초기 커밋 및 Push**
4. **팀원들 Git으로 전환**
5. **OneDrive 중단**

**소요**: 1-2시간

```
편집 권한 없이 보기만 허용
```
