# 대형교회 카테고리 구조 예시

> **실제 사례: 비전교회 (출석 500명)**
>
> 창립 20년, 담임목사 + 부목사 3명, 사역자 15명, 부서 15개, 다수의 팀

---

## 교회 프로필

```yaml
교회명: 비전교회
출석 인원: 500명
목회자: 담임목사 1, 부목사 3 (교육/청년/행정)
사역자: 15명
장로: 10명
안수집사: 30명

부서 (15개):
  교육부:
    - 영아부, 유아부, 유치부
    - 유년부, 초등부 (저/고학년)
    - 중등부, 고등부
  청년장년부:
    - 청년1부, 청년2부
    - 장년부, 노년부
  사역팀:
    - 찬양팀, 미디어팀, 안내팀
    - 주차팀, 식사팀

연간 문서: 약 2000-3000개
```

## 주요 특징

### 1. 세분화된 교육 부서 (3단계)

```
300.교육사역/
  310.영유아부/         (1단계)
    311.영아부/         (2단계)
    312.유아부/
    313.유치부/
  320.어린이부/
    321.유년부/
    322.초등1-3학년/
    323.초등4-6학년/
```

**이유**:
- 연령별 맞춤 교육
- 담당자 명확화
- 출석 통계 세분화

## 연간 문서 예상량

| 카테고리 | 주간 | 연간 |
|----------|------|------|
| 주보 | 1 | ~52 |
| 설교 | 8 | ~400 |
| 회의록 | 3 | ~150 |
| 재정 | 0.5 | 25 |
| 심방 | 20 | ~1000 |
| 교육 부서 | 15 | ~750 |
| 행사 | 2 | ~100 |
| 기타 | 10 | ~500 |
| **합계** | ~60 | **~2977개** |

**저장 용량**: 연간 ~30MB, 10년 후 ~300MB

## 팀 협업 체계

### 폴더별 권한 관리

| 폴더 | 총괄 | 편집자 | 열람자 |
|------|------|--------|--------|
| 100.교회운영 | 담임목사 | 행정팀 | 전체 |
| 300.교육사역 | 부목사(교육) | 각 부서장 | 부서원 |
| 400.선교전도 | 선교위원장 | 선교팀 | 전체 |
| 800.인사총무 | 담임목사 | 총무 | 제한 |

### 동기화 전략

**OneDrive 비즈니스** (추천):
- 500GB ~ 1TB 용량
- 팀원별 권한 관리
- 버전 관리 (30일)
- 실시간 동기화

**대안: Obsidian Sync** ($10/월):
- End-to-end 암호화
- 모바일 지원
- 버전 관리
