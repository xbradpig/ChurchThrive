## 1. Claude Code란?

### 🤖 개념

**Claude Code = AI 코딩 어시스턴트 + Obsidian 통합**

**전통적인 ChatGPT**:
```
사용자: "우리 교회 작년 부활절 어떻게 했어?"
ChatGPT: "죄송하지만 귀 교회 자료에 접근할 수 없습니다"
```

**Claude Code**:
```
사용자: "우리 교회 작년 부활절 어떻게 했어?"
Claude: (Vault의 CLAUDE.md 읽기 → 부활절 검색 → 관련 문서 찾기)
        "2024년 부활절 행사 요약:
        - 일시: 2024-03-31
        - 예산: 300만원
        - 참석: 250명
        첨부: [[2024년_부활절_행사_기획서]]"
```

---

### ⚡ 주요 기능

1. **Vault 직접 접근**
   - 모든 노트 읽기 가능
   - 파일 생성/수정 가능
   - 검색 및 분석

2. **CLAUDE.md 자동 로드**
   - 교회 구조 자동 파악
   - 카테고리 이해
   - 표준 규칙 적용

3. **3-Layer 시스템 최적화**
   - Layer 0 (CLAUDE.md) 항상 로드
   - Layer 1 (가이드) 필요 시 로드
   - Layer 2 (문서) 선택적 로드

---

### 💰 가격

**Claude 구독 플랜**:

| 플랜 | 가격 | Claude Code | 추천 |
|------|------|-------------|------|
| Free | 무료 | ❌ 없음 | 테스트용 |
| Pro | $20/월 | ✅ 포함 | **강력 추천** |
| Team | $25/월 (per user) | ✅ 포함 | 대형교회 |

**교회 예산 고려**:

- 중형교회: Pro 2-3개 계정 ($40-60/월)
- 대형교회: Team 플랜 고려

## 2. 설치 방법

### 🔧 사전 준비

**필요한 것**:
- ✅ Claude Pro 또는 Team 구독
- ✅ Obsidian Vault (이미 생성됨)
- ✅ 인터넷 연결

---

### Step 1: Claude 계정 생성 (5분)

1. **브라우저에서 접속**:
   ```
   https://claude.ai
   ```

2. **"Sign Up" 클릭**

3. **이메일 입력 및 인증**
   - 교회 이메일 권장
   - Gmail도 가능

4. **Pro 플랜 구독** (필수)
   - "Upgrade to Pro" 클릭
   - 신용카드 정보 입력
   - $20/월 결제

---

### Step 2: Claude Code 다운로드 (3분)

1. **Claude 웹사이트에서 Code 페이지 접속**:
   ```
   https://claude.ai/code
   ```

2. **다운로드 버튼 클릭**
   - Windows: `.exe` 다운로드
   - Mac: `.dmg` 다운로드

3. **설치 파일 실행**
   - Windows: 설치 마법사 따라하기
   - Mac: Applications 폴더로 드래그

4. **첫 실행**
   - Claude Code 실행
   - Claude 계정으로 로그인
   - 권한 승인

---

## 3. Obsidian Vault 연결

### 📁 Vault 경로 설정 (2분)

1. **Claude Code 실행**

2. **Settings 열기**
   - 톱니바퀴 아이콘 클릭

3. **Working Directory 설정**
   ```
   Windows:
   C:\Users\[사용자명]\Documents\우리교회

   Mac:
   /Users/[사용자명]/Documents/우리교회
   ```

4. **"Set as working directory" 클릭**

5. **확인**
   - Claude Code에서 파일 목록 보임
   - 왼쪽 패널에 Vault 폴더 구조 표시

**완료!** 🎉

## 4. CLAUDE.md 작성

### 📝 왜 CLAUDE.md가 중요한가?

**CLAUDE.md = AI에게 주는 "교회 설명서"**

- ✅ AI가 제일 먼저 읽는 파일
- ✅ 교회 구조 설명
- ✅ 폴더 규칙 정의
- ✅ YAML 표준 명시

---

### 교회용 CLAUDE.md 템플릿

**Vault 루트에 `CLAUDE.md` 파일 생성**:

```markdown
# CLAUDE.md

## 교회 개요

**교회명**: 새소망교회
**규모**: 중형교회 (150명)
**목회자**: 담임목사 홍길동
**주요 사역**: 주일예배, 교육, 전도, 양육

---

## Vault 구조

```
우리교회/
├── 100.교회운영/
│   ├── 110.주보/              # 매주 주일 주보
│   ├── 120.설교/              # 설교 노트
│   ├── 130.회의록/            # 운영위원회 회의록
│   ├── 140.행사/              # 특별 행사 기획
│   ├── 150.교육/              # 주일학교, 제자훈련
│   ├── 160.심방/              # 심방 기록
│   ├── 170.새가족/            # 새가족 관리
│   ├── 180.재정/              # 재정 보고
│   └── 190.자료실/            # 각종 양식
├── Templates/                  # 템플릿 모음
└── Daily/                      # 일일 노트

## 폴더 넘버링 규칙

- 상위 폴더: 100, 200, 300... (100씩 증가)
- 중위 폴더: 110, 120, 130... (10씩 증가)
- 하위 폴더: 111, 112, 113... (1씩 증가)

---

## YAML 표준

**모든 마크다운 파일은 YAML frontmatter 포함**:

```yaml
---
type: bulletin               # 문서 유형
date: 2025-11-17            # 날짜 (ISO 8601)
author:
  - "[[홍길동]]"            # 작성자 (wikilink, 따옴표 필수)
tags: [주보, 2025]          # 태그
organization: "[[새소망교회]]"  # 조직명
---
```

**문서 유형**:
- `bulletin`: 주보
- `sermon`: 설교
- `meeting`: 회의록
- `event`: 행사
- `visit`: 심방

---

## Wikilink 규칙

**내부 링크는 [[대괄호]] 사용**:
- `[[2025-11-17 주보]]` - 문서 링크
- `[[파일명|표시명]]` - 별칭
- `[[파일명#제목]]` - 헤딩 링크

**YAML에서는 따옴표 필수**:
- ✅ `organization: "[[새소망교회]]"`
- ❌ `organization: [[새소망교회]]`

---

## 주요 업무

1. **주보 작성**: 매주 일요일 주보 생성 (110.주보/)
