## 왜 웹 퍼블리싱이 필요한가?

### 현재 문제점

**Obsidian만 사용 시**:
```
❌ 목회자/사역자만 볼 수 있음
❌ 성도들은 Obsidian 설치 필요
❌ 기술적 진입 장벽 높음
❌ 모바일에서 불편함
```

### 웹 퍼블리싱 후

**웹사이트로 공개 시**:
```
✅ 누구나 브라우저로 접근
✅ 설치 불필요
✅ 모바일 최적화
✅ 검색 기능
✅ 링크로 공유 가능
```

## 실제 사용 시나리오

### 시나리오 1: 주보 및 설교 아카이브

**목회자 (Obsidian)**:
```
1. 주보 작성: "2025-01-19 주보.md"
2. 설교 노트 작성: "하나님의 사랑.md"
3. Git에 push (버튼 클릭)
```

**성도들 (웹사이트)**:
```
1. church.com 접속
2. "주보" 메뉴 클릭
3. 최신 주보 확인 (자동 업데이트됨!)
4. 과거 설교 검색 및 다운로드
```

---

### 시나리오 2: 교회 소식 및 공지

**사역자 (Obsidian)**:
```
1. "2025-02-15 청년부 수련회.md" 작성
2. 사진 첨부
3. 자동 배포 → 웹사이트 업데이트
```

**성도들 (웹사이트)**:
```
1. 스마트폰으로 접속
2. "행사" 메뉴에서 수련회 정보 확인
3. 신청 링크 클릭
```

---

### 시나리오 3: 교육 자료 및 강의

**교육부 (Obsidian)**:
```
1. "새가족반 교재.md" 작성
2. 주차별 강의 노트 추가
3. YouTube 영상 링크
4. 자동 배포
```

**새가족 (웹사이트)**:
```
1. 집에서 웹사이트 접속
2. 복습 자료 확인
```

## 방법 1: Quartz (추천!)

### 🌟 특징

- Obsidian 전용으로 설계
- Wiki-link `[[]]` 자동 변환
- Graph View 지원
- 검색 기능 내장
- GitHub Pages 무료 호스팅
- **자동 배포**: Git push → 웹사이트 업데이트

---

### Step 1: GitHub 계정 생성 (5분)

1. https://github.com 접속
2. "Sign up" 클릭
3. 이메일, 비밀번호 입력
4. 계정 생성 완료!

---

### Step 2: Quartz 템플릿 복제 (5분)

1. **템플릿 접속**
   - https://github.com/jackyzha0/quartz

2. **Fork (복제)**
   - 우측 상단 "Fork" 버튼 클릭
   - Repository 이름: `church-knowledge` (원하는 이름)
   - "Create fork" 클릭

3. **완료!** 이제 당신의 GitHub에 Quartz 복사본 생성됨

---

### Step 3: GitHub Desktop 설치 (10분)

**왜 필요한가?**
- Obsidian → GitHub → 웹사이트 자동 배포
- 버튼 클릭만으로 업로드

**설치**:

1. **다운로드**
   - https://desktop.github.com
   - Windows/Mac 버전 다운로드

2. **설치 및 로그인**
   - 설치 파일 실행
   - GitHub 계정으로 로그인

3. **Repository 클론**
   - File → Clone Repository
   - `church-knowledge` 선택
   - 저장 위치: `C:\Users\[사용자]\Documents\church-knowledge`
   - Clone!

---

### Step 4: Obsidian Vault와 연결 (5분)

**방법 1: 기존 Vault 복사** (추천)

```bash
# Windows
xcopy "C:\Users\[사용자]\Documents\우리교회" "C:\Users\[사용자]\Documents\church-knowledge\content" /E /I

# Mac
cp -r ~/Documents/우리교회/* ~/Documents/church-knowledge/content/
```

**방법 2: Symbolic Link** (고급)

```bash
# Windows (관리자 권한)
mklink /D "C:\Users\[사용자]\Documents\church-knowledge\content" "C:\Users\[사용자]\Documents\우리교회"

# Mac
ln -s ~/Documents/우리교회 ~/Documents/church-knowledge/content
```

**결과**: Obsidian에서 작성 → `content/` 폴더에 자동 저장

---

### Step 5: Quartz 설정 (5분)

**파일**: `church-knowledge/quartz.config.ts`

```typescript
const config: QuartzConfig = {
  configuration: {
    pageTitle: "우리교회 지식 베이스",
    enableSPA: true,
    enablePopovers: true,
    analytics: null,
    locale: "ko-KR",
    baseUrl: "yourchurch.github.io/church-knowledge",
    ignorePatterns: [
      "private",
      "templates",
      ".obsidian",
      "900.개인노트"  // 비공개 폴더
    ],
  },
  // ... 나머지 설정
}
```

**중요 설정**:
- `pageTitle`: 웹사이트 제목
- `locale`: 한국어 설정
- `ignorePatterns`: 공개하지 않을 폴더

---

### Step 6: GitHub Pages 활성화 (5분)

1. **GitHub 웹사이트 접속**
   - https://github.com/[사용자명]/church-knowledge

2. **Settings 메뉴**
   - 상단 "Settings" 클릭

3. **Pages 설정**
   - 좌측 "Pages" 클릭
   - Source: "GitHub Actions" 선택
   - Save

4. **완료!** 자동 배포 설정 완료

---

### Step 7: 첫 배포! (5분)

**GitHub Desktop에서**:

1. **변경 사항 확인**
   - 좌측에 변경된 파일 목록

2. **커밋 메시지 작성**
   - Summary: "첫 배포"
   - Description: "교회 지식 베이스 퍼블리싱"

3. **Commit to main**
   - 하단 "Commit to main" 버튼 클릭

4. **Push origin**
   - 상단 "Push origin" 버튼 클릭

5. **배포 시작!** 🚀
   - GitHub Actions가 자동으로 빌드
   - 5-10분 후 웹사이트 생성

---

### Step 8: 웹사이트 확인! (1분)

**주소**:
```
https://[사용자명].github.io/church-knowledge/
```

**예시**:
```
https://newhopeChurch.github.io/church-knowledge/
```

**확인 사항**:
- ✅ 주보 목록 보이나?
- ✅ 설교 노트 읽히나?
- ✅ 검색 작동하나?
- ✅ 링크 클릭 가능한가?

## 방법 2: Digital Garden

### 🌿 특징

- Obsidian 플러그인
- 버튼 클릭 하나로 퍼블리시
- Netlify 무료 호스팅
- 가장 쉬운 방법!

---

### Step 1: 플러그인 설치 (3분)

1. **Obsidian Settings**
   - Settings → Community plugins
   - "Digital Garden" 검색
   - Install → Enable

2. **플러그인 설정**
   - 좌측 "Digital Garden" 클릭
   - 설정 대기 (Step 2에서 계속)

---

### Step 2: GitHub 연동 (5분)

1. **GitHub Personal Access Token 생성**
   - GitHub 접속 → Settings
   - Developer settings → Personal access tokens
   - Generate new token (classic)
   - 권한 선택: `repo` 전체 체크
   - Generate token
   - **토큰 복사** (한 번만 보임!)

2. **Obsidian에 토큰 입력**
   - Digital Garden 설정
   - "GitHub Token" 필드에 붙여넣기

3. **Repository 이름**
   - "GitHub Repo Name": `church-garden`

4. **저장!**

---

### Step 3: Netlify 연동 (10분)

1. **Netlify 가입**
   - https://netlify.com
   - "Sign up" → GitHub 계정으로 가입

2. **Repository 연결**
   - "New site from Git"
   - GitHub 선택
   - `church-garden` 선택

3. **빌드 설정**
   - Build command: (자동 입력)
   - Publish directory: `dist`
   - Deploy site!

4. **도메인 확인**
   - `random-name-123.netlify.app`
   - Settings → Domain management → "Edit site name"
   - 원하는 이름: `newhopeChurch.netlify.app`

---

### Step 4: 퍼블리시! (1분)

**Obsidian에서**:

1. **노트 열기**
   - 퍼블리시할 주보 또는 설교 노트

2. **YAML 추가**
   ```yaml
   ---
   dg-publish: true
   ---
   ```

3. **Publish 버튼 클릭**
   - 명령 팔레트 (`Ctrl+P`)
   - "Digital Garden: Publish Single Note"
   - 또는 리본 아이콘 클릭 🌱

4. **완료!** 웹사이트에 즉시 반영

---


→ 🌱 버튼 클릭 → 웹사이트 업데이트!

## 자동 배포 설정

### GitHub Actions (Quartz/MkDocs)

**파일**: `.github/workflows/deploy.yml`

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v3

      - name: Build Quartz
        run: |
          npm install
          npx quartz build

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public
```

**효과**: `git push` → 자동 빌드 → 웹사이트 업데이트!

---

### Netlify (Digital Garden)

**설정**: `netlify.toml`

```toml
[build]
  command = "npm run build"
  publish = "dist"

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200
```

**효과**: GitHub push → Netlify 자동 빌드 → 즉시 배포!

## 권한 관리

### 공개/비공개 구분

#### 방법 1: 폴더로 구분 (Quartz)

**설정**: `quartz.config.ts`

```typescript
ignorePatterns: [
  "900.개인노트",      // 개인 노트 비공개
  "private",           // private 폴더 비공개
  "800.인사총무",      // 인사 정보 비공개
  "Templates"          // 템플릿 비공개
]
```

**결과**: 위 폴더는 웹사이트에 표시 안 됨!

---

#### 방법 2: YAML로 구분 (Digital Garden)

**공개 노트**:
```yaml
---
dg-publish: true   ← 웹사이트 공개
---
```

**비공개 노트**:
```yaml
---
dg-publish: false  ← 웹사이트 비공개
---
```

또는 YAML 없음 = 기본 비공개

### 첫 방문자 가이드 페이지

```markdown

title: 시작하기

# 🏛 우리교회 지식 베이스에 오신 것을 환영합니다!

## 📖 이 사이트는?

우리교회의 주보, 설교, 교육자료 등을 모아놓은 온라인 도서관입니다.

## 🚀 빠른 시작
```

## FAQ

### Q1: 정말 완전 무료인가요?

**A**: 네!

**무료 항목**:
- GitHub (무제한 public repository)
- GitHub Pages (무제한 트래픽)
- Netlify Free (월 100GB 대역폭)
- Quartz/Digital Garden (오픈소스)

**선택 비용**:
- 커스텀 도메인: $10/년 (예: church.com)
  - 없어도 됨! `.github.io`나 `.netlify.app` 무료

---

### Q2: 방문자 제한이 있나요?

**A**: 거의 없습니다.

**GitHub Pages**:
- 월 100GB 대역폭
- 월 10만 빌드
- → 중소형 교회 충분! (월 수만 명 가능)

**Netlify Free**:
- 월 100GB 대역폭
- 월 300빌드 분
- → 역시 충분!

---

### Q3: 백업은 어떻게 하나요?

**A**: 자동 백업됨!

**GitHub = 백업**:
- 모든 변경 이력 저장
- 언제든 이전 버전 복원 가능
- 서버 고장해도 안전

---

### Q4: 비밀번호를 걸 수 있나요?

**A**: 부분적으로 가능

**방법 1**: GitHub Private Repository
- Repository를 private으로 설정
- 초대받은 사람만 접근
- → 팀원만 편집 가능
- **단점**: 웹사이트도 비공개

**방법 2**: Netlify Identity (유료, $19/월)
- 로그인 기능 추가
- 회원별 권한 관리
- **추천 안 함**: 비용 발생

**방법 3**: 공개/비공개 분리 (추천)
- 공개 사이트: 주보, 공지
- 비공개 Obsidian: 심방, 상담

---

### Q5: 도메인을 바꾸고 싶어요 (church.com)

**A**: 가능합니다!

**Step 1**: 도메인 구입
- Namecheap, GoDaddy 등에서 구입
- 연 $10-15

**Step 2**: DNS 설정
```
A Record: 185.199.108.153
CNAME: yourname.github.io
```

**Step 3**: GitHub 설정
- Settings → Pages → Custom domain
- `www.ourchurch.com` 입력

**Step 4**: 완료!
- 1-24시간 후 적용

---

### Q6: 웹사이트 디자인을 바꿀 수 있나요?

**A**: 네!

**Quartz**:
- `quartz/styles/custom.scss` 파일 수정
- CSS 지식 필요

**예시**: 색상 변경
```scss
:root {
  --light: #ffffff;
  --dark: #1a1a1a;
  --primary: #2563eb;  // 교회 브랜드 색상
}
```

**템플릿**:
- Quartz 갤러리에서 다양한 테마 제공
- 복사해서 적용

---

### Q7: 사진이나 PDF도 올릴 수 있나요?

**A**: 가능합니다!

**사진**:
```markdown
![[교회전경.jpg]]
```

**PDF**:
```markdown
[주보 다운로드](주보.pdf)
```

**파일 위치**:
- `content/attachments/` 폴더에 저장
- 자동으로 웹사이트에 포함

---

### Q8: 접속 통계를 볼 수 있나요?

**A**: 네!

**Google Analytics (무료)**:

1. **Google Analytics 계정 생성**
   - https://analytics.google.com

2. **추적 ID 발급**
   - 예: `G-XXXXXXXXXX`

3. **Quartz 설정**
   ```typescript
   analytics: {
     provider: "google",
     tagId: "G-XXXXXXXXXX"
   }
   ```

4. **통계 확인**
   - 일일 방문자 수
   - 인기 페이지
   - 체류 시간

---

### Q9: 여러 명이 동시에 편집하면?

**A**: Git 충돌 발생 가능

**해결책**:
1. **시간대 분리**
   - 목회자: 오전
   - 사역자: 오후

2. **폴더 분리**
   - 목회자: `100.교회운영`
   - 청년부: `300.사역부서/청년부`

3. **GitHub Desktop 사용**
   - 충돌 자동 감지
   - 수동 병합

---

### Q10: 모바일 앱처럼 만들 수 있나요?

**A**: PWA로 가능!

**설정**: `quartz.config.ts`

```typescript
pwa: {
  manifest: {
    name: "우리교회",
    short_name: "교회",
    theme_color: "#2563eb",
    background_color: "#ffffff"
  }
}
```

**효과**:
- "홈 화면에 추가" 버튼
- 아이콘 생성
- 오프라인 지원 (선택)
