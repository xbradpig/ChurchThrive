## 템플릿 사용법

### 방법 1: 수동 복사

1. 아래에서 필요한 템플릿 **전체 복사** (`---`부터 끝까지)
2. 새 노트 생성 (`Ctrl+N`)
3. 붙여넣기 (`Ctrl+V`)
4. 노란색 `[채워넣기]` 부분만 수정
5. 저장 (`Ctrl+S`)

### 방법 2: Templater 플러그인 (추천)

1. `350.교회_템플릿_모음/` 폴더에 각 템플릿을 별도 파일로 저장
2. Templater 설정:
   - Settings → Templater → Template folder location
   - `350.교회_템플릿_모음/` 선택
3. 사용 시:
   - `Ctrl+P` → "Templater: Insert Template"
   - 템플릿 선택 → 자동 입력!

**주의**:
- 참석자에게 녹음 동의 받기
- 민감한 안건은 녹음 금지

---

### Tip 3: 템플릿 미리 준비

**방법**:
1. 회의 전날 템플릿 복사
2. 안건 미리 입력
3. 회의 중 결과만 추가

**효과**:
- 회의 시간 50% 단축
- 빠뜨리는 항목 없음

---

### Tip 4: 체크리스트 활용

회의록 완성도 체크:
- [ ] 참석자 명단 정확한가?
- [ ] 모든 안건 다루었나?
- [ ] 결정 사항 명확한가?
- [ ] 담당자와 기한 지정했나?
- [ ] 차기 회의 일정 잡았나?

---

### Tip 5: 48시간 내 승인

**프로세스**:
1. **회의 당일**: 초안 작성 및 저장
2. **+1일**: 검토 요청 (담임목사, 의장 등)
3. **+2일**: 최종 승인 및 배포

**지키면**:
- 기억이 선명할 때 수정
- 이견 발생 최소화

## FAQ

### Q1: 회의록을 꼭 작성해야 하나요?

**A**: 네, 특히 다음 회의는 필수입니다.

**필수 회의**:
- 운영위원회
- 제직회
- 당회
- 공동의회

**이유**:
1. 결정의 근거 (법적 효력)
2. 책임 소재 명확화
3. 히스토리 관리

**선택 회의**:
- 부서 모임 (권장)
- 비공식 미팅 (선택)

---

### Q2: 누가 작성하나요?

**A**: 회의 종류별로 다릅니다.

| 회의 | 작성자 |
|------|--------|
| 운영위원회 | 서기 |
| 제직회 | 서기 |
| 당회 | 서기장로 |
| 공동의회 | 서기 |
| 부서 회의 | 총무 또는 부서 서기 |

---

### Q3: 녹취록 형식으로 써야 하나요?

**A**: 아니요, 요약 형식이 더 좋습니다.

❌ **녹취록 형식** (비추천):
```
담임목사: 오늘 크리스마스 행사에 대해 논의하겠습니다.
장로1: 네, 좋습니다. 저는...
(대화 전부 기록)
```

✅ **요약 형식** (추천):
```
### 안건 1: 크리스마스 행사

**논의 내용**:
- 일정: 12월 25일로 제안됨 (장로1)
- 예산: 300만원 적정 (안수집사1)

**결정 사항**:
- 12월 25일 확정
- 예산 300만원 승인
```

---

### Q4: 민감한 내용도 기록해야 하나요?

**A**: 회의록 보안 등급을 나누세요.

**공개 회의록**:
- 일반 안건
- 사업 계획
- 예산안

**기밀 회의록** (🔒):
- 인사 문제
- 권징 사안
- 민감한 재정

**처리 방법**:
```yaml
tags: [회의록, 당회, 🔒기밀]
security: confidential
```

Obsidian에서 기밀 태그로 검색 제한

---

### Q5: 회의록 승인은 어떻게 받나요?

**A**: 3단계 프로세스:

**Step 1: 초안 작성**
- 서기가 회의 당일 또는 익일 작성

**Step 2: 검토 요청**
- 의장(담임목사)에게 검토 요청
- 이메일 또는 카톡으로 Obsidian 파일 공유

**Step 3: 승인**
- 의장 확인 후 status를 `approved`로 변경
- YAML에 승인일 기록

```yaml
status: approved
approved_by: "[[담임목사]]"
approved_date: 2025-11-16
```

---

### Q6: 과거 회의록을 어떻게 관리하나요?

**A**: 연도별 폴더 구조 추천:

```
100.교회운영/130.회의록/
├── 운영위원회/
│   ├── 2025/
│   │   ├── 2025-01-14 운영위원회.md
│   │   ├── 2025-02-14 운영위원회.md
│   │   └── ...
│   └── 2024/
│       └── ...
├── 제직회/
└── 당회/
```

**검색 방법**:
```dataview
LIST
FROM "100.교회운영/130.회의록"
WHERE type = "meeting"
  AND meeting_type = "운영위원회"
  AND date >= date("2025-01-01")
SORT date DESC
```

### Q8: 결석자 의견은 어떻게 반영하나요?

**A**: 사전 의견 수렴 섹션 추가:

```markdown
## 👥 참석자

### 참석
- [[담임목사]], [[장로1]], [[장로2]]

### 결석 (사전 의견 제출)
- **[[장로3]]** (출장):
  > "안건 1에 대해 찬성합니다. 예산은 적정하다고 봅니다."

### 결석 (의견 없음)
- [[안수집사1]]
```

### Q9: 온라인 회의는 어떻게 기록하나요?

**A**: 추가 정보를 YAML에 포함:

```yaml
meeting_format: online
meeting_platform: ZOOM
meeting_link: "https://zoom.us/j/123456789"
recording: true
recording_link: "[[ZOOM 녹화본]]"
```

본문:
```markdown
## 📋 회의 정보

- **방식**: 온라인 (ZOOM)
- **참석**: 8명 (온라인 6명, 현장 2명)
- **녹화**: 예 (링크: [[ZOOM 녹화본]])
```

### Q10: 회의록을 PDF로 출력하려면?

**A**: 3가지 방법:

**방법 1: Obsidian 기본 기능**
1. Reading View로 전환
2. `Ctrl+P` (Print)
3. "PDF로 저장" 선택

**방법 2: Pandoc 플러그인**
1. Community Plugins → "Pandoc" 설치
2. `Ctrl+P` → "Pandoc: Export to PDF"

**방법 3: 온라인 변환기**
1. 마크다운 복사
2. https://www.markdowntopdf.com/ 접속
3. 붙여넣기 → 변환
