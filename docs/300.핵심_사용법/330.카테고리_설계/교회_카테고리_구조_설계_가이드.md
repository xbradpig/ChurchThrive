## 카테고리 설계 원칙

### 원칙 1: MECE (Mutually Exclusive, Collectively Exhaustive)

**서로 배타적이고, 전체를 포괄**

✅ **좋은 예시**:
```
100.교회운영/
200.성도관리/
300.교육사역/
400.선교전도/
```
→ 중복 없이 모든 영역 커버

❌ **나쁜 예시**:
```
100.행정/
200.회의록/    ← 행정에 포함될 수 있음 (중복)
300.재정/      ← 행정에 포함될 수 있음 (중복)
```
→ 카테고리 간 경계 모호

---

### 원칙 2: 7±2 법칙 (최상위 카테고리는 5-9개)

**인간의 단기 기억은 7±2개 항목**

✅ **적절한 수** (5-9개):
```
100.교회운영
200.성도관리
300.교육사역
400.선교전도
500.예배사역
600.시설관리
```
→ 한눈에 파악 가능

❌ **너무 많음** (15개+):
```
100.주보
200.설교
300.회의록
400.재정
500.성도
600.교육
700.청년
800.찬양
900.시설
1000.전도
1100.선교
...
```
→ 압도되어 사용 안 함

---

### 원칙 3: 깊이보다 넓이 (3단계 이내)

**폴더 깊이는 최대 3단계**

✅ **3단계** (권장):
```
100.교회운영/
  110.주보/
    111.2025년/
```

❌ **5단계** (비추천):
```
100.교회운영/
  110.문서/
    111.주보/
      1111.2025년/
        11111.1월/
```
→ 너무 깊어서 찾기 어려움

---

### 원칙 4: 번호 체계 활용

**숫자 접두사로 순서 고정**

✅ **번호 사용**:
```
100.교회운영
200.성도관리
300.교육사역
```
→ 알파벳 순서와 무관하게 원하는 순서 유지

❌ **번호 없음**:
```
교회운영
성도관리
교육사역
```
→ 알파벳 순서로 자동 정렬 (ㄱ, ㅅ, ㅅ)

---

### 원칙 5: 행동 중심 vs 명사 중심

**목적에 따라 선택**

**행동 중심** (업무 중심):
```
100.계획하기
200.실행하기
300.점검하기
```

**명사 중심** (영역 중심) - **교회에 적합**:
```
100.교회운영
200.성도관리
300.교육사역
```

**교회는 명사 중심이 더 적합합니다.**

## 교회 규모별 전략

### 소형 교회 (출석 50-100명)

**특징**:
- 담임목사 1명
- 사역자 없거나 1-2명
- 부서 수 적음 (3-5개)
- 문서량 적음

**전략**:
```
✅ 간단한 구조 (5개 카테고리)
✅ 2단계 깊이
✅ 통합 관리 (주보+설교 한 폴더)
```

**권장 구조**:
```
100.교회운영/
  110.주보_설교/
  120.회의록/
  130.재정/

200.성도관리/
  210.출석부/
  220.심방기록/

300.행사/

400.문서보관/

500.개인노트/

## 교회별 맞춤 설계

### Case 1: 개척 교회 (출석 20-50명)

**특징**: 초기 단계, 문서 적음, 간소화 필요

**추천 구조** (3단계만):
```
100.교회운영/
  110.주보_설교/
  120.회의록/

200.성도관리/

300.행사/
```

**성장 전략**:
1. 출석 50명 도달 → `110.주보_설교`를 분리
2. 출석 100명 도달 → 중형 구조로 확장

---

### Case 2: 다민족 교회 (한국어 + 영어)

**특징**: 언어별 분리 필요

**추천 구조**:
```

  120.설교/

  110.Bulletin/
  120.Sermon/

```

**또는 통합 관리**:
```
100.교회운영/
  110.주보/
    111.한국어/
    112.English/
```

### Case 4: 온라인 중심 교회

**특징**: 온라인 콘텐츠 중심

**추천 구조**:
```

## 폴더 네이밍 규칙

### 규칙 1: 번호 체계

**3단계 번호 규칙**:
```
100, 200, 300...        (최상위, +100)
110, 120, 130...        (중위, +10)
111, 112, 113...        (하위, +1)
```

**예시**:
```
100.교회운영/
  110.주보/
    111.2025년/
    112.2024년/

    121.주일설교/
    122.수요예배/
```

---

### 규칙 2: 한글 vs 영문

**한글 권장** (교회는 한국어 사용자 대부분):
```
✅ 100.교회운영
✅ 110.주보
✅ 120.설교
```

**영문은 선택사항**:
```
⚠️ 100.Church_Operations
⚠️ 110.Bulletin
⚠️ 120.Sermon
```

**혼용 가능**:
```
✅ 100.교회운영_Operations
```

---

### 규칙 3: 띄어쓰기 vs 언더스코어

**언더스코어(_) 권장**:
```
✅ 100.교회운영
✅ 110.주보_설교
✅ 120.회의록_운영위원회
```

**띄어쓰기 가능하지만 비추천**:
```
⚠️ 100.교회 운영
⚠️ 110.주보 설교
```

**이유**: 일부 시스템에서 띄어쓰기 문제 발생 가능

❌ * (별표)
❌ ? (물음표)
❌ " (따옴표)

❌ | (파이프)

**사용 가능**:
```

```

## 카테고리 확장 전략

### 단계 1: 최소 구조로 시작 (첫 3개월)

```
100.교회운영/
200.성도관리/
300.행사/
```

**왜?**
- 복잡한 구조는 압도됨
- 실제 필요를 파악하는 시간 필요
- 언제든 확장 가능

**추가 기준**:
1. **문서 수 20개 이상** → 분리 고려
2. **매주 사용** → 독립 카테고리
3. **담당자 명확** → 부서별 분리

**예시**:
```
# 초기
100.교회운영/
  110.주보_설교/    ← 20개 문서

# 확장
100.교회운영/
  110.주보/         ← 분리!
  120.설교/         ← 분리!
```

## 마이그레이션 가이드

### 시나리오 1: 기존 폴더 → Obsidian

**기존 파일 구조**:
```
내 문서/
  교회/
    주보 2025.docx
    설교 모음.docx
    회의록.docx
```

**이동 프로세스**:
1. **Obsidian Vault 생성**
2. **폴더 구조 생성** (위 가이드 따라)
3. **파일 이동**:
   ```bash
   # 주보 이동
   cp "주보 2025.docx" "100.교회운영/110.주보/"

   # Markdown 변환 (Pandoc 사용)
   pandoc "주보 2025.docx" -o "2025-01-05 주보.md"
   ```
4. **YAML 추가** (각 파일에)
5. **링크 재연결**

### 시나리오 2: 기존 Obsidian Vault 재구성

**기존 구조**:
```
주보/

회의록/

```

**재구성 프로세스**:

2. **파일 복사 (이동 아님)**
3. **2주 테스트**
4. **문제없으면 기존 폴더 삭제**

**안전 팁**:
```bash
# 백업 먼저!
cp -r vault vault_backup_2025-11-14
```

### 시나리오 3: 소형 → 중형 확장

### Q10: 카테고리가 너무 많아 보여요. 숨길 수 있나요?

**A**: 폴더 접기 (Collapse) 사용

**방법**:
1. 폴더명 옆 화살표 클릭
2. 폴더가 접힘 ✅

**팁**:
- 자주 쓰는 폴더만 펼침
- 나머지는 접어둠
- 필요할 때만 펼침
