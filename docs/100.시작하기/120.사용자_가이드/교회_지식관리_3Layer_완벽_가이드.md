## 1. 왜 교회에 3-Layer 시스템이 필요한가?

### 🎯 교회가 직면한 지식관리 문제들

#### 문제 1: 정보가 흩어져 있습니다

**현실**:
```
주보: 한글 파일
설교 노트: 워드 문서
회의록: 이메일
심방 기록: 수첩
행사 기획: 카카오톡 대화
재정 보고: 엑셀
```

**결과**:
- ❌ 필요한 자료를 찾는데 10분 이상 소요
- ❌ "작년에 비슷한 행사 했는데 어떻게 했지?" → 기억에 의존
- ❌ 담당자가 바뀌면 모든 노하우가 사라짐
- ❌ 같은 질문을 반복해서 받고, 같은 답변을 반복

#### 문제 2: AI 도구를 활용하기 어렵습니다

**시나리오**:
```
목회자: "ChatGPT야, 우리 교회 작년 부활절 행사 어떻게 했어?"
ChatGPT: "죄송하지만 귀 교회의 자료에 접근할 수 없습니다..."
```

**왜 그럴까요?**
- AI는 파편화된 파일들을 읽을 수 없음
- 구조가 없으면 AI가 맥락을 파악할 수 없음
- 파일만 많아도 AI가 무엇을 읽어야 할지 모름

#### 문제 3: 팀 협업이 어렵습니다

**소형교회 (50-100명)**:
```
담임목사: "이번 주일 주보 어디 있어?"
교육전도사: "제 컴퓨터에 있어요..."
→ 한 사람만 모르면 업무 마비
```

**중형교회 (100-300명)**:
```
행정직원: "작년 재정 보고서 양식 어디 있죠?"
회계담당: "퇴사한 OO 집사님이 관리하셨는데..."
→ 인수인계 실패
```

**대형교회 (300명+)**:
```
각 부서가 각자의 방식으로 문서 관리
→ 통합된 보고가 불가능
```

### ✅ 3-Layer 시스템이 해결하는 것들

#### 해결 1: 모든 정보를 한곳에

```
Obsidian Vault (하나의 폴더)
├── 주보 (매주 자동 생성)
├── 설교 노트 (검색 가능)
├── 회의록 (링크로 연결)
├── 심방 기록 (태그로 분류)
├── 행사 기획 (과거 기록 참조)
└── 재정 보고 (통계 자동 생성)

→ 모든 것이 "하나의 진실 출처"에 있음
```

**효과**:
- ✅ 5초만에 원하는 자료 찾기
- ✅ "작년 자료"를 즉시 참조
- ✅ 담당자가 바뀌어도 모든 기록 유지
- ✅ AI가 과거 기록을 학습하여 제안

#### 해결 2: AI와 완벽한 협업

**3-Layer 적용 후**:
```
목회자: "Claude야, 우리 교회 작년 부활절 행사 어떻게 했어?"

AI (3단계로 응답):
1. Layer 0 읽기: "교회 구조 파악" (0.1초)
2. Layer 1 읽기: "행사 카테고리 확인" (0.5초)
3. Layer 2 검색: "2024년 부활절 행사 기획서" 발견 (1초)

AI 응답:
"2024년 부활절 행사는 다음과 같이 진행되었습니다:
- 일시: 2024년 3월 31일 (부활절 주일)
- 예산: 300만원
- 프로그램: 새벽 예배 + 계란 나눔 + 오찬
- 참석: 250명
- 만족도: 4.8/5.0
- 개선사항: 주차 공간 부족

첨부: [[2024년_부활절_행사_기획서]]"
```

**효과**:
- ✅ AI가 교회의 모든 역사를 "기억"
- ✅ 과거 경험에서 배워서 제안
- ✅ 실수 방지 ("작년에 이 실수 했었어요")
- ✅ 업무 속도 10배 향상

#### 해결 3: 완벽한 팀 협업

**3-Layer 시스템 적용**:
```
모든 팀원이 같은 Vault 공유 (클라우드 동기화)

담임목사 (PC)  ←→  Obsidian Vault  ←→  교육전도사 (노트북)
                         ↕
                   행정직원 (PC)
```

**워크플로우**:
1. 행정직원이 주보 작성
2. 교육전도사가 교육부 공지 추가
3. 담임목사가 최종 검토
4. 모든 변경사항 실시간 동기화

**효과**:
- ✅ 누가 어디서나 최신 버전 접근
- ✅ "내 컴퓨터에만 있어요" 문제 해결
- ✅ 인수인계 완벽 (모든 기록 보존)
- ✅ 병가/휴가 중에도 업무 연속성

## 2. 3-Layer 시스템이란?

### 🏗️ 핵심 개념: Progressive Disclosure

> **정보를 계층적으로 제공하여, 필요한 것만 빠르게 찾는다**

**비유**:
```
도서관에 가면...
❌ 나쁜 도서관: 모든 책이 바닥에 쌓여있음
✅ 좋은 도서관: 층별 → 서가별 → 책

3-Layer도 마찬가지:
❌ 나쁜 방식: 모든 문서를 폴더에 무작위로 저장
✅ 3-Layer: 계층별로 구조화하여 즉시 찾기
```

### 📐 3개 레이어 구조

```
┌─────────────────────────────────────────────────────────────┐
│ Layer 0: CLAUDE.md (항상 로드)                               │
│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ │
│ 크기: ~40KB 이하                                             │
│ 역할: 핵심 지침 + 빠른 네비게이션                            │
│ 내용:                                                        │
│ • 교회 개요 (규모, 특성)                                     │
│ • 폴더 구조 (어디에 무엇이 있는가)                           │
│ • YAML 표준 (문서 작성 규칙)                                 │
│ • 빠른 의사결정 트리                                         │
│                                                              │
│ 👉 AI가 "교회 전체 구조"를 0.1초에 파악                     │
└─────────────────────────────────────────────────────────────┘
                    ↓ 링크 (필요 시 로드)
┌─────────────────────────────────────────────────────────────┐
│ Layer 1: 빠른 가이드 (AI가 필요 시 로드)                     │
│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ │
│ 크기: ~200KB (10-15개 문서)                                  │
│ 역할: 카테고리별 가이드 + 템플릿                             │
│ 내용:                                                        │
│ • 주보 작성 가이드 (템플릿 + 예시)                           │
│ • 회의록 작성 가이드                                         │
│ • 설교 노트 템플릿                                           │
│ • 행사 기획 체크리스트                                       │
│ • 재정 보고 양식                                             │
│                                                              │
│ 👉 AI가 "작업 방법"을 0.5초에 학습                          │
└─────────────────────────────────────────────────────────────┘
                    ↓ 링크 (구체적 내용)
┌─────────────────────────────────────────────────────────────┐
│ Layer 2: 실제 문서들 (선택적 로드)                           │
│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ │
│ 크기: 무제한 (수백~수천 개 문서)                             │
│ 역할: 실제 교회 운영 문서                                    │
│ 내용:                                                        │
│ • 52주치 주보 (매주 추가)                                    │
│ • 100+ 설교 노트                                             │
│ • 50+ 회의록                                                 │
│ • 30+ 행사 기록                                              │
│ • 1,000+ 심방 기록                                           │
│                                                              │
│ 👉 AI가 "필요한 문서만" 선택적으로 로드 (2초)               │
└─────────────────────────────────────────────────────────────┘
```

### 💡 왜 3개 레이어인가?

**AI의 기술적 한계**:
```
Claude/ChatGPT: 약 600KB 텍스트만 한 번에 읽기 가능
일반적인 교회 문서: 2년치 = 약 10MB

→ 전체의 6%만 읽을 수 있음 ❌
→ 나머지 94%는 "보이지 않는" 지식
```

**3-Layer 해결책**:
```
Layer 0 (40KB) + Layer 1 (200KB) = 240KB
→ AI가 교회 전체 구조 + 업무 방법을 완벽히 파악 ✅
→ 필요한 문서만 Layer 2에서 가져옴 (효율 99%+)
```

## 3. 교회 맞춤 설계

### 🏛️ 교회 규모별 추천 구조

#### 소형교회 (50-100명)

**특징**:
- 적은 인력 (목회자 1-2명 + 자원봉사자)
- 예산 제한
- 간단한 업무 흐름

**추천 구조**:
```
100.교회운영/
├── 110.주보/                    # 매주 주보
├── 120.설교/                    # 설교 노트
├── 130.회의록/                  # 운영위원회 회의록
├── 140.행사/                    # 특별 행사 기획
├── 150.교육/                    # 주일학교, 제자훈련
├── 160.심방/                    # 심방 기록
├── 170.새가족/                  # 새가족 관리
├── 180.재정/                    # 재정 보고
└── 190.자료실/                  # 각종 양식

→ 9개 카테고리, 심플하게 시작
```

**예상 효과**:
- ✅ 1시간만에 구축 완료
- ✅ 주당 5시간 시간 절약
- ✅ 무료 도구만 사용 (Obsidian 무료, AI 무료 버전)

**특징**:

│   └── 123.심방/

## 4. 구축 로드맵

### 🚀 4단계 구축 프로세스

#### Phase 1: 기초 (1-2시간) - 오늘 완료!

**목표**: Obsidian 설치 + 첫 주보 작성

**단계**:
1. **Obsidian 설치** (10분)
   - [[320.Obsidian_설치_및_설정]] 가이드 참조
   - Windows/Mac 다운로드
   - Vault 생성

2. **기본 폴더 구조 생성** (15분)
   ```
   100.교회운영/
   ├── 110.주보/
   ├── 120.설교/
   └── 130.회의록/
   ```

3. **CLAUDE.md 작성** (20분)
   - 교회 이름, 규모 기록
   - 폴더 구조 명시
   - [[YAML_프론트매터_표준]] 복사

4. **첫 주보 작성** (30분)
   - [[주보_템플릿]] 사용
   - YAML 추가
   - 저장 및 확인

**교육 프로그램** (팀원별 1시간):
1. Obsidian 기초 (15분)
   - 설치, Vault 공유 설정
   - 기본 조작법

2. 실전 템플릿 사용 (20분)
   - 주보 템플릿
   - 회의록 템플릿
   - 본인 업무 템플릿 찾기

3. AI 활용 (15분)
   - Claude에게 질문하는 방법
   - 좋은 프롬프트 예시

4. 실습 (10분)
   - 직접 문서 1개 작성
   - AI에게 검색 요청

#### Step 4: 고급 기능 (한 달 후)

**마스터하면 좋은 기능들**:

1. **Dataview 쿼리** (통계 자동화)

   - 출석 통계 자동 생성
   - 헌금 추이 그래프

   - 설교 주제 간 연결
   - 성도 관계망

3. **대시보드** (한 눈에 보기)
   - 이번 달 일정
   - 미완료 작업
   - 최근 문서

```
❌ 나쁜 시작: "모든 과거 문서를 Obsidian으로 옮기자!"
              → 1주일 걸리고 포기

              → 1시간만에 시작
              → 점진적 확장
```

#### Tip 2: 완벽하지 않아도 괜찮습니다
```
처음엔 YAML이 빠질 수도 있고,
링크를 안 만들 수도 있고,
구조가 엉망일 수도 있습니다.

→ 계속 사용하다 보면 자연스럽게 개선됩니다.
→ AI가 도와줍니다.
→ 3개월 후엔 전문가가 되어 있을 겁니다.
```

```
혼자 하면: 외롭고, 느리고, 포기하기 쉬움
팀과 함께: 재미있고, 빠르고, 동기부여 됨

- 어려운 점 함께 해결
```
