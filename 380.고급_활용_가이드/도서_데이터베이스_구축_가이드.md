## 왜 도서 데이터베이스가 필요한가?

### 목회자를 위해

**Before (도서 DB 없이)**:
```
❌ "작년에 읽은 그 책 제목이 뭐였지?"
❌ "설교 준비할 때 본 주석이 어디 있지?"
❌ "이 책 누구한테 빌려줬더라?"
```

**After (도서 DB 사용)**:
```
✅ 검색: "칭의" → 즉시 관련 서적 목록
✅ 설교 노트에 참고 서적 링크
✅ 대출 현황 한눈에 파악
```

## Book Search 플러그인 설치

### Step 1: 플러그인 설치

1. **Settings** (⚙️) → **Community plugins**
2. **Browse** 클릭
3. 검색: `Book Search`
4. **Install** → **Enable**

---

### Step 2: API 설정 (Google Books)

#### Google Books API 키 발급 (무료)

1. **Google Cloud Console** 접속
   - https://console.cloud.google.com/

2. **프로젝트 생성**
   - 프로젝트 이름: "교회도서관"

3. **API 라이브러리**
   - "Books API" 검색
   - "사용 설정" 클릭

4. **API 키 생성**
   - 사용자 인증 정보 → API 키 만들기
   - API 키 복사

5. **Obsidian 설정**
   - Settings → Book Search
   - "Google Books API Key" 붙여넣기

**제한**: 일 1,000회 (충분함)

---

### Step 3: 기본 설정

**Settings → Book Search**:

```yaml
Default folder: 700.도서관리/710.책
Template file: Templates/책_템플릿.md
New file name format: {{title}} - {{author}}
```

## 도서 추가 방법

### 방법 1: ISBN으로 추가 (가장 정확!)

**Step-by-Step**:

1. **명령 실행**
   - `Ctrl+P` → "Book Search: Create new book note"

2. **ISBN 입력**
   ```
   ISBN: 9788936501082
   ```
   (책 뒷표지 바코드 아래 13자리 숫자)

3. **자동 완성!** ✨
   - 제목, 저자, 출판사, 페이지, 썸네일 자동 입력

---

### 방법 2: 제목으로 검색

1. `Ctrl+P` → "Book Search: Create new book note"
2. 제목 입력: `칭의란 무엇인가`
3. 검색 결과 리스트에서 선택
4. 자동 완성!

## 📝 독서 노트

### 읽게 된 계기
- 로마서 3-5장 강해 설교 준비 중
- 칭의 교리에 대한 깊이 있는 이해 필요

### 핵심 내용

#### 1장: 칭의 교리의 역사
- 어거스틴부터 현대까지
- 루터와 종교개혁의 중심 교리

#### 2장: 성경적 기초
- 로마서 3:21-26 핵심 구절
- 칭의 = 법정적 선언 (forensic declaration)

#### 3장: 칭의와 성화
- 칭의: 신분의 변화 (즉시)
- 성화: 상태의 변화 (점진)

### 인상 깊은 구절

> "칭의는 하나님께서 죄인을 의롭다고 '선언'하시는 것이지,
> 의롭게 '만드시는' 것이 아니다." (p.87)

> "칭의의 근거는 오직 그리스도의 의이며,
> 우리는 믿음으로 이를 받는다." (p.145)

### 나의 생각

- 칭의와 성화를 명확히 구분하면서도 연결하는 설명이 탁월
- 역사적, 성경적, 체계적 접근이 균형잡힘
- 개신교 칭의론을 변호하면서 가톨릭과의 차이 명확히 함

### 설교 적용 아이디어

**적용 설교 계획**:
- 로마서 3:21-26 강해 (2주)
- "칭의: 하나님의 선언" 시리즈 (4주)

**핵심 메시지**:
1. 우리는 행위가 아닌 믿음으로 의롭게 됨
2. 칭의는 일회적, 성화는 평생
3. 칭의의 확신이 성화의 동력

**예화**:
- 법정 판결 예화 (p.92 참조)
- 은행 계좌 이체 예화 (p.156 참조)

## 📖 대출 현황

### 현재 대출 중

```dataview
TABLE
  title AS "책 제목",
  borrower AS "대출자",
  borrow_date AS "대출일",
  return_due AS "반납 예정"
FROM "700.도서관리"
WHERE type = "book"
  AND status = "대출중"
SORT return_due ASC
```

### 연체 도서

```dataview
TABLE
  title AS "책 제목",
  borrower AS "대출자",
  return_due AS "반납 예정",
  (date(today) - return_due) AS "연체일수"
FROM "700.도서관리"
WHERE type = "book"
  AND status = "대출중"
  AND return_due < date(today)
SORT return_due ASC
```

## FAQ

### Q1: Book Search 플러그인 없이도 가능한가요?

**A**: 가능합니다!

**수동 방법**:
1. 템플릿 만들기
2. 직접 정보 입력
3. YAML frontmatter만 잘 작성하면 됨

**장점**: 플러그인 의존도 ↓
**단점**: 정보 입력 시간 ↑

---

### Q2: ISBN이 없는 오래된 책은?

**A**: 제목으로 검색하거나 수동 입력

**대안**:
1. 제목 + 저자로 Google Books 검색
2. 수동으로 정보 입력
3. 썸네일은 사진 찍어서 추가

---

### Q3: 교회 도서관 바코드 시스템과 연동되나요?

**A**: 직접 연동은 안 되지만, 병행 가능

**방법**:
1. 바코드 시스템: 대출/반납 처리
2. Obsidian: 도서 정보 + 독서 노트
3. 주기적으로 동기화

---

### Q4: 여러 사람이 함께 사용하려면?

**A**: 클라우드 동기화 + 권한 관리

**설정**:
1. OneDrive/Google Drive에 Vault
2. 팀원 초대
3. 동시 편집 주의 (시간대 분리)

---

### Q5: 모바일에서도 되나요?

**A**: 네!

**방법**:
1. Obsidian 모바일 앱 설치
2. 동일한 Vault 동기화
3. Book Search 플러그인 활성화

**활용**:
- 도서관에서 바로 책 추가
- 이동 중 독서 노트 작성

---

### Q6: 전자책(e-book)도 관리할 수 있나요?

**A**: 가능합니다!

```yaml
---
type: book
format: ebook
ebook_platform: "Kindle"
purchase_link: "https://..."
---
```

---

### Q7: 추천 도서 목록을 성도들과 공유하려면?

**A**: 3가지 방법

**방법 1: PDF 변환**
- Obsidian → Print → PDF
- 주보에 첨부

**방법 2: 웹 퍼블리싱**
- Obsidian Publish ($10/월)
- 교회 웹사이트에 링크

**방법 3: 복사-붙여넣기**
- 마크다운 복사
- 이메일/카톡으로 전송

---

### Q8: 서평을 블로그에도 올리고 싶어요

**A**: Obsidian에서 작성 → 복사 → 블로그 붙여넣기

**장점**:
- Obsidian = 원본 보관소
- 블로그 = 공개 버전
- 이미지 함께 복사 가능

---
