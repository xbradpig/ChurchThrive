## 왜 설교 피드백 시스템인가?

### 현재 문제점

**전통적 방법**:
```
❌ 설교 후 구두 질문 → 시간 부족
❌ 종이 설문지 → 집계 어려움, 분실 위험
❌ 카카오톡 개별 메시지 → 산발적, 관리 어려움
❌ 피드백 없음 → 성도 반응 모름
```

---

### 시스템의 장점

**✅ 성도 입장**:
- 편하게 스마트폰으로 응답 (1-2분)
- 익명 가능 (솔직한 피드백)
- 개인 질문도 가능
- 언제든 접근 가능

**✅ 목회자 입장**:
- 모든 피드백 자동 수집
- 성도별 히스토리 관리
- 설교 개선점 발견
- 목회 방향 설정
- 심방 대상 파악 (기도제목)

**✅ 교회 입장**:
- 성도 참여도 향상
- 소통 활성화
- 데이터 기반 목회
- 투명한 운영

## 방법 1: Google Forms (가장 쉬움)

### 🌟 장점

- ✅ **완전 무료**
- ✅ **설정 5분**
- ✅ **자동 Google Sheets 연동**
- ✅ **모바일 최적화**
- ✅ **익명 응답 가능**
- ✅ **다양한 질문 유형**

---

### Step 1: Google Forms 생성 (10분)

#### 1-1. 폼 생성

1. **Google Forms 접속**
   - https://forms.google.com
   - "새 양식" 클릭

2. **제목 설정**
   ```
   제목: 설교 피드백 (2025년 1월 19일)
   설명: 오늘 설교에 대한 여러분의 소중한 의견을 나눠주세요.
   ```

---

#### 1-2. 질문 구성

**질문 1: 이름 (선택)**
```yaml
질문 유형: 단답형
질문: 이름 (익명 원하시면 비워두세요)
필수: 아니요
```

**질문 2: 설교 만족도**
```yaml
질문 유형: 객관식 질문
질문: 오늘 설교가 도움이 되셨나요?

선택지:
  ⭐⭐⭐⭐⭐ 매우 은혜로웠습니다
  ⭐⭐⭐⭐ 도움이 되었습니다
  ⭐⭐⭐ 보통이었습니다
  ⭐⭐ 아쉬웠습니다
  ⭐ 개선이 필요합니다

필수: 예
```

**질문 3: 가장 기억에 남는 내용**
```yaml
질문 유형: 단답형
질문: 설교에서 가장 기억에 남는 내용이나 구절은?

예시 답변: "하나님의 사랑은 조건 없다는 말씀"

필수: 아니요
```

**질문 4: 설교 소감**
```yaml
질문 유형: 장문형
질문: 오늘 설교를 듣고 느낀 점이나 깨달은 점을 자유롭게 나눠주세요.

필수: 아니요
```

**질문 5: 질문/궁금한 점**
```yaml
질문 유형: 장문형
질문: 설교나 성경 본문에 대해 궁금한 점이 있으신가요?

설명: "목사님께서 개별적으로 답변해드립니다."

필수: 아니요
```

**질문 6: 개인 기도제목**
```yaml
질문 유형: 장문형
질문: 함께 기도하고 싶은 개인 기도제목이 있으신가요?

설명: "비공개로 처리되며, 목사님만 확인합니다."

필수: 아니요
```

**질문 7: 심방 요청**
```yaml
질문 유형: 객관식 질문
질문: 목사님의 심방을 원하시나요?

선택지:
  예, 심방을 원합니다
  아니요, 괜찮습니다

필수: 아니요
```

**질문 8: 연락처 (심방 요청 시)**
```yaml
질문 유형: 단답형
질문: 연락처 (심방 요청하신 경우만 입력)

필수: 아니요
```

---

#### 1-3. 설정

**설정 → 응답**:
```yaml
✅ 응답 1회로 제한: 아니요 (매주 응답 가능)
✅ 응답 편집 허용: 예
✅ 요약 차트 및 텍스트 응답 보기: 아니요 (프라이버시)
✅ 응답자에게 영수증 사본 전송: 선택 사항
```

**설정 → 프레젠테이션**:
```yaml
✅ 진행률 표시줄 표시: 예
✅ 질문 순서 섞기: 아니요
✅ 확인 메시지:
   "피드백 감사합니다! 여러분의 의견은 설교 준비에 큰 도움이 됩니다. 🙏"
```

---

### Step 2: 폼 공유 (3분)

#### 2-1. 링크 생성

```
우측 상단 "보내기" 클릭
→ 링크 탭 선택
→ "URL 단축" ✅
→ 링크 복사

예시: https://forms.gle/ABC123xyz
```

---

#### 2-2. QR 코드 생성

**QR 코드 생성기 사용**:
- https://www.qr-code-generator.com
- 링크 입력 → QR 코드 다운로드

**주보에 삽입**:
```markdown
## 📝 설교 피드백

오늘 설교에 대한 여러분의 의견을 들려주세요!

[QR 코드 이미지]

또는 링크: https://forms.gle/ABC123xyz
```

---

#### 2-3. 카카오톡 공유

**단체방에 메시지**:
```
📢 오늘 설교 피드백 부탁드립니다!

오늘 말씀이 어떠셨나요?
1-2분이면 완료됩니다 😊

👉 https://forms.gle/ABC123xyz

여러분의 의견은 다음 설교 준비에
큰 도움이 됩니다. 🙏

감사합니다!
```

---

### Step 3: 응답 수집 (자동)

**Google Sheets 자동 생성**:
```
Forms → 응답 탭
→ "스프레드시트에 연결" (녹색 아이콘)
→ "새 스프레드시트 만들기"
→ 이름: "설교 피드백 응답"
```

**결과**:
- 응답이 실시간으로 Google Sheets에 추가됨
- 타임스탬프, 모든 응답 자동 기록

## 방법 2: Notion Form (고급)

### 🌟 장점

- ✅ Notion 데이터베이스 직접 연동
- ✅ 예쁜 인터페이스
- ✅ 데이터베이스 뷰 활용 (캘린더, 갤러리)
- ✅ 팀원들과 바로 협업

---

### Step 1: Notion 데이터베이스 생성

1. **Notion 페이지 생성**
   ```
   페이지명: 설교 피드백 데이터베이스
   ```

2. **Database 생성**
   ```
   /table 입력 → "Table - Full page" 선택
   ```

3. **속성 설정**
   ```yaml
   이름: Title (기본)
   날짜: Date
   설교제목: Text
   만족도: Select (⭐⭐⭐⭐⭐, ⭐⭐⭐⭐, ...)
   기억에_남는_내용: Text
   소감: Text (Long)
   질문: Text (Long)
   기도제목: Text (Long)
   심방요청: Checkbox
   연락처: Phone
   ```

---

### Step 2: Notion Form 설정

**Notion 내장 Form 기능 사용** (2023년 추가):

1. **Database 우측 상단 "..."**
   → "Forms" → "New form"

2. **Form 설정**
   ```yaml
   제목: 설교 피드백
   설명: 오늘 설교에 대한 의견을 나눠주세요

   필드:
     - 이름 (선택)
     - 만족도 (필수)
     - 기억에 남는 내용
     - 소감
     - 질문
     - 기도제목
     - 심방 요청
     - 연락처
   ```

3. **공개 링크 생성**
   ```
   "Share" → "Share to web" ✅
   → 링크 복사
   ```

---

### Step 3: 응답 확인

**Notion 데이터베이스에서**:
- 실시간으로 새 행 추가
- 다양한 뷰로 확인:
  - 📅 Calendar View (날짜별)
  - 📊 Table View (전체 목록)
  - 📈 Gallery View (카드 형태)

## 방법 3: Typeform (아름다운 UI)

### 🌟 장점

- ✅ **가장 예쁜 UI**
- ✅ 한 번에 한 질문씩 (사용자 친화적)
- ✅ 조건부 로직 (스킵, 분기)
- ✅ 응답률 높음

### ⚠️ 단점

- ❌ 무료 플랜: 월 10개 질문, 10개 응답 제한
- ❌ 유료: $25/월 (무제한)

---

### Step 1: Typeform 생성

1. **Typeform 접속**
   - https://www.typeform.com
   - 회원가입 (무료)

2. **New Typeform**
   - "Start from scratch"

3. **질문 추가**
   ```
   Welcome Screen:
     제목: 설교 피드백
     설명: 오늘 설교가 어떠셨나요?
     버튼: 시작하기

   질문 1: Opinion Scale
     오늘 설교가 도움이 되셨나요?
     1 (아쉬움) - 5 (매우 은혜)

   질문 2: Short Text
     가장 기억에 남는 내용은?

   질문 3: Long Text
     설교 소감을 자유롭게 나눠주세요

   질문 4: Long Text
     궁금한 점이 있으신가요?

   Ending Screen:
     감사합니다!
     여러분의 의견은 큰 힘이 됩니다 🙏
   ```

4. **조건부 로직 (Pro 기능)**
   ```
   만족도 1-2점 → 개선 의견 추가 질문
   심방 요청 "예" → 연락처 입력 질문
   ```

---

## 자동화: Zapier/Make 연동

### 목표

**Google Forms 응답 → 자동으로 Obsidian 마크다운 생성**

---

### 방법 A: Zapier

#### Step 1: Zap 생성

1. **Zapier 로그인**
   - https://zapier.com

2. **Create Zap**

3. **Trigger 설정**
   ```yaml
   App: Google Sheets
   Event: New Spreadsheet Row
   Account: Google 계정 연결
   Spreadsheet: "설교 피드백 응답"
   Worksheet: "Form Responses 1"
   Trigger Column: A (타임스탬프)
   ```

4. **Test Trigger**
   - 최근 응답 확인

---

#### Step 2: Action 설정 (Dropbox/OneDrive)

**Option 1: Dropbox 사용**

```yaml
App: Dropbox
Event: Create Text File

Folder: /Obsidian/우리교회/200.성도관리/240.설교피드백/성도별/{{이름}}/
Filename: {{날짜}}_피드백.md

Content: |
  ---
  type: feedback
  date: {{타임스탬프}}
  sermon_date: {{설교날짜}}
  person: "[[{{이름}}]]"
  satisfaction: {{만족도}}
  tags: [설교피드백]
  ---

  # {{날짜}} 설교 피드백 - {{이름}}

  ## 만족도
  {{만족도}}

  ## 기억에 남는 내용
  {{기억에_남는_내용}}

  ## 소감
  {{소감}}

  ## 질문
  {{질문}}

  ## 기도제목
  {{기도제목}}

  ## 심방 요청
  {{심방요청}}

  ## 연락처
  {{연락처}}

  ---

  **피드백 일시**: {{타임스탬프}}
```

---

**Option 2: Google Drive 사용**

```yaml
App: Google Drive
Event: Create File From Text

Folder: Obsidian/우리교회/200.성도관리/240.설교피드백/
File Name: {{날짜}}_{{이름}}_피드백.md
Content: (위와 동일)
```

## FAQ

### Q1: 성도들이 솔직하게 응답할까요?

**A**: 익명 옵션 제공!

**전략**:
- 이름 필드를 "선택 사항"으로
- "익명 환영" 명시
- "모든 의견은 소중합니다" 강조

**결과**: 솔직한 피드백 증가

---

### Q2: 응답률을 높이려면?

**A**: 동기 부여!

**방법**:
1. **QR 코드 편의성**
   - 주보에 QR 코드 → 스캔만 하면 끝
   - 1-2분이면 완료 강조

2. **즉각적 감사 표현**
   - 카카오톡: "피드백 감사합니다!"
   - 다음 주 설교에서 언급: "지난주 여러분 의견 반영했습니다"

3. **투명한 활용**
   - 피드백이 실제로 반영됨을 보여주기
   - "이번 예화는 여러분 제안 덕분입니다"

---

### Q3: 부정적 피드백은 어떻게 처리하나요?

**A**: 성장의 기회로!

**마인드셋**:
- 비판 ≠ 공격
- 성도의 솔직한 의견 = 목회 개선 기회

**처리 방법**:
1. **감정적 반응 자제**
2. **구체적 개선점 파악**
3. **다음 설교에 반영**
4. **결과 공유**

**예시**:
```
피드백: "설교가 너무 길어요"
→ 분석: 평균 55분
→ 개선: 45분 목표
→ 공유: "여러분 의견 반영하여 시간 조정했습니다"
```

---

### Q4: 개인 질의가 너무 많으면?

**A**: 우선순위 설정

**분류**:
1. **긴급**: 신앙 위기, 심각한 고민 → 24시간 내 답변
2. **일반**: 신학적 질문 → 3일 내 답변
3. **단순**: 교회 행정 문의 → 전도사 위임

**효율화**:
- FAQ 문서 작성 (자주 묻는 질문)
- 그룹 Q&A 세션 (월 1회)

---

### Q5: 자동화가 복잡한데 수동으로도 가능한가요?

**A**: 가능합니다!

**수동 프로세스**:
```
1. Google Forms 응답 확인
2. 수동으로 Obsidian 파일 생성
3. 복사-붙여넣기

소요: 응답 1건당 2-3분
```

**권장**: 처음에는 수동, 익숙해지면 자동화

---

### Q6: 모바일에서도 가능한가요?

**A**: 완벽하게!

**성도 (응답)**:
- ✅ 스마트폰으로 폼 작성 (최적화됨)

**목회자 (확인)**:
- ✅ Obsidian 모바일 앱으로 피드백 확인
- ⚠️ 답변은 PC 추천 (타이핑 편의)

### Q8: 여러 예배(주일 오전/오후/수요)별로 관리 가능한가요?
