# Graph View 활용법

> **"연결이 보이면 통찰이 보입니다"**
>
> 수백 개의 문서가 어떻게 연결되어 있는지 한눈에!

---

## Graph View란?

**개념**: 모든 노트를 점(node)으로, 링크를 선(edge)으로 표시한 네트워크 그래프

**활용**:
- 문서 간 관계 파악
- 고립된 문서 발견
- 지식 클러스터 시각화
- 아이디어 연결 탐색

---

## 열기

### 전체 그래프 보기

**방법 1**: 좌측 사이드바 → 🔗 Graph view 아이콘 클릭

**방법 2**: `Ctrl+G` (단축키)

---

### 로컬 그래프 (현재 노트 중심)

**방법**: 노트 열기 → 우측 상단 `...` → "Open local graph"

**용도**: 현재 문서와 직접 연결된 문서만 보기

---

## 교회 지식 네트워크 예시

### 패턴 1: 중심 허브

**설교 노트가 중심**:
```
     [요한복음 시리즈]
            |
    --------+---------
    |       |        |
[설교1] [설교2] [설교3]
    |       |        |
[주보1] [주보2] [주보3]
```

**의미**: 설교가 주보와 강하게 연결

---

### 패턴 2: 클러스터

**청년부 클러스터**:
```
    [청년부]
      / | \
     /  |  \
[모임] [수련회] [봉사]
  |      |       |
[기록] [기획서] [보고]
```

**의미**: 청년부 관련 문서들이 묶여있음

---

### 패턴 3: 고립된 노드

```
[문서A]    [문서B]    [문서C]

      (링크 없음)
```

**문제**: 다른 문서와 연결 없음 → 찾기 어려움

**해결**: 관련 문서에 링크 추가

---

## 필터 활용

### 폴더별 필터

**설정**:
1. Graph view 열기
2. 좌측 "Filters" 섹션
3. "Search files": 입력

**예시**:
```
path:100.교회운영/120.설교
```

**효과**: 설교 노트만 그래프에 표시

---

### 태그별 필터

**예시**:
```
tag:#설교
```

**효과**: #설교 태그 있는 문서만

---

### 복합 필터

```
path:100.교회운영 tag:#주보
```

**효과**: 교회운영 폴더 + 주보 태그

---

## 색상 구분

### 그룹별 색상 설정

**방법**:
1. Graph view → 좌측 "Groups" 섹션
2. "New group" 클릭
3. 쿼리 입력: `path:100.교회운영/120.설교`
4. 색상 선택 (빨간색)

**효과**: 설교 노트가 빨간색으로!

**그룹 예시**:
- 🔴 설교: `path:120.설교`
- 🔵 주보: `path:110.주보`
- 🟢 심방: `path:220.심방기록`
- 🟡 행사: `path:600.행사`

---

## 실전 활용 사례

### 사례 1: 설교 시리즈 확인

**목표**: "요한복음 시리즈" 설교들이 제대로 연결되었는지 확인

**방법**:
1. [[요한복음 시리즈]] 노트 열기
2. Local graph 열기
3. 연결된 설교 노드 확인

**판단**:
- 모든 설교가 연결됨 ✅
- 일부 설교 고립됨 ⚠️ → 링크 추가 필요

---

### 사례 2: 심방 네트워크

**목표**: 누구를 자주 심방하는지 시각화

**방법**:
1. Graph view 열기
2. Filter: `path:220.심방기록`
3. 크기 큰 노드 = 많이 심방한 성도

---

### 사례 3: 고립된 문서 찾기

**목표**: 링크 없는 문서 발견

**방법**:
1. Graph view 열기
2. 외곽에 홀로 있는 노드 찾기
3. 해당 문서 클릭
4. 관련 문서에 링크 추가

---

## Graph Analysis 플러그인

### 설치

1. Community Plugins → "Graph Analysis" 검색
2. Install → Enable

### 기능

**중심성 분석**:
- 어떤 문서가 가장 중요한가?
- 허브 역할하는 문서는?

**클러스터 탐지**:
- 자동으로 그룹 발견

---

## 시각화 팁

### Tip 1: 노드 크기 = 링크 수

**설정**:
- Graph view → Display
- "Node size": "Links"

**효과**: 링크 많은 문서가 크게 표시

---

### Tip 2: 연결 강도 조절

**설정**:
- Display → "Link thickness": "Links"

**효과**: 링크 많을수록 선이 굵어짐

---

### Tip 3: 물리 엔진 조정

**설정**:
- Forces → "Center force", "Repel force"

**조정**:
- Center force ↑ → 가운데로 모임
- Repel force ↑ → 노드 간 거리 증가

---

## 교회 네트워크 건강도 체크

### 건강한 네트워크 특징

✅ **중심 허브 존재**:
- 주보, 설교 등이 중심

✅ **클러스터 형성**:
- 부서별, 사역별 묶임

✅ **고립 노드 없음**:
- 모든 문서가 최소 1개 링크

✅ **균형 잡힌 분포**:
- 특정 영역에만 집중 안 됨

---

### 문제 있는 네트워크

❌ **고립 노드 많음**:
- 링크 추가 필요

❌ **단일 중심 (Star)**:
- 하나의 노트에만 모든 링크
- → 분산 필요

❌ **분리된 섬들**:
- 여러 그룹이 서로 연결 없음
- → 브릿지 노트 생성

## MOC (Map of Content) 활용

### MOC란?

**개념**: 여러 노트를 모아놓은 목차 노트

**예시**: `교회운영_MOC.md`

```markdown
# 교회운영 MOC

## 주보
- [[2025-01-05 주보]]
- [[2025-01-12 주보]]
- [[2025-01-19 주보]]

## 설교
- [[요한복음 시리즈]]
- [[히브리서 시리즈]]

## 회의록
- [[운영위원회 MOC]]
```

**Graph View에서**:
- MOC가 중심 허브 역할
- 모든 관련 노트 연결

## 스크린샷 활용

### 그래프 캡처

**방법**:
1. Graph view 열기
2. 원하는 각도 조정
3. `PrtSc` (스크린샷)
4. 보고서에 첨부

**활용**:
- 연간 보고서
- 공동의회 자료
- 팀 교육 자료

**예시 문구**:
> "우리 교회의 지식은 이렇게 연결되어 있습니다"
